# 要件定義書 — ステータスバー機能

## 1. 概要

SpecPrompt の画面下部に常設ステータスバーを配置し、「現在の Git ブランチ名」および「アクティブファイルのビューア種別」を一目で把握できるようにする。

---

## 2. 背景・動機

### 現状の課題

- 現在のアプリ UI にはブランチ状態を確認できる領域がない。開発者はターミナルに切り替えて `git branch` を実行するか、別途エディタ／CLI で確認する必要がある。
- 作業中のファイルがどのビューアで表示されているか（Markdown プレビューか、コードビューアか）を UI 上で明示する導線がない。

### 狙い

- **誤作業の予防**: ユーザーが「main ブランチと気付かずに生成 AI へ編集指示を出す」といった事故を防ぐ。
- **認知負荷の軽減**: ブランチ情報を常時表示することで、コンテキストスイッチを減らす。
- **VS Code 代替としての完成度向上**: 軽量ながらエディタ的な常識的 UI を備えることで、SpecPrompt のプロダクト価値を底上げする。

---

## 3. 対象ユーザー・ユースケース

- **対象ユーザー**: SpecPrompt を日常的に利用する個人開発者／仕様駆動開発の実践者
- **代表的なユースケース**:
  1. コンテンツビューアで Markdown を編集指示する前に、ステータスバーのブランチ名を目視確認して作業ブランチが想定どおりか確かめる。
  2. ツリーから開いたファイルが「Markdown」か「Code」か「Plain」かを一目で把握する。
  3. ブランチを切り替え（ターミナルで `git checkout` 実行）後、自動的にステータスバーの表示が更新されることで切り替え完了を確認する。

---

## 4. 機能要件

### FR-01: Git ブランチ表示

| 項目 | 内容 |
|---|---|
| 対象 | アクティブプロジェクト（`useAppStore` の active project cwd）が Git リポジトリである場合 |
| 実現方式 | Rust 側で `git rev-parse --abbrev-ref HEAD` を実行して取得。フロントから `invoke("git_branch", { cwd })` で呼び出す |
| 表示形式 | Git アイコン + ブランチ名（例: ` feature/status-bar`） |
| 検出不可時 | Git リポジトリでない、または取得失敗の場合は非表示（スペースを詰める） |
| detached HEAD 時 | 短縮コミット SHA を表示（例: ` 63f33f1`） |

### FR-02: ファイル種別表示

| 項目 | 内容 |
|---|---|
| 対象 | コンテンツモード（`activeMainTab === 'content'`）かつアクティブタブにファイルがある場合 |
| 実現方式 | 既存の `getViewMode(filePath)` を流用し、`markdown` / `code` / `image` / `plain` の 4 値を日本語ラベル化 |
| 表示形式 | 種別アイコン + ラベル（例: `Markdown`, `Code`, `Image`, `Plain`） |
| 非表示条件 | ターミナルモード、ファイル未選択、分割レイアウトでどちらのペインもフォーカスが当たっていない場合 |

### FR-03: 自動更新

| 項目 | 内容 |
|---|---|
| Git ブランチ | `.git/HEAD` のファイル監視（tauri-plugin-fs の `watch`）で変更を検知して即時再取得 |
| ファイル種別 | Zustand ストア（`contentStore.activeTabId`）の変化に追従して都度再計算（React state 経由） |
| 初回取得 | プロジェクト切替時・アプリ起動時にそれぞれ 1 回実行 |

### FR-04: 表示制御（モード連動）

| 項目 | 内容 |
|---|---|
| コンテンツモード時 | Git ブランチ + ファイル種別を表示 |
| ターミナルモード時 | Git ブランチのみ表示（ファイル種別は非表示） |
| プロジェクト未オープン時 | ステータスバー枠は維持したうえで中身を空にする |

---

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | ステータスバー常時描画による追加メモリ消費は 5MB 以下。Git ブランチ取得の IPC 往復は 100ms 以内を目標 |
| レイアウト | 高さ 28px（`h-7` 相当）。既存の MainArea / ContentArea の高さ計算を調整 |
| 対応 OS | macOS / Windows / Linux（Tauri v2 のマルチ OS 対応に準拠） |
| 可視性 | 既存の CSS 変数（`--color-bg-elevated`, `--color-border`, `--color-text-muted`）を使用してテーマ統一 |
| 互換性 | 既存の `git_status` コマンド・ファイルツリー・ビューア動作への影響ゼロ |

---

## 6. スコープ

### 含むもの

- 画面下部へのステータスバー常設
- Git ブランチ名取得・表示（`.git/HEAD` 監視による自動更新を含む）
- ファイル種別表示（既存 `getViewMode` の流用）
- コンテンツ／ターミナルモード切替への追従

### 含まないもの

- ahead/behind 表示、変更ファイル数表示（将来拡張の余地として残す）
- ステータスバー項目の表示 ON/OFF 切替 UI（初期リリースは固定）
- クリックによるブランチ切替ポップアップ
- カーソル位置・エンコーディング・改行コード表示
- シェル種別・cwd 等ターミナル情報の表示

---

## 7. 既存コードベースとの関係

### 再利用する既存実装

- `src/lib/viewMode.ts` — `getViewMode(filePath)` で拡張子→ビューア種別を判定済み。そのまま活用する。
- `src/stores/appStore.ts` — `activeMainTab`（`'content' | 'terminal'`）とアクティブプロジェクトを提供。表示制御の判定に利用。
- `src/stores/contentStore.ts` — `primary`/`secondary` それぞれのアクティブタブ `filePath` を提供。ファイル種別表示の入力源とする。
- `src-tauri/src/commands/git.rs` — `git_status` コマンドが `std::process::Command` で `git` を呼ぶパターンを確立済み。同ファイルに `git_branch` を追加する。
- `src/lib/tauriApi.ts` — `getGitStatus` と同じラッパーパターンで `getBranch(cwd)` を追加する。
- `src/index.css` の CSS 変数群 — 背景・枠線・テキスト色トークンを流用。

### 改修対象の既存実装

- `src/components/Layout/AppLayout.tsx` — 最上位レイアウトにステータスバー配置用の垂直 flex コンテナを挿入。
- `src/components/MainArea/MainArea.tsx` — `h-full` 前提の高さ計算を、ステータスバー分を控除する形に変更。
- `src/components/ContentView/ContentArea.tsx` — 分割レイアウトの高さ計算を合わせて調整。

### 新規実装

- `src-tauri/src/commands/git.rs` に `git_branch(cwd)` コマンドを追加（`git rev-parse --abbrev-ref HEAD`、detached HEAD は短縮 SHA）。
- `src/lib/tauriApi.ts` に `getBranch(cwd: string): Promise<string | null>` を追加。
- `src/components/StatusBar/StatusBar.tsx` を新規作成（ブランチとファイル種別の表示を担当）。
- `src/hooks/useGitBranch.ts`（仮）— `.git/HEAD` 監視 + IPC 取得のカスタムフック。

---

## 8. 制約・前提

- Git がインストールされていない環境では `git_branch` はエラーを返すため、ステータスバーは空表示にフォールバックする。
- Tauri v2 の capability（`fs:read-all`）が `.git/HEAD` の監視に必要。既存設定で満たされているか確認する。
- アクティブプロジェクトが変更された場合、従前のプロジェクトに対する `.git/HEAD` 監視は適切に解除する。

---

## 9. 成功指標

- 新しいブランチを `git checkout` で作成した直後（1 秒以内）にステータスバーの表示が更新される。
- ファイルを切り替えた瞬間にファイル種別ラベルが更新される。
- アプリ全体のメモリフットプリント増加が 5MB 以下に収まる。
- ユーザーが「間違ったブランチで編集指示を出した」という誤作業を（自己申告ベースで）ゼロにする。
