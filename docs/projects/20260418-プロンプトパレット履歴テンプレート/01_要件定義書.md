# 要件定義書 — プロンプトパレット 履歴・テンプレート機能

## 1. 概要

プロンプトパレット（`F4` で起動する送信用ダイアログ）に、過去の送信内容を呼び戻す**履歴機能**と、定型文を呼び出す**テンプレート機能**を追加する。いずれも「選択 → textarea に流し込む → そのまま、または編集して `Cmd+Enter` / `Ctrl+Enter` で送信」を一貫した主動線とする。

---

## 2. 背景・動機

### 現状の課題

- 同じプロンプトを再送信する場合、毎回手入力または外部エディタからコピペする必要がある
- 仕様駆動開発で頻用する定型プロンプト（「設計書を読み込んで要件を要約」「指定ファイルのテストを書き起こす」等）の呼び出しコストが高い
- パレットは送信で draft をクリアする仕様（`promptPaletteStore.clearDraft`）のため、誤送信後に同じ内容を手で復元するしかない

### 狙い

- よく使うプロンプトを即呼び出し → 編集なし再送、または軽微な編集のみで送信できるようにする
- プレースホルダ（`{{path}}` 等）で動的な差し込みを可能にし、仕様駆動開発のルーチン入力を高速化する
- 既存の textarea 中心の UX を壊さず、キーボード主導で完結できる導線を用意する

---

## 3. 対象ユーザー・ユースケース

- **対象ユーザー**: SpecPrompt を日常的に使用し、AI エージェント CLI（Claude Code / Codex / Aider 等）に繰り返しプロンプトを送る開発者

- **代表的なユースケース**:
  1. 直前に送信したプロンプトを微修正して再送する（`↑` キー 1 回で復元）
  2. 「設計書を読んで要約」等の定型テンプレートを呼び出し、`{{path}}` を当該ファイルに差し替えて送信する
  3. テンプレート一覧からスクロール・検索して任意のものを textarea に流し込み、そのまま送信する
  4. 過去 100 件の履歴を検索（fuzzy）して特定プロンプトを復元する
  5. よく使うプロンプトをテンプレートとして保存する（履歴から昇格、または新規作成）

---

## 4. 機能要件

### FR-01: 直近履歴の `↑` / `↓` 巡回

| 項目 | 内容 |
|---|---|
| 対象 | プロンプトパレット内 textarea |
| 実現方式 | textarea が**空のときのみ** `↑` で直近履歴を textarea に流し込み、連続 `↑` でさらに過去、`↓` で新しい側へ戻る。最新より新しい側で `↓` を押すと空に戻る |
| 既定挙動 | 流し込み後も textarea にフォーカスは残し、末尾にカーソルを置く。`Esc` で巡回状態をリセット |
| 発動条件 | IME 変換中ではない、かつ textarea 値が空、かつ履歴が 1 件以上ある |

### FR-02: 履歴ドロップダウン

| 項目 | 内容 |
|---|---|
| 対象 | パレット内のインライン・ドロップダウン（別モーダルにしない） |
| 起動方法 | ヘッダ「履歴」アイコン、またはショートカット `Cmd+H` / `Ctrl+H`（パレット内のみ） |
| 表示内容 | 保存された送信履歴を新しい順に一覧。各行は 1 行プレビュー（長文は省略）＋相対日時 |
| 操作 | 検索入力で fuzzy フィルタ、`↑`/`↓` で選択、`Enter` で textarea に流し込み、`Esc` で閉じる |
| 送信 | 流し込み後は通常の textarea 状態。送信は `Cmd+Enter` / `Ctrl+Enter` のみ（選択 = 即送信はしない） |
| 件数上限 | 直近 100 件（超過分は自動で古い側から破棄） |
| 重複排除 | 完全一致の連続重複は 1 件にまとめる（間に別プロンプトが入った再登場は別エントリとして残す） |

### FR-03: テンプレートドロップダウン

| 項目 | 内容 |
|---|---|
| 対象 | パレット内のインライン・ドロップダウン |
| 起動方法 | ヘッダ「テンプレ」アイコン、またはショートカット `Cmd+T` / `Ctrl+T`（パレット内のみ） |
| 表示内容 | 保存済みテンプレートを名前順に一覧。名前＋本文 1 行プレビュー |
| 操作 | 検索入力で fuzzy フィルタ、`↑`/`↓` で選択、`Enter` で textarea に流し込み |
| 流し込み後 | 本文中に `{{placeholder}}` があれば最初の 1 つを選択状態にし、`Tab` で次のプレースホルダへ移動できる（= スニペット風ナビゲーション） |

### FR-04: インライン候補サジェスト（`/` プレフィックス）

| 項目 | 内容 |
|---|---|
| 対象 | textarea 内 |
| トリガ | textarea の先頭で `/` を入力、または空 textarea での `/` 入力 |
| 挙動 | ポップオーバーでテンプレ名の候補を表示。以降の入力でフィルタ、`↑`/`↓` で選択、`Enter` で置換的に流し込み、`Esc` でポップオーバーのみ閉じる |
| 誤爆防止 | 行頭以外の `/`（コマンドオプション記述など）では発動しない |

### FR-05: テンプレートエディタ

| 項目 | 内容 |
|---|---|
| 対象 | テンプレートの CRUD |
| 起動方法 | テンプレ一覧から「新規作成」「編集」「削除」、または履歴行の「テンプレに保存」メニュー |
| 編集項目 | 名前（必須・ユニーク）、本文（複数行・プレースホルダ記法対応）、タグ（任意・将来拡張用フィールドとして定義のみ） |
| プレースホルダ | `{{path}}`, `{{project}}`, `{{selection}}` を基本セットとして定義。流し込み時に選択状態の `{{...}}` を Tab で巡回 |
| 永続化 | アプリ設定ストアに保存（下記 FR-07） |

### FR-06: 履歴からテンプレートへの昇格

| 項目 | 内容 |
|---|---|
| 対象 | 履歴行のアクション |
| 操作 | 履歴行のコンテキストメニューまたは選択中の操作キー（例: `Cmd+S`）で「テンプレートとして保存」→ 名前入力モーダル |
| 既定値 | 本文は履歴原文、名前は未入力必須 |

### FR-07: 永続化

| 項目 | 内容 |
|---|---|
| 履歴 | `promptPaletteStore` を `persist` middleware でラップし、localStorage に保存（ブラウザ標準の永続化・暗号化不要のプレーンテキスト） |
| テンプレート | 同上、あるいは将来的に `~/.config/spec-prompt/config.json` へ移行する（現時点では localStorage で統一） |
| マイグレーション | `persist` の `version` / `migrate` でスキーマ進化に追随 |

### FR-08: 送信挙動

| 項目 | 内容 |
|---|---|
| 送信時の記録 | 送信成功時に本文を履歴へ push（末尾空白は trim 済み）。送信失敗時は履歴に残さない |
| プライバシー | 履歴は端末ローカルにのみ保存。外部送信・同期は行わない |

### FR-09: 国際化

`src/i18n/locales/ja.json` と `en.json` に以下のキー名前空間を追加：

- `promptPalette.history.*`（アイコン label、空状態、検索プレースホルダ、アクション名）
- `promptPalette.template.*`（同上、エディタ用キー含む）
- `promptPalette.hint.*`（既存にショートカットヒントを追加）

---

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | 履歴 100 件、テンプレ 50 件規模で一覧レンダ 16 ms 以下。fuzzy 検索入力は 60 fps を維持 |
| メモリ | 永続化データは合計 1 MB 以下を目安（1 件あたり平均 1 KB 想定） |
| 対応OS | macOS / Windows / Linux（既存パレットに準じる） |
| 互換性 | 既存の `Cmd+Enter` / `Ctrl+Enter` 送信、`Esc` クローズ、IME 変換ガードはすべて維持 |
| アクセシビリティ | ドロップダウンは `role="listbox"`、選択中は `aria-selected`。prefers-reduced-motion 時はフラッシング演出を抑止（既存の `lastInsertAt` ロジックと整合） |
| プライバシー | 履歴・テンプレは端末ローカルのみ。テレメトリ送信なし |

---

## 6. スコープ

### 含むもの

- FR-01 〜 FR-09（履歴・テンプレ・プレースホルダ・`↑` 巡回・`/` サジェスト・エディタ・昇格・永続化・i18n）
- 既存 `PromptPalette.tsx` および `promptPaletteStore.ts` への改修
- `PathPalette` の UI パターンを参考にした `PromptHistoryDropdown` / `PromptTemplateDropdown` の新規実装

### 含まないもの

- テンプレートのクラウド同期・共有・インポート/エクスポート（将来検討）
- 履歴の暗号化保存（localStorage 平文で運用）
- タグによる絞り込み UI（データ構造としてのフィールドは定義するが UI は未実装）
- `PathPalette` 側の変更（本機能は `PromptPalette` に閉じる）
- 送信履歴の複数端末同期

## 8. 制約・前提

- Tauri v2 / React 19 / TypeScript / Zustand persist middleware を使用する（`appStore` と同じ構成）
- 永続化先は localStorage。ユーザーが別プロファイルで SpecPrompt を起動すると履歴も別になる（同一端末 1 ユーザー前提）
- IME 変換中の誤発火防止は既存ガード（`isComposing` と `e.nativeEvent.isComposing` の OR）を流用
- `PathPalette` とパレット間の ↑↓ 衝突は発生しない（`PathPalette` は別ダイアログ）

---

## 9. 成功指標

- 定型プロンプトの再送信にかかるキーストローク数が **10 ストローク以内**（`F4` → `Cmd+T` → テンプレ選択 → `Enter` → `Cmd+Enter`）で完結すること
- 直近の再送は **3 ストローク以内**（`F4` → `↑` → `Cmd+Enter`）で完結すること
- 既存の送信・IME・フォーカス挙動にリグレッションが発生しないこと（`PromptPalette.test.tsx` / `promptPaletteStore.test.ts` の既存テストが全件グリーン）
- ユーザー設定（履歴 / テンプレ）が SpecPrompt を再起動しても保持されること
