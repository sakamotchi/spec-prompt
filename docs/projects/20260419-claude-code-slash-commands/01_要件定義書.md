# 要件定義書 — Claude Code スラッシュコマンドサジェスト統合

**バージョン**: 1.0
**作成日**: 2026-04-19
**対象機能 ID**: FR-15 拡張（PE-46 を具体化）
**スコープ**: Phase A + Phase B（Phase C は将来拡張として明記）

---

## 1. 概要

プロンプト編集パレットの `/` インラインサジェスト（`SlashSuggest`）に、**Claude Code の組み込みスラッシュコマンド** と **ユーザー／プロジェクト Skill** を候補として混在表示する機能を追加する。

現状、`/` サジェストは SpecPrompt 内で定義した `PromptTemplate` のみを候補とするため、Claude Code 側の `/resume` や `/clear` といった実コマンドがサジェストされず、ユーザーは候補を把握するためにアプリ外の記憶に頼る必要がある。本機能により、プロンプトパレット上で Claude Code のスラッシュコマンド体系を完結的に発見・挿入できるようにする。

---

## 2. 背景・動機

### 現状の課題

- `SlashSuggest` は `PromptTemplate` 単一種別のみ表示で、Claude Code 実コマンドの発見性がゼロ
- steering `docs/steering/features/prompt-palette.md` の **PE-46** に「将来 `SlashSuggestItem = { kind: 'template' | 'command' }` で混在表示」と方針のみ記載されているが、具体仕様・実装タスクは未定
- Claude Code のコマンド体系は **5 系統**（組み込み・バンドル Skill・ユーザー Skill・プロジェクト Skill・プラグイン Skill）あり、ユーザーが全てを記憶することは非現実的

### 狙い

- プロンプトパレット内で Claude Code コマンド／Skill を **fuzzy 検索・プレビュー・挿入** できるようにする
- 既存 `PromptTemplate` とシームレスに共存させ、ユーザーの「書き始めのコンテキスト切り替え」を最小化
- 外部の `claude` CLI 呼び出しに依存せず、**静的リスト + ローカルファイルスキャン** で実現し、起動レイテンシゼロを維持

---

## 3. 対象ユーザー・ユースケース

- **対象ユーザー**: SpecPrompt 上で Claude Code（対話 CLI）を操作する開発者
- **代表的なユースケース**:
  1. プロンプトパレットで `/` を打ち、`/resume` を選択してセッション再開コマンドを挿入
  2. `/` → `rev` と入力すると組み込みの `/review` と自作テンプレ `/review-pr` が両方候補に出る
  3. プロジェクト固有 Skill（`<projectRoot>/.claude/skills/deploy-check/SKILL.md`）を fuzzy で絞り込んで挿入
  4. ユーザー Skill（`~/.claude/skills/my-audit/SKILL.md`）を別プロジェクトでも呼び出す

## 4. 機能要件

### FR-01: SlashSuggest のアイテム型拡張（Phase A 基盤）

| 項目 | 内容 |
|---|---|
| 対象 | `src/components/PromptPalette/SlashSuggest.tsx` 及び関連型 |
| 実現方式 | `SlashSuggestItem` 判別共用体型を導入。`kind: 'template' \| 'builtin' \| 'user-skill' \| 'project-skill'` の 4 種別で単一リストに集約 |
| 既定挙動 | fuzzy マッチは `name` に対して行い、候補は 最大 10 件（現状維持）。種別バッジを左端に表示 |

### FR-02: 組み込みコマンド静的リスト（Phase A）

| 項目 | 内容 |
|---|---|
| 対象 | Claude Code 本体の組み込みコマンド・バンドル Skill |
| 実現方式 | `src/lib/builtInCommands.ts`（新規）に `BUILT_IN_COMMANDS: BuiltInCommand[]` として静的定義。`name` / `description` / `kind: 'builtin'` を持つ |
| 既定挙動 | アプリ起動時から常にサジェスト候補に含まれる。ユーザーによる編集・削除は不可 |
| 収録対象 | `/resume` `/clear` `/compact` `/help` `/model` `/config` `/context` `/cost` `/usage` `/doctor` `/feedback` `/login` `/logout` `/exit` `/permissions` `/effort` `/theme` `/rename` `/rewind` `/branch` `/diff` `/desktop` `/teleport` 等の組み込みコマンド、および `/debug` `/simplify` `/batch` `/loop` `/claude-api` 等のバンドル Skill |

### FR-03: ユーザー Skill スキャン（Phase B）

| 項目 | 内容 |
|---|---|
| 対象 | `~/.claude/skills/<name>/SKILL.md` |
| 実現方式 | Rust コマンド `list_claude_skills` が `~/.claude/skills/` を列挙し、各 `SKILL.md` の YAML frontmatter をパースして `SkillMetadata` を返す |
| 既定挙動 | パレット初回オープン時に非同期ロード。結果は `promptPaletteStore` の runtime state にキャッシュ（永続化対象外） |

### FR-04: プロジェクト Skill スキャン（Phase B）

| 項目 | 内容 |
|---|---|
| 対象 | `<projectRoot>/.claude/skills/<name>/SKILL.md` |
| 実現方式 | `list_claude_skills` の第 2 引数に `projectRoot` を渡し、プロジェクト側もスキャン |
| 既定挙動 | 同名 Skill がユーザー側とプロジェクト側にある場合、Claude Code の解決優先度に合わせて **ユーザー側を優先表示**（プロジェクト側は隠す） |

### FR-05: Skill frontmatter フィルタ（Phase B）

| 項目 | 内容 |
|---|---|
| 対象 | SKILL.md の YAML frontmatter |
| 実現方式 | `serde_yaml` でパース。以下のフィールドを認識: `name`, `description`, `argument-hint`, `user-invocable`, `disable-model-invocation` |
| 既定挙動 | `user-invocable: false` のスキルは候補から除外（Claude 自動呼び出し専用のため）。`disable-model-invocation: true` は手動発動対象としてむしろ含める |

### FR-06: 候補選択時の挿入挙動

| 項目 | 内容 |
|---|---|
| 対象 | `PromptPalette.tsx` の `handleSlashSelect` |
| 実現方式 | `kind` で分岐。`template` は既存 `applyTemplateBodyToDraft`（全置換＋プレースホルダ選択）。`builtin` / `user-skill` / `project-skill` は `/<name> `（末尾スペース）を draft に挿入 |
| 既定挙動 | 挿入のみで送信はしない（誤爆防止、PE-35 の思想を踏襲）。送信は引き続き `⌘Enter` 必須 |

### FR-07: UI 差分化（セクション／バッジ）

| 項目 | 内容 |
|---|---|
| 対象 | `SlashSuggest` の候補行レイアウト |
| 実現方式 | 種別ごとに見出し（`Claude Code` / `User Skills` / `Project Skills` / `Templates`）でグルーピング。各行の左端に小バッジ（`CMD` / `USER` / `PROJ` / `TPL`）を表示。組み込みコマンドとバンドル Skill は Claude Code 本体由来のため "Claude Code" セクションに統合 |
| 既定挙動 | 検索クエリが空でも上記順序。fuzzy 時は種別横断で relevance 降順。同名衝突はユーザー優先で 1 件のみ表示 |

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | Phase A はランタイム無オーバーヘッド（静的配列）。Phase B の Skill スキャンはパレット初回オープン時 1 回、**200ms 以内**の完了を目標。キャッシュヒット時は 0ms |
| メモリ | Skill 最大 500 件想定で追加メモリ **1MB 未満** |
| 対応 OS | macOS / Windows / Linux（Tauri v2 サポート範囲） |
| セキュリティ | 既存 `fs:read-all` capability で完結。追加権限なし。`~/.claude/skills/` 以外はスキャンしない |
| 互換性 | 既存 `PromptTemplate` の永続化形式・`SlashSuggest` のキーボード操作（↑↓/Enter）は非破壊で継続 |
| I18N | 種別バッジ・セクション見出しは `src/i18n/locales/{ja,en}.json` に追加 |

## 6. スコープ

### 含むもの（Phase A + B）

- 組み込みコマンドの静的リスト定義と UI 混在表示
- バンドル Skill（`/debug` `/simplify` 等）の静的リスト収録
- `~/.claude/skills/` と `<projectRoot>/.claude/skills/` のファイルスキャン
- SKILL.md YAML frontmatter のパース（`name` / `description` / `user-invocable` / `argument-hint`）
- 種別バッジ + セクショングルーピング UI
- 同名衝突時のユーザー Skill 優先表示

### 含まないもの

- **Phase C（プラグイン Skill）**: `<plugin>/skills/<name>/SKILL.md` の `/plugin-name:name` 形式。有効化判定方法の調査が必要なため、ニーズが高まってから着手
- **MCP プロンプト統合**: `/mcp__<server>__<prompt>` 形式。Claude Code プロセスとの通信手段が未確立
- **Skill 実行時の引数入力 UI**: `argument-hint` を表示するのみで、構造化入力フォームは提供しない
- **Skill body のプレビュー**: 候補行の右側には `description` のみ表示。SKILL.md 本文のレンダリングはしない
- **組み込みコマンド一覧の自動追従**: Claude Code のバージョンアップで増えた組み込みは、手動で `builtInCommands.ts` を更新する運用
- **Skill エディタ**: SpecPrompt から SKILL.md を作成・編集する UI は提供しない（外部エディタで編集）

## 7. 既存コードベースとの関係

### 再利用する既存実装

- `src/components/PromptPalette/SlashSuggest.tsx` — fuzzy 検索・ActiveIndex 管理・キー操作ハンドラは構造ごと流用。候補データソースのみ拡張
- `src/lib/slashQuery.ts` — `parseSlashQuery` の regex は改修不要。Phase A/B とも `/` + 空白なしトークンで十分
- `src/lib/templateApply.ts` — `applyTemplateBodyToDraft` はテンプレ系統でそのまま使用
- `src/stores/promptPaletteStore.ts` — `templates` state の永続化枠組みをそのまま維持。Skill スキャン結果は runtime state に別管理
- `src/components/PromptPalette/PromptPalette.tsx` — `handleSlashSelect` の分岐追加のみ
- `src-tauri/src/commands/filesystem.rs` — `read_file` を内部利用し Rust 側で YAML パース
- `src-tauri/capabilities/default.json` — 既存の `fs:read-all` で `~/.claude/skills/` 読み取り可。追加権限なし

### 改修対象の既存実装

- `SlashSuggest.tsx` — 単一 kind 前提から複数種別対応へ。セクション区切り・バッジ・アイコン切替を追加
- `promptPaletteStore.ts` — `SlashSuggestItem` 判別共用体型を `src/lib/` に切り出し、store 側は Skill キャッシュ state（`skills: SkillMetadata[]`）を追加
- `PromptPalette.tsx` の `handleSlashSelect` — kind による分岐（template 全置換 vs command/skill の `/<name> ` 挿入）

### 新規実装

- `src/lib/builtInCommands.ts` — 組み込み + バンドル Skill の静的リスト
- `src/lib/slashSuggestItem.ts` — `SlashSuggestItem` 型定義（または `promptPaletteStore.ts` に同居）
- `src-tauri/src/commands/skills.rs` — `list_claude_skills` コマンド実装
- `src-tauri/Cargo.toml` に `serde_yaml` 依存追加
- `src/lib/tauriApi.ts` に `listSkills(projectRoot?)` wrapper 追加

## 8. 制約・前提

- Claude Code の組み込みコマンド一覧は **2026-04 時点の公式ドキュメント**（https://code.claude.com/docs/en/commands.md）を基準とする。将来の増減は手動追随
- `~` 展開は Rust 側 `std::env::var("HOME")` で実施（Windows は `USERPROFILE`）
- Skill スキャンは **ディレクトリ列挙 + `SKILL.md` 読み込み** のみ。`allowed-tools` / `model` 等の frontmatter は取得するが UI 表示はしない
- Skill 同名衝突時はユーザー側優先。Claude Code 公式の優先度（企業 > 個人 > プロジェクト > プラグイン）のうち、アプリでハンドルできる「個人 > プロジェクト」部分のみを実装
- プラグイン Skill の解決（Phase C）は将来拡張。MCP プロンプトは恒久スコープ外

---

## 9. 成功指標

- プロンプトパレットで `/` を入力したとき、**組み込みコマンド 20 件以上** と **インストール済み Skill すべて** が候補表示されること
- `~/.claude/skills/` に配置した SKILL.md を、フォルダ追加後の**アプリ再起動なし**でパレット再オープンにより候補に反映できること
- fuzzy 検索で種別横断の絞り込みが 50ms 以内に完了すること（500 件ロード時）
- 既存 `PromptTemplate` の挙動に**回帰バグが発生しない**こと（既存テストがグリーンを維持）
- 種別バッジ・セクション見出しで **テンプレ / コマンド / スキル** の発生源がユーザーにとって明確であること
