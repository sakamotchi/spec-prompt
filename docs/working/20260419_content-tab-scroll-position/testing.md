# テスト手順書 - content-tab-scroll-position

関連 Issue: [#9](https://github.com/sakamotchi/spec-prompt/issues/9)

## 概要

コンテンツタブ切替時のスクロール位置保持機能のテスト手順を記載する。GUI 操作による手動テストを主とし、ストアロジックのみ軽く自動テストを用意する。

## 実施結果サマリー

- 実施日: 2026-04-19
- 実施者: ユーザー
- 結果: 全ケース OK（手動テスト 10 ケース、エッジケース 4 件、回帰 7 項目）

## 前提条件

- `npx tauri dev` でアプリが起動していること
- 十分な長さ（スクロールが発生する程度）のマークダウンファイル複数（例: `A.md`, `B.md`）を持つプロジェクトが開かれていること
- 同様に十分な長さのコードファイル（例: `a.ts`, `b.ts`）・プレーンテキストファイル（例: `a.txt`, `b.txt`）も用意

## 手動テスト

### ケース 1: Markdown タブ切替時のスクロール位置保持（Issue #9 再現シナリオ）

**手順:**

1. 十分な長さの `A.md` をダブルクリックでタブを開く
2. 続けて `B.md` をダブルクリックで別タブとして開く
3. `A.md` タブをアクティブにし、ページ最下部までスクロール
4. `B.md` タブをクリックして切り替える
5. `B.md` の表示位置を確認
6. `A.md` タブに戻る

**期待結果:**

- 手順 5 時点で `B.md` が **上端（`scrollTop = 0`）** から表示される
- 手順 6 で `A.md` に戻ると、手順 3 で止めた **最下部付近** のスクロール位置が復元される

**確認結果:**

- [ ] OK / NG

---

### ケース 2: Markdown - A/B 双方向で別々のスクロール位置を保持

**手順:**

1. `A.md` を 1/3 ほどスクロール（例: `scrollTop` ≈ 400px 相当）
2. `B.md` に切替、`B.md` を 2/3 ほどスクロール
3. `A.md` に戻る
4. 再度 `B.md` に戻る

**期待結果:**

- 手順 3: `A.md` は 1/3 の位置で復元される
- 手順 4: `B.md` は 2/3 の位置で復元される

**確認結果:**

- [ ] OK / NG
