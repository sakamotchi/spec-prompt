# テスト手順書 - macos-window-tabs-f2-f3

F2+F3 で担保する **受け入れ基準 1, 6, 7**（元要件 `docs/local/20260418-macOSウィンドウタブ統合/01_要件定義書.md` §7）の手動確認と、自動テストの実行方法をまとめる。

## 概要

- F2: `⌘N` / File > New Window での新規ウィンドウ生成（受け入れ基準 1）
- F3: 複数ウィンドウで状態が独立、PTY ID が衝突しない（受け入れ基準 6, 7）

主要検証は手動 E2E。persist キー分離は vitest でユニット検証。

## 前提条件

- `feature/macos-window-tabs` ブランチをチェックアウト済み
- `npm install` 済み
- `.app` は F1 と同様、`npx tauri build` → Finder 起動が確実。`npx tauri dev` でも UI 操作は確認可能
- macOS で検証（Windows / Linux は F4）
- macOS システム設定「書類をタブで開く」は **"常に"** または **"しない"** のどちらでも OK（F1 で両対応済み）
- テスト用プロジェクトフォルダを 3 つ用意（例: `~/tmp/proj-a`, `~/tmp/proj-b`, `~/tmp/proj-c`）

### ケース 3: プロンプト編集パレット表示中の `⌘N`（macOS 標準挙動に委譲）

**背景:**

`⌘N` は File > New Window メニューの accelerator として登録されているため、macOS/AppKit がキー入力を menu 側で捕捉し、webview の keydown 抑止ロジック（パレット open 時の早期 return）はバイパスされる。v1 ではこれを受け入れ、**macOS 標準挙動に合わせて「パレット表示中でも `⌘N` は新規ウィンドウを開く」** とする。

※ 以前は JS 側にも `⌘N` ハンドラを持っていたが、menu accelerator と二重発火して 2 ウィンドウ同時生成していた。この二重発火を修正するため JS 側の `⌘N` ハンドラは削除済み。

**手順:**

1. `Cmd+Shift+P` でプロンプト編集パレットを開き、textarea に何か入力する。
2. `⌘N` を押す。
3. 新規ウィンドウを閉じてから元のウィンドウに戻り、パレットを再度開く（または前面に戻す）。

**期待結果:**

- 新規ウィンドウが **1 つだけ** 開く（二重発火していない）
- 元のウィンドウのパレット state はメモリ保持されている（drafts は `ptyId` キーで保持されるため、元のターミナルタブで `Cmd+Shift+P` を押し直すと下書きが復元される）
- パレットは新規ウィンドウが開いた時点で元ウィンドウの裏になるが、自動で閉じることはない（= `close()` が呼ばれない）

**確認結果:**

- [x] OK（2026-04-18 確認）
- 新規ウィンドウの個数: [x] 1 個（isFocused ガード修正後、正しく 1 個だけ開く）

### ケース 4: ウィンドウごとに状態が独立（受け入れ基準 6）

**手順:**

1. SpecPrompt でプロジェクト A を開く（ウィンドウ A）。
2. `⌘N` で新規ウィンドウ（ウィンドウ B）を開き、プロジェクト B を開く。
3. ウィンドウ A で以下を操作：
   - メインタブを「ターミナル」に切替
   - ファイルツリーのディレクトリを複数展開
   - 特定ファイルを選択
4. ウィンドウ B で状態を確認。
5. 逆にウィンドウ B で以下を操作：
   - メインタブを「コンテンツ」のまま維持
   - 別のディレクトリを展開
6. ウィンドウ A の状態を再確認。

**期待結果:**

- ウィンドウ A の `activeMainTab` / `expandedDirs` / `selectedFile` はウィンドウ B に伝播しない
- ウィンドウ B の操作もウィンドウ A に伝播しない
- 各ウィンドウを閉じて再起動しても、`windowSession.ts` で各ウィンドウの projectRoot は復元される

**確認結果:**

- [x] OK（2026-04-18 確認）

## 補足

- **Windows / Linux 回帰は F4 で実施**。F2+F3 の受け入れ基準は macOS 限定だが、`⌘N` / persist 分離自体は全プラットフォームで動作するため、Windows / Linux でも `Ctrl+N` で新規ウィンドウが開く動作確認はしておきたい（tasklist の任意項目）。
- F2-2（Tauri メニュー）を省略した場合、ケース 2 はスキップ。メニュー項目の代わりに `⌘N` ショートカットのみで機能するが、macOS ユーザーには不親切（メニューバーに出ていないショートカットは発見しにくい）。PR の判断で決定。
