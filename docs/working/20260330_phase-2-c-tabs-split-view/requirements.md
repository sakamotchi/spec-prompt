# 要件定義書 - Phase 2-C: タブ・分割表示

## 概要

コンテンツビューアとターミナルそれぞれに複数タブ・分割表示機能を追加する。複数のファイルやターミナルセッションを同時に参照できるようにし、仕様書を見ながら AI と対話する作業効率を向上させる。

## 背景・目的

現状のコンテンツビューアは1ファイルしか表示できない。ターミナルは複数タブ対応済み（Phase 1-C-7）だが分割表示は未実装。設計書を左ペインで参照しながら右ペインでコードを確認するといった並列作業ができないため、本フェーズで対応する。

## 要件一覧

### 機能要件

#### F-1: コンテンツ複数タブ

- **説明**: 複数のファイルをタブで同時に開ける
- **受け入れ条件**:
  - [ ] ファイルツリーからファイルをクリックするとタブが追加される
  - [ ] タブバーに開いているファイル名が表示される
  - [ ] タブをクリックするとそのファイルの内容が表示される
  - [ ] タブの × ボタンでタブを閉じられる
  - [ ] 最後の1つのタブは閉じられない（または閉じると空状態に戻る）
  - [ ] すでに開いているファイルをクリックするとそのタブがアクティブになる

#### F-2: コンテンツ分割表示

- **説明**: コンテンツエリアを左右に分割し、2つのファイルを同時表示できる
- **受け入れ条件**:
  - [ ] 分割ボタン（またはショートカット）で分割モードに切り替えられる
  - [ ] 左ペイン・右ペインそれぞれに独立したタブバーが表示される
  - [ ] 左右それぞれ別のファイルを表示できる
  - [ ] SplitPane のドラッグで左右の幅を調整できる
  - [ ] 再度ボタンを押すと分割を解除できる

#### F-3: ターミナルタブ（実装済み）

- **説明**: Phase 1-C-7 で実装済み。本フェーズでは確認のみ。
- **受け入れ条件**:
  - [x] 複数ターミナルをタブで開ける
  - [x] タブ追加・閉じる操作ができる

#### F-4: ターミナル分割表示

- **説明**: ターミナルエリアを左右に分割し、2つのターミナルセッションを同時表示できる
- **受け入れ条件**:
  - [ ] 分割ボタンで分割モードに切り替えられる
  - [ ] 左右それぞれ独立したターミナルタブが表示される
  - [ ] 左右それぞれ独立した PTY セッションで操作できる
  - [ ] SplitPane のドラッグで左右の幅を調整できる
  - [ ] 再度ボタンを押すと分割を解除できる
  - [ ] PTY セッションは分割解除後も破棄されない

### 非機能要件

- **パフォーマンス**: タブ切り替えは即時（レンダリング遅延なし）
- **ユーザビリティ**: タブバーはファイル名が長い場合に省略表示する
- **外観・デザイン**: 既存の CSS カスタムプロパティ（`--color-*`）を使用。アクティブタブは `--color-accent` でハイライト

## スコープ

### 対象

- コンテンツタブ（複数ファイルを同時に開く）
- コンテンツ分割表示（左右2ペイン）
- ターミナル分割表示（左右2ペイン）

### 対象外

- 3ペイン以上の分割
- 垂直（上下）分割
- タブのドラッグ並べ替え
- コンテンツの編集機能

## 既知の制約

- ターミナル分割では PTY セッションをアンマウントで破棄しないため、Phase 1-E と同様に `display: none` で常時マウントする方式を採用する
- コンテンツタブは `contentStore` を単一状態からタブ配列に変更するため、`ContentView.tsx` の全面的な書き換えが必要
