# テスト手順書 - Phase 2-C: タブ・分割表示

## 概要

コンテンツタブ・コンテンツ分割表示・ターミナル分割表示のテスト手順を記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト用プロジェクトとして `spec-prompt` リポジトリを開いておく

---

## 手動テスト

### ケース 1: コンテンツタブ — 複数ファイルを開く

**手順:**

1. ファイルツリーから任意の `.md` ファイルをクリックする
2. 別の `.md` ファイルをクリックする
3. さらに `.ts` ファイルをクリックする

**期待結果:**

- タブバーに3つのタブが表示される
- 各タブにファイル名が表示される
- 最後にクリックしたファイルがアクティブタブとしてハイライトされる

**確認結果:**

- [ ] OK / NG

---

### ケース 2: コンテンツタブ — タブの切り替え

**手順:**

1. 複数ファイルを開いた状態で、タブバーの別タブをクリックする

**期待結果:**

- クリックしたタブがアクティブになり、対応するファイルの内容が表示される

**確認結果:**

- [ ] OK / NG

---

### ケース 3: コンテンツタブ — タブを閉じる

**手順:**

1. 複数ファイルを開いた状態で、タブの × ボタンをクリックする

**期待結果:**

- タブが閉じる
- 閉じたタブがアクティブだった場合、隣のタブがアクティブになる

**確認結果:**

- [ ] OK / NG

---

### ケース 4: コンテンツタブ — 同じファイルを再クリック

**手順:**

1. ファイル A を開いた状態で、ファイルツリーからファイル B を開く
2. ファイルツリーからファイル A を再度クリックする

**期待結果:**

- タブが重複せず、ファイル A のタブがアクティブになる

**確認結果:**

- [ ] OK / NG

---

### ケース 5: コンテンツ分割表示 — 分割・解除

**手順:**

1. コンテンツタブバーの分割ボタン（`□□` アイコン）をクリックする
2. 左右2ペインに分割されることを確認する
3. 再度分割ボタンをクリックする

**期待結果:**

- 分割時: 左右2ペインが表示され、それぞれ独立したタブバーを持つ
- 解除時: 単一ペインに戻る

**確認結果:**

- [ ] OK / NG

---

### ケース 6: コンテンツ分割表示 — 左右で別ファイルを表示

**手順:**

1. 分割表示にする
2. 左ペインでファイル A を開く
3. 右ペインをクリックしてフォーカスを移し、ファイル B を開く

**期待結果:**

- 左ペインにファイル A、右ペインにファイル B が表示される

**確認結果:**

- [ ] OK / NG

---

### ケース 7: コンテンツ分割表示 — 幅のリサイズ

**手順:**

1. 分割表示にする
2. 左右の境界をドラッグしてリサイズする

**期待結果:**

- ドラッグで左右の幅が変わる

**確認結果:**

- [ ] OK / NG

---

### ケース 8: ターミナル分割表示 — 分割・解除

**手順:**

1. ターミナルタブに切り替える
2. ターミナルタブバーの分割ボタンをクリックする
3. 左右2ペインに分割されることを確認する
4. 再度ボタンをクリックして解除する

**期待結果:**

- 分割時: 左右2ペインにそれぞれターミナルが表示される
- 解除時: 単一ペインに戻る

**確認結果:**

- [ ] OK / NG

---

### ケース 9: ターミナル分割表示 — 左右独立操作

**手順:**

1. ターミナルを分割表示にする
2. 左ペインで `echo left` と入力して Enter
3. 右ペインで `echo right` と入力して Enter

**期待結果:**

- 左右それぞれ独立したシェルセッションで動作する

**確認結果:**

- [ ] OK / NG

---

### ケース 10: ターミナル分割 — 分割解除後も PTY セッションが維持される

**手順:**

1. ターミナルを分割表示にする
2. 右ペインのターミナルで `pwd` を実行して確認
3. 分割を解除する
4. 右ペインのターミナルだったタブをアクティブにして操作する

**期待結果:**

- 分割解除後もセッションが継続している（履歴が残っている）

**確認結果:**

- [ ] OK / NG
