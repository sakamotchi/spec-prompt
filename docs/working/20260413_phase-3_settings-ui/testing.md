# テスト手順書 - Phase 3: 設定 UI + 仕上げ

## 概要

通知設定 UI が正しく動作し、ON/OFF が PTY 起動時の hooks 注入に反映されることを確認する。

## 前提条件

- Phase 1, 2 が完了していること
- `npx tauri dev` でアプリが起動していること

## 手動テスト

### ケース 1: 設定画面にトグルが表示される

**手順:**

1. アプリを起動する
2. 設定画面を開く

**期待結果:**

- 「Claude Code 通知」のトグルスイッチが表示されている
- デフォルトは ON 状態

**確認結果:**

- [ ] OK / NG

---

### ケース 2: トグル ON → 通知が動作する

**手順:**

1. 設定画面でトグルが ON であることを確認
2. 新しいターミナルタブを開く
3. `echo $SPEC_PROMPT_NOTIFICATION` を実行する

**期待結果:**

- `1` が出力される

**確認結果:**

- [ ] OK / NG

---

### ケース 3: トグル OFF → 通知が無効になる

**手順:**

1. 設定画面でトグルを OFF にする
2. 新しいターミナルタブを開く
3. `echo $SPEC_PROMPT_NOTIFICATION` を実行する

**期待結果:**

- 空行が出力される（環境変数が設定されていない）

**確認結果:**

- [ ] OK / NG

---

### ケース 4: 設定が永続化される

**手順:**

1. 設定画面でトグルを OFF にする
2. アプリを終了する
3. `npx tauri dev` でアプリを再起動する
4. 設定画面を開く

**期待結果:**

- トグルが OFF のままである

**確認結果:**

- [ ] OK / NG

---

### ケース 5: 既存ターミナルには影響しない

**手順:**

1. トグル ON の状態でターミナルタブ A を開く
2. 設定画面でトグルを OFF にする
3. ターミナルタブ A で `echo $SPEC_PROMPT_NOTIFICATION` を実行する
4. 新しいターミナルタブ B を開く
5. ターミナルタブ B で `echo $SPEC_PROMPT_NOTIFICATION` を実行する

**期待結果:**

- タブ A: `1`（起動時の設定が維持される）
- タブ B: 空（新しい設定が反映される）

**確認結果:**

- [ ] OK / NG

---

### ケース 6: E2E 通知テスト（ON → OFF → ON）

**手順:**

1. トグル ON、新ターミナルで `claude` 起動、承認待ちで通知が出ることを確認
2. トグル OFF、新ターミナルで `claude` 起動、承認待ちで通知が**出ない**ことを確認
3. トグル ON に戻し、新ターミナルで `claude` 起動、通知が出ることを確認

**期待結果:**

- ON/OFF に応じて通知の有無が正しく切り替わる

**確認結果:**

- [ ] OK / NG
