# テスト手順書 - Phase 3-D: 拡張子別ファイルアイコン

## 概要

拡張子別ファイルアイコンの手動テスト手順を記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- 各種拡張子のファイル（`.ts`, `.md`, `.rs` など）を含むプロジェクトが開かれていること

---

## 手動テスト

### ケース 1: TypeScript ファイルのアイコン

**手順:**

1. ツリーパネルで `.ts` ファイルを探す

**期待結果:**

- TypeScript のアイコン（青い `TS` マーク）が表示される

**確認結果:**

- [x] OK

---

### ケース 2: Markdown ファイルのアイコン

**手順:**

1. ツリーパネルで `.md` ファイルを探す

**期待結果:**

- Markdown のアイコンが表示される

**確認結果:**

- [x] OK

---

### ケース 3: Rust ファイルのアイコン

**手順:**

1. ツリーパネルで `.rs` ファイルを探す

**期待結果:**

- Rust のアイコン（歯車マーク）が表示される

**確認結果:**

- [x] OK

---

### ケース 4: JSON ファイルのアイコン

**手順:**

1. ツリーパネルで `.json` ファイルを探す

**期待結果:**

- JSON のアイコンが表示される

**確認結果:**

- [x] OK

---

### ケース 5: 対応外拡張子のアイコン

**手順:**

1. ツリーパネルで対応外の拡張子（例: `.xyz`, `.log`）のファイルを探す

**期待結果:**

- デフォルトファイルアイコンが表示される（クラッシュや空白にならない）

**確認結果:**

- [x] OK

---

### ケース 6: フォルダアイコン（閉じた状態）

**手順:**

1. ツリーパネルで折りたたまれたフォルダを確認する

**期待結果:**

- 閉じたフォルダアイコン（`default-folder`）が表示される

**確認結果:**

- [x] OK

---

### ケース 7: フォルダアイコン（展開状態）

**手順:**

1. ツリーパネルでフォルダをクリックして展開する

**期待結果:**

- 開いたフォルダアイコン（`default-folder-opened`）に切り替わる

**確認結果:**

- [x] OK

---

### ケース 8: フォルダ読み込み中のローダー

**手順:**

1. 未読み込みのフォルダをクリックする

**期待結果:**

- 読み込み中はスピナー（`Loader2`）が表示され、読み込み完了後にフォルダアイコンに切り替わる

**確認結果:**

- [x] OK

---

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| 拡張子なしファイル（例: `Makefile`） | デフォルトアイコンを表示 | OK |
| `.tsx` ファイル | React TSX アイコン（`file-type-reactts`）を表示 | OK |
| `.jsx` ファイル | React JSX アイコン（`file-type-reactjs`）を表示 | OK |

## 回帰テスト

- [x] フォルダの展開・折りたたみが動作する

- [x] ターミナルが正常に起動・入力できる
