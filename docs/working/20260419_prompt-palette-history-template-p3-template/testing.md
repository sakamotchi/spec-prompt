# テスト手順書 - prompt-palette-history-template-p3-template

## 概要

Phase 3（テンプレート機能）のテスト手順。UI を直接操作する手動テストを主軸とし、ロジック単体は自動テストでカバー。

## 前提条件

- `npm install` 済み
- `npx tauri dev` でアプリが起動できる状態
- Phase 2 完了時点の挙動が全てグリーン（履歴送信・巡回・`⌘H`・Esc 段階剥離）

## 手動テスト

### ケース 1: 新規テンプレート作成（T3-1 + T3-3）

**手順:**

1. パレット起動（`⌘⇧P`）
2. `⌘T`（パレット内）またはヘッダのテンプレアイコンでドロップダウンを開く
3. 「+ 新規作成」をクリック
4. エディタが開く。name に `要約`、body に `以下を要約してください: {{path}}` を入力
5. `⌘Enter` または「保存」ボタンで保存

**期待結果:**

- エディタが閉じ、テンプレドロップダウンに「要約」行が追加される
- DevTools の localStorage に `spec-prompt:prompt-palette` が更新され、`templates` 配列に新テンプレが含まれる

**確認結果:**

- [x] OK

---

### ケース 2: テンプレ選択 → プレースホルダ選択状態化（T3-1 + T3-2）

**手順:**

1. ケース 1 で作った「要約」テンプレがある状態
2. パレット起動 → `⌘T` でドロップダウンを開く
3. 「要約」行を `↓`/`↑` で選択 → `Enter`

**期待結果:**

- ドロップダウンが閉じ、textarea に `以下を要約してください: {{path}}` が流し込まれる
- `{{path}}` が選択状態（ハイライト）になっている
- textarea にフォーカスが戻る
- `⌘Enter` で送信可能

**確認結果:**

- [x] OK

---

### ケース 2-A: `{{path}}` 選択状態でツリーからパス挿入（T3-2 派生）

**手順:**

1. body が `以下を読む: {{path}}` のテンプレを作成
2. テンプレドロップダウンから流し込む → `{{path}}` が選択状態
3. ファイルツリーで任意のファイルを `⌘+Click`（macOS）/ `Ctrl+Click`（win/linux）

**期待結果:**

- 選択されていた `{{path}}` が該当ファイルの（相対）パス + 末尾スペースで置換される（例: `以下を読む: src/foo.ts `）
- textarea 本文のうち他の部分は変更されない

**確認結果:**

- [x] OK

### ケース 2-B: `{{path}}` 選択状態で右クリック「パスをターミナルに挿入」（T3-2 派生）

**期待結果:**

- `{{path}}` がパスで置換される（ケース 2-A と同じ）

**確認結果:**

- [x] OK

### ケース 2-C: `{{path}}` 選択状態で `⌘P` パスパレットから挿入（T3-2 派生）

**手順:**

1. ケース 2-A 同様に `{{path}}` 選択状態
2. `⌘P`（または `Ctrl+P`）でパスパレットを開く
3. ファイル名を入力 → `Enter`

**期待結果:**

- パスパレットが閉じ、選択されていた `{{path}}` がパスで置換される
- 選択が失われて末尾追記になる場合は**保険対応（ストアに `lastSelection` を保持）が必要** → 実装時に挙動を確認し、必要なら個別タスクを追加する

**確認結果:**

- [x] OK（挙動と所要工数の両方を観察）

### ケース 3: Tab でプレースホルダを次々に埋める（T3-4）

**手順:**

1. エディタで body が `名前: {{name}}, 年齢: {{age}}, 住所: {{addr}}` のテンプレを作る
2. ドロップダウンから流し込み
3. 最初 `{{name}}` が選択状態 → 任意の文字（例: `Alice`）を入力
4. `Tab` を押す
5. `{{age}}` が選択状態になることを確認 → `30` を入力
6. `Tab` を押す
7. `{{addr}}` が選択状態になる

**期待結果:**

- 3 つのプレースホルダが順に選択状態になり、入力で上書きできる
- 最後の `{{addr}}` を入力後に `Tab` を押すと、プレースホルダが無いので通常の Tab 動作（フォーカス移動）に戻る

**確認結果:**

- [x] OK
