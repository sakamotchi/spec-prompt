# テスト手順書 - alacritty-terminal Phase 2（WebView セルレンダラー実装）

## 概要

Phase 2 は Canvas ベースのセルレンダラー（`TerminalRenderer`）の実装確認が中心。
Feature Flag ON 時の新レンダラー動作と、Feature Flag OFF 時の既存 xterm.js の回帰確認を行う。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- DevTools が使用できること（`Cmd+Option+I`）
- Feature Flag ON の起動コマンド: `VITE_USE_CANVAS_RENDERER=true npx tauri dev`

---

## 手動テスト（Feature Flag ON）

### ケース 1: Canvas レンダラーで基本的なテキストが表示される

**手順:**

1. `VITE_USE_CANVAS_RENDERER=true npx tauri dev` でアプリを起動する
2. ターミナルタブを開く
3. `echo "hello world"` を入力して Enter する
4. `ls -la` を入力して Enter する

**期待結果:**

- `hello world` が Canvas 上に文字化けなく表示される
- `ls -la` のファイル一覧が表示される（色付きであることが望ましい）
- 文字が均等なセルグリッドとして整列している

**確認結果:**

- [x] OK / NG

---

### ケース 2: CJK 文字・全角文字が正しく表示される

**手順:**

1. `echo "こんにちは"` を実行する
2. `echo "日本語テスト 🎉"` を実行する
3. `claude` を起動し、Claude Code のウェルカムメッセージを確認する（ボックス描画文字を含む）

**期待結果:**

- 日本語文字が cellWidth × 2 の幅で表示される（文字が重なったり欠けたりしない）
- 絵文字が表示される（または空白として表示される）
- Claude Code のボックス描画文字（`─`, `│`, `╭`, `╮` 等）が文字化けなく表示される

**確認結果:**

- [x] OK / NG

---

### ケース 3: ANSI カラーが正しく表示される

**手順:**

1. `ls --color=always` を実行する
2. `echo -e "\033[31mRed\033[0m \033[32mGreen\033[0m \033[34mBlue\033[0m"` を実行する
3. `claude` を起動し、カラー表示を確認する

**期待結果:**

- ディレクトリは青、実行ファイルは緑など、ANSI 16色が正しく表示される
- Red/Green/Blue のテキストがそれぞれ正しい色で表示される
- Claude Code のカラー出力（ハイライト等）が表示される

**確認結果:**

- [x] OK / NG

---

### ケース 4: カーソルが表示・点滅する

**手順:**

1. ターミナルを開く
2. コマンドを打たずにカーソルを観察する
3. `echo "` と入力して（閉じずに）カーソル位置を確認する

**期待結果:**

- カーソルが約 500ms ごとに点滅する
- 文字入力後にカーソルが正しい位置に移動する

**確認結果:**

- [x] OK / NG

---

### ケース 5: キーボード入力が正常に動作する

**手順:**

1. `echo "test"` の途中で Backspace を押して文字を削除する
2. 矢印キー（↑）で履歴を遡る
3. `echo -n ""; ` と入力して Ctrl+C で中断する
4. `cat` を実行して Ctrl+D で EOF を送る
5. `man ls` を実行して矢印キーでスクロールし、`q` で終了する

**期待結果:**

- Backspace で直前の文字が削除される
- ↑キーでコマンド履歴が遡れる
- Ctrl+C でコマンドが中断される
- Ctrl+D で EOF が送信され `cat` が終了する
- `man ls` でページャーが起動し、矢印キーでスクロールできる

**確認結果:**

- [x] OK / NG

---

### ケース 6: ウィンドウリサイズ後に正しく再描画される

**手順:**

1. ターミナルを起動し `tput cols` でカラム数を確認する
2. ウィンドウサイズを変更する
3. 再度 `tput cols` を実行する
4. `ls -la` を実行して表示が崩れていないことを確認する

**期待結果:**

- リサイズ後に `tput cols` の値が変化する
- Canvas がコンテナサイズに合わせて再描画される
- 文字・セルが正しい位置に整列している

**確認結果:**

- [x] OK / NG

---

### ケース 7: テーマ切り替えで Canvas が再描画される

**手順:**

1. 設定を開き、テーマを「ライト」に変更する
2. ターミナルの背景色・前景色が変化することを確認する
3. テーマを「ダーク」に戻す

**期待結果:**

- ライトテーマ時は背景が白系・前景が黒系になる
- ダークテーマ時は背景が黒系・前景が白系になる
- テーマ変更後もテキストが正しく表示される

**確認結果:**

- [x] OK / NG
