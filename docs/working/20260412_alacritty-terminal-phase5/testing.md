# テスト手順書 - alacritty-terminal Phase 5（テスト・品質）

## 概要

Phase 5 は品質確認が主体のフェーズ。手動テストでターミナルの表示品質・リサイズ安定性・
フォント切り替え・パフォーマンスを検証し、Rust テストを追加する。

## 前提条件

- `npx tauri dev` でアプリが起動していること

---

## 手動テスト - 表示品質（5-A）

### ケース 1: CJK 文字・ボックス描画文字の表示

**手順:**

1. ターミナルを開く
2. `claude` を起動する（または任意の CJK 出力コマンドを実行）
3. 日本語テキスト・ボックス描画文字（`─`, `│`, `┌`, `└` 等）が表示されることを確認する

**期待結果:**

- CJK 文字が全角幅で正しく描画される
- ボックス描画文字のセルが隣接セルと揃っている
- 文字化けが発生していない

**確認結果:**

- [x] OK

---

### ケース 2: ANSI カラー出力

**手順:**

1. ターミナルで以下を実行する:
   ```
   echo -e "\e[31mRed \e[32mGreen \e[33mYellow \e[34mBlue \e[35mMagenta \e[36mCyan\e[0m"
   ```
2. `ls --color=auto` でカラー出力を確認する

**期待結果:**

- 各 ANSI カラーが正しい色で描画される

**確認結果:**

- [x] OK

---

### ケース 3: テキスト属性

**手順:**

1. ターミナルで以下を実行する:
   ```
   echo -e "\e[1mBold\e[0m \e[3mItalic\e[0m \e[4mUnderline\e[0m \e[9mStrikeout\e[0m \e[7mInverse\e[0m \e[2mDim\e[0m"
   ```

**期待結果:**

- Bold: 太字で描画
- Italic: 斜体で描画
- Underline: 下線が表示
- Strikeout: 打ち消し線が表示
- Inverse: 前景色と背景色が反転
- Dim: 半透明（薄い色）で描画

**確認結果:**

- [x] OK

---

## 手動テスト - リサイズ安定性（5-B）

### ケース 4: ウィンドウリサイズ

**手順:**

1. ターミナルで `echo "resize test"` を実行
2. ウィンドウの端をドラッグして連続的にリサイズする（大→小→大）
3. プロンプトが表示されていることを確認

**期待結果:**

- 描画が崩れない
- テキストが途切れたり重なったりしない
- プロンプトが正しい位置に表示される

**確認結果:**

- [x] OK

---

### ケース 5: コンテンツ↔ターミナル切り替え

**手順:**

1. ターミナルで `echo "tab switch"` を実行
2. コンテンツタブに切り替える（Ctrl+Tab）
3. ターミナルタブに戻る（Ctrl+Tab）
4. 表示が正しいことを確認

**期待結果:**

- ターミナルに戻ったときサイズが正しく反映されている
- 以前の出力が表示されている

**確認結果:**

- [x] OK

---

### ケース 6: 分割レイアウト

**手順:**

1. ターミナルで `echo "split test"` を実行
2. 分割レイアウトボタンを押して ON にする
3. 両方のペインでターミナルが正しく表示されることを確認
4. 分割レイアウトを OFF にする

**期待結果:**

- 分割時に各ペインのサイズが正しく計算される
- ON/OFF 切り替え後もターミナルが正しく再描画される

**確認結果:**

- [x] OK

---

## 手動テスト - フォント切り替え（5-C）

### ケース 7: フォントファミリー変更

**手順:**

1. ターミナルで `echo "font test"` を実行
2. 設定画面を開き、ターミナルフォントファミリーを別のフォントに変更する
3. ターミナルの表示を確認

**期待結果:**

- フォントが即座に切り替わる
- 文字が正しい幅・高さで描画される

**確認結果:**

- [x] OK

---

### ケース 8: フォントサイズ変更

**手順:**

1. ターミナルの cols/rows を確認（プロンプトの折り返し位置等）
2. 設定画面でフォントサイズを大きく変更する（例: 14 → 20）
3. ターミナルの cols/rows が減少していることを確認
4. フォントサイズを元に戻す

**期待結果:**

- フォントサイズ変更後に cols/rows が再計算される
- PTY にリサイズが送信される（コマンド実行で確認可能）

**確認結果:**

- [x] OK

## 手動テスト - パフォーマンス（5-D）

### ケース 9: 大量出力中の入力応答

**手順:**

1. `yes` コマンドを実行（無限出力）
2. 2〜3 秒後に Ctrl+C を押す

**期待結果:**

- Ctrl+C が速やかに反映されて出力が停止する（体感 100ms 以内）
- アプリが固まらない

**確認結果:**

- [x] OK
