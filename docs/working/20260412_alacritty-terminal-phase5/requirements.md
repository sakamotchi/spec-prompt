# 要件定義書 - alacritty-terminal Phase 5（テスト・品質）

## 概要

Phase 1〜3 で実装した alacritty-terminal ベースの Canvas レンダラーに対して、
品質確認・パフォーマンス検証・テスト追加を行い、リリース品質に到達させる。

## 背景・目的

Phase 3 完了時点で xterm.js は完全に削除され、Canvas レンダラーがデフォルトで動作している。
基本的な手動テスト（Phase 1〜3 の testing.md）は完了しているが、以下の項目が未検証:

- リサイズの安定性（ウィンドウリサイズ・分割レイアウト切り替え）
- フォント切り替え（設定画面からの動的変更）
- 大量出力時のパフォーマンス
- Rust 側の grid 変換ロジックのユニットテスト

---

## 機能要件

### 5-A: 表示品質確認

| ID | 要件 |
|----|------|
| 5-A-1 | Claude Code を起動し、CJK 文字・ボックス描画文字・絵文字が正しく表示される |
| 5-A-2 | ANSI カラー（16色・256色・truecolor）が正しく描画される |
| 5-A-3 | bold / italic / underline / strikeout / inverse / dim が正しく表現される |

### 5-B: リサイズ安定性

| ID | 要件 |
|----|------|
| 5-B-1 | ウィンドウサイズを連続変更しても表示が崩れない |
| 5-B-2 | コンテンツ↔ターミナルの切り替え後にサイズが正しく反映される |
| 5-B-3 | 分割レイアウト ON/OFF でターミナルサイズが正しく再計算される |

### 5-C: フォント切り替え

| ID | 要件 |
|----|------|
| 5-C-1 | 設定画面でフォントファミリーを変更すると Canvas が即座に再描画される |
| 5-C-2 | フォントサイズを変更すると cols/rows が再計算され PTY にリサイズが送信される |

### 5-D: パフォーマンス

| ID | 要件 |
|----|------|
| 5-D-1 | `yes` コマンド（無限出力）実行中にキー入力が 100ms 以内に反映される |
| 5-D-2 | 大量出力後のスクロール操作が滑らか（16ms/フレーム以内） |

### 5-E: Rust テスト追加

| ID | 要件 |
|----|------|
| 5-E-1 | `grid.rs` の `color_to_data` 変換ロジックにユニットテストが追加されている |
| 5-E-2 | `grid.rs` の `flags_to_cell_flags` 変換ロジックにユニットテストが追加されている |
| 5-E-3 | `TerminalInstance` の `new` / `advance` / `collect_damage` / `scroll` / `resize` にテストが追加されている |

---

## 非機能要件

| 項目 | 要件 |
|------|------|
| テストカバレッジ | `grid.rs` の変換関数は全分岐をカバー |
| パフォーマンス | 大量出力時の入力遅延 < 100ms |

---

## 対象外

- 新機能の追加
- UI デザインの変更
- Windows / Linux 向けビルドの確認
