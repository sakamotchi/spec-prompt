# テスト手順書 - prompt-palette-f2

F2 で担保する **受け入れ基準 6, 7, 8, 12**（元要件 §7）の手動確認と、自動テストの実行方法をまとめる。

## 概要

可能な限り `npx tauri dev` 上での手動操作で確認する。ユニットテストは vitest。F1 で担保済みの項目（1〜5）と F3 以降の項目（9〜11, 13〜15）はここでは確認しない（回帰の軽い確認のみ実施）。

## 前提条件

- `feature/prompt-edit-palette` ブランチをチェックアウト済み（F1 のコミット `d6076ed` を含む）
- `npm install` 済み
- `npx tauri dev` でアプリが起動している
- テスト用プロジェクトフォルダを開き、複数のファイルがツリーに並んでいる
- ターミナルタブを最低 1 つ起動してある
- macOS を主対象（Win/Linux はショートカットのみ別途確認）

## 手動テスト

### ケース 1: Cmd+Click で textarea に挿入される（受け入れ基準 6）

**手順:**

1. `Cmd+Shift+P` でプロンプトパレットを開く。
2. textarea に「読んで: 」と入力する。
3. そのままファイルツリーの `README.md` を `Cmd+Click` する。

**期待結果:**

- textarea の本文が `読んで: README.md ` になる（キャレットは末尾）
- パレットは開いたまま、ターミナルには何も書き込まれない
- フォーカスは textarea に残り、続けてタイプできる

**確認結果:**

- [x] OK（2026-04-18 確認）

**期待結果:**

- textarea が `前foo.md 後` になり、キャレットは `foo.md ` の直後

### ケース 5: 複数選択 →「選択中 N 件をすべて挿入」（受け入れ基準 7 の補強）

**手順:**

1. ツリーで 1 つ目のファイルを `Shift+Click`（または `Cmd+Shift+Click`）で選択する。選択状態のハイライトが付くことを確認。
2. 同じ要領で 2 つ目・3 つ目のファイルを選択する。選択中のファイルで `Cmd+Click` すると追加トグル可能（1 件以上選択中に限る）。
3. パレットを `Cmd+Shift+P` で開く。
4. 選択中のいずれかのファイルを右クリック →「選択中 3 件をすべて挿入」を選ぶ。

**期待結果:**

- 3 ファイルのパスがスペース区切りで textarea に挿入される（末尾にスペース 1 つ）
- パレットは閉じずに textarea にフォーカスが残る

**備考:**

- トリガーキーは `Shift+Click`（または `Cmd+Shift+Click` / `Ctrl+Shift+Click`）。プレーンな `Cmd+Click` は単発挿入であって選択ではない。
- 実装参照: `src/components/TreePanel/TreeNode.tsx:109-124`（`toggleFileSelection` 呼び出し条件）

**確認結果:**

- [x] OK（2026-04-18 確認）

### ケース 6: Ctrl+P から textarea に挿入 + フォーカス戻し（受け入れ基準 8）

- textarea が `参照: README.md ` になる
- フォーカスは textarea に戻り、続けてタイプできる（ターミナルにフォーカスは移らない）

**確認結果:**

- [x] OK（2026-04-18 確認）

- いずれの経路でもアクティブターミナルに直接パスが書き込まれる（従来挙動の維持）

**確認結果:**

- [x] OK（2026-04-18 確認）
