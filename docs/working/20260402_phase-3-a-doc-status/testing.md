# テスト手順書 - Phase 3-A: ドキュメントステータス管理

## 概要

フロントマター解析・ツリーへのバッジ表示・コンテキストメニューからのステータス変更の手動テスト手順を記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- `spec-prompt` リポジトリを開いた状態にしておく
- テスト用 MD ファイルを用意しておく（手順内で作成）

---

## 手動テスト

### ケース 1: ステータスなし MD ファイルのバッジ非表示

**手順:**

1. フロントマターのない MD ファイル（例: `README.md`）を含むフォルダを展開する

**期待結果:**

- `README.md` の行にステータスバッジが表示されない

**確認結果:**

- [x] OK / NG

---

### ケース 2: `draft` ステータスのバッジ表示

**手順:**

1. テスト用ファイル `test_draft.md` を作成し、以下の内容を書く（外部エディタまたはターミナルで）：
   ```
   ---
   status: draft
   ---

   # テスト
   ```
2. ツリーでそのフォルダを折りたたみ → 再展開する

**期待結果:**

- `test_draft.md` の行にグレーの丸点（●）が表示される

**確認結果:**

- [x] OK / NG

---

### ケース 8: 現在のステータスにチェックマークが付く

**手順:**

### ケース 9: ステータスを変更するとファイルが更新される

**手順:**

1. `test_draft.md` を右クリック → 「ステータス」→「レビュー中」を選択する
2. ターミナルで `cat test_draft.md` を実行してファイル内容を確認する

**期待結果:**

- ファイルの `status: draft` が `status: reviewing` に変更されている

**確認結果:**

- [x] OK / NG

### ケース 10: ステータス変更後、ツリーのバッジが即座に更新される

**手順:**

1. `test_draft.md`（グレーバッジ）を右クリック → 「ステータス」→「承認済」を選択する

**期待結果:**

- ツリーのバッジがグレー → グリーンに即座に切り替わる
- ファイルの再展開なしに更新される

**確認結果:**

- [x] OK / NG

---

### ケース 11: フロントマターのないファイルにステータスを設定する

**手順:**

1. フロントマターのない `test_no_frontmatter.md` を作成する（内容: `# テスト` のみ）
2. そのファイルを右クリック → 「ステータス」→「草稿」を選択する
3. ターミナルで `cat test_no_frontmatter.md` を確認する

**期待結果:**

- ファイルの先頭に `---\nstatus: draft\n---\n` が追加される
- 元のコンテンツ（`# テスト`）は保持される

**確認結果:**

- [x] OK / NG

---

### ケース 12: フロントマターに他フィールドがある場合の更新

**手順:**

1. 以下の内容の `test_multi.md` を作成する：
   ```
   ---
   title: テストドキュメント
   status: draft
   author: test
   ---

   # 本文
   ```
2. 右クリック → 「ステータス」→「承認済」を選択する
3. ファイル内容を確認する

**期待結果:**

- `status: draft` → `status: approved` に変更される
- `title` / `author` フィールドは保持される
- プレビューにフロントマターが表示されない（`remark-frontmatter` により除去）

**確認結果:**

- [x] OK / NG
