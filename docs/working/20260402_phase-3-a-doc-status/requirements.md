# 要件定義書 - Phase 3-A: ドキュメントステータス管理

## 概要

マークダウンファイルのYAMLフロントマターにステータスを持たせ、ツリーパネルへの表示とコンテキストメニューからの変更を可能にする。

## 背景・目的

仕様書駆動開発では「この仕様書は今どの状態か」が重要な情報になる。SpecPrompt はファイルツリーでプロジェクト全体を俯瞰できる設計になっており、各ドキュメントのステータスをツリー上で一目で把握できれば、AI CLI との作業サイクル（Draft → Review → Approve）を効率化できる。

## 要件一覧

### 機能要件

#### F-1: フロントマター解析

- **説明**: MDファイルの冒頭にあるYAMLフロントマターから `status` フィールドを読み取る。
- **受け入れ条件**:
  - [ ] `---\nstatus: draft\n---` 形式のフロントマターを正しく解析できる
  - [ ] フロントマターがない場合はステータスなし（未設定）として扱う
  - [ ] `status` フィールドがない場合も未設定として扱う
  - [ ] `.md` / `.mdx` ファイルのみ対象（その他の拡張子は無視）

#### F-2: ツリーへのステータス表示

- **説明**: ファイルツリーのMDファイル行に、ステータスを示す小さな色付きバッジを表示する。
- **ステータスと表示**:
  - `draft`（草稿）: グレーのバッジ
  - `reviewing`（レビュー中）: イエローのバッジ
  - `approved`（承認済）: グリーンのバッジ
  - 未設定: 表示なし
- **受け入れ条件**:
  - [ ] ディレクトリを展開したとき、MD/MDXファイルのステータスを非同期で読み込みバッジを表示する
  - [ ] ステータス読み込み中はバッジを表示しない（ローディング表示なし）
  - [ ] ステータス変更後、ツリーのバッジが即座に更新される（キャッシュ反映）

#### F-3: ステータス変更UI

- **説明**: 右クリックコンテキストメニューからステータスを変更し、ファイルに書き戻す。
- **受け入れ条件**:
  - [ ] MDファイルの右クリックメニューに「ステータス」サブメニューが表示される（非MD/MDXファイルには表示しない）
  - [ ] サブメニューに「草稿」「レビュー中」「承認済」の3項目が表示される
  - [ ] 現在のステータスにチェックマークが付く
  - [ ] 選択するとフロントマターを書き換えてファイルに保存する
  - [ ] フロントマターがないファイルに設定する場合は、ファイル先頭にフロントマターを追加する

### 非機能要件

- **パフォーマンス**: ステータス読み込みは非同期・遅延評価。ディレクトリ展開時に MD ファイル分だけ `read_file` を追加呼び出しするが、ファイル数が多い場合でもUIブロックを起こさない。
- **ユーザビリティ**: 右クリック → サブメニュー → 選択の3操作でステータス変更完了。
- **外観・デザイン**: バッジは 6px 程度の小さな丸点（`●`）。CSS カスタムプロパティに依存せず固定色（草稿: `#6b7280`, レビュー中: `#eab308`, 承認済: `#22c55e`）を使用する。

## スコープ

### 対象

- TypeScript でのフロントマター解析・書き換えユーティリティ（新規）
- `write_file` Rust コマンド（ファイル書き戻し用、新規）
- `appStore` へのステータスキャッシュ追加
- `TreeNode` へのバッジ表示
- `ContextMenu` へのステータスサブメニュー追加

### 対象外

- マークダウンプレビュー内でのステータス表示（ヘッダー等）
- ステータス変更時のGitコミット自動化
- フロントマターの `status` 以外のフィールド操作

## 実装対象ファイル（予定）

- `src-tauri/src/commands/filesystem.rs`（変更: `write_file` 追加）
- `src-tauri/src/lib.rs`（変更）
- `src/lib/tauriApi.ts`（変更）
- `src/lib/frontmatter.ts`（新規: 解析・書き換えユーティリティ）
- `src/stores/appStore.ts`（変更: `docStatuses` キャッシュ追加）
- `src/components/TreePanel/TreeNode.tsx`（変更: バッジ表示）
- `src/components/TreePanel/ContextMenu.tsx`（変更: ステータスサブメニュー）

## 依存関係

- Phase 1-B: ファイルツリー（`TreeNode`, `updateDirChildren`）
- Phase 2-E: ファイル操作（`read_file` は実装済み、`write_file` を追加）

## 既知の制約

- 外部ライブラリは追加しない。フロントマター解析は正規表現ベースの軽量実装とする（仕様書のフロントマターは `key: value` 形式のシンプルな構造のみを想定）。
