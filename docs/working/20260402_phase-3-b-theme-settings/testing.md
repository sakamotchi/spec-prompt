# テスト手順書 - Phase 3-B: テーマ・外観設定

## 概要

テーマ切り替え・フォント設定・設定の永続化の手動テスト手順を記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- `spec-prompt` リポジトリを開いた状態

---

## 手動テスト

### ケース 1: 設定モーダルを開く

**手順:**

1. TreePanel ヘッダー右端のギアアイコン（⚙）をクリックする

**期待結果:**

- 設定モーダルが開く
- 「テーマ」「コンテンツ」「ターミナル」の3セクションが表示される

**確認結果:**

- [x] OK

---

### ケース 2: 設定モーダルを閉じる

**手順:**

1. 設定モーダルを開く
2. モーダル外をクリックする（または `Esc` を押す）

**期待結果:**

- モーダルが閉じる

**確認結果:**

- [x] OK

---

### ケース 3: ライトテーマへ切り替え

**手順:**

1. 設定モーダルを開く
2. テーマセクションで「ライト」を選択する

**期待結果:**

- アプリ全体が白背景のライトテーマに切り替わる
- ファイルツリー・コンテンツ・ヘッダーすべてのカラーが変わる
- モーダルを閉じても設定が維持される

**確認結果:**

- [x] OK

---

### ケース 4: ダークテーマへ戻す

**手順:**

1. ライトテーマの状態で設定モーダルを開く
2. 「ダーク」を選択する

**期待結果:**

- ダークテーマに戻る

**確認結果:**

- [x] OK

---

### ケース 5: システムテーマ

**手順:**

1. 設定モーダルで「システム」を選択する
2. macOS のシステム設定でダーク/ライトを切り替える

**期待結果:**

- OS のテーマに追従してアプリのテーマが変わる

**確認結果:**

- [x] OK

---

### ケース 6: ライトテーマでマークダウンプレビューが読める

**手順:**

1. ライトテーマに切り替える
2. `README.md` などの MD ファイルをクリックして開く

**期待結果:**

- ライトテーマで MD プレビューが正常に表示される（文字が見える）

**確認結果:**

- [x] OK

---

### ケース 7: テーマ切り替え時に Shiki テーマが連動する

**手順:**

1. コードを含む MD ファイルを開く（例: コードブロックがある仕様書）
2. ライトテーマに切り替える

**期待結果:**

- コードブロックのシンタックスハイライトがライト用（`github-light`）に切り替わる
- ダークテーマに戻すと `github-dark` に戻る

**確認結果:**

- [x] OK

---

### ケース 8: コンテンツフォントサイズを変更する

**手順:**

1. 設定モーダルを開く
2. コンテンツセクションのスライダーを右に動かし、サイズを 20px にする

**期待結果:**

- スライダー横のラベルが「20px」と表示される
- MD プレビューやコードビューアのフォントサイズが大きくなる

**確認結果:**

- [x] OK

---

### ケース 9: コンテンツフォントファミリーを変更する

**手順:**

1. 設定モーダルのコンテンツセクションでフォントファミリーを「system-ui」に変更する

**期待結果:**

- マークダウンプレビューのフォントが変わる

**確認結果:**

- [x] OK

---

### ケース 10: ターミナルフォントサイズを変更する

**手順:**

1. ターミナルタブを開いておく
2. 設定モーダルのターミナルセクションでフォントサイズを 18px に変更する

**期待結果:**

- ターミナルのフォントサイズが大きくなる
- ターミナルのレイアウトが自動調整される（`fitAddon.fit()`）

**確認結果:**

- [x] OK

---

### ケース 11: ライトテーマでターミナルの配色が変わる

**手順:**

1. ターミナルタブを開いておく
2. ライトテーマに切り替える

**期待結果:**

- ターミナルが白背景・黒文字に切り替わる

**確認結果:**

- [x] OK

---

### ケース 12: 設定がアプリ再起動後も保持される

**手順:**

1. ライトテーマ・コンテンツフォントサイズ 18px に設定する
2. アプリを終了する
3. アプリを再起動する

**期待結果:**

- ライトテーマで起動する
- コンテンツフォントサイズが 18px になっている
- 設定モーダルを開くと「ライト」が選択されている

**確認結果:**

- [x] OK

---

### ケース 13: config.json に設定が保存されている

**手順:**

1. 設定を変更した後、ターミナルで以下を実行する：
   ```bash
   cat ~/Library/Application\ Support/spec-prompt/config.json
   ```

**期待結果:**

- `"appearance"` フィールドに変更した設定が保存されている

**確認結果:**

- [x] OK
