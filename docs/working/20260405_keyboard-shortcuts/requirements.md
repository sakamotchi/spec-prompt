# 要件定義書 - Phase 3-C: キーボードショートカット整備

## 概要

アプリ全体のキーボードショートカットを整備し、マウスを使わずに主要操作が完結できる体験を提供する。
また `?` キーでショートカット一覧を表示するヘルプ機能を追加する。

## 背景・目的

現状は `Ctrl+Tab`（コンテンツ/ターミナル切り替え）と `Ctrl+P`（パスパレット）のみが実装されており、
タブ操作・分割・フォーカス移動など日常的な操作にショートカットが存在しない。
生成AI CLIとの併用を想定したユースケースでは、キーボード中心の操作が重要であるため、
VS Code 互換に近いショートカット体系を整備することで生産性を向上させる。

## 要件一覧

### 機能要件

#### F-1: タブ操作ショートカット

- **説明**: コンテンツタブ・ターミナルタブの新規作成・閉じる操作をキーボードで行える
- **受け入れ条件**:
  - [ ] `Cmd+T` でコンテンツタブを新規作成できる
  - [ ] `Cmd+W` でアクティブなタブを閉じることができる
  - [ ] `Cmd+Shift+T` でターミナルタブを新規作成できる
  - [ ] タブが1枚のみの場合は `Cmd+W` を無効にする（タブが消えてしまうのを防ぐ）

#### F-2: タブ切り替えショートカット

- **説明**: 数字キーまたは左右キーで隣接タブへ移動できる
- **受け入れ条件**:
  - [ ] `Cmd+1` ～ `Cmd+9` でコンテンツ/ターミナルの n番目のタブをアクティブにできる
  - [ ] `Ctrl+Tab` で次のタブ、`Ctrl+Shift+Tab` で前のタブへ移動できる（既存の「コンテンツ/ターミナルペイン切り替え」と競合しないよう調整）

#### F-3: 分割表示ショートカット

- **説明**: コンテンツペインの分割/統合をキーボードで切り替えられる
- **受け入れ条件**:
  - [ ] `Cmd+\` でコンテンツペインの分割/統合を切り替えられる
  - [ ] `Cmd+Shift+\` でターミナルペインの分割/統合を切り替えられる

#### F-4: フォーカス移動ショートカット

- **説明**: ツリーペイン・コンテンツペイン・ターミナルペイン間をキーボードで移動できる
- **受け入れ条件**:
  - [ ] `Cmd+0` でツリーパネルにフォーカスを移動できる
  - [ ] `Cmd+1`（コンテンツタブが存在する場合）でコンテンツペインにフォーカスを移動できる
  - [ ] ターミナルタブがアクティブな状態で `Cmd+0` を押すとツリーパネルにフォーカスが移る

#### F-5: ショートカット一覧表示（ヘルプ）

- **説明**: `?` キーを押すとショートカット一覧モーダルが表示される
- **受け入れ条件**:
  - [ ] ターミナル・テキスト入力欄にフォーカスがない状態で `?` を押すとヘルプモーダルが開く
  - [ ] `Esc` または再度 `?` でモーダルを閉じることができる
  - [ ] 全ショートカットがカテゴリ別に一覧表示される

### 非機能要件

- **ユーザビリティ**: macOS 標準キーバインド（`Cmd+T`, `Cmd+W`）に準拠する
- **保守性**: ショートカット定義を1ファイルに集約し、一覧表示と実際のハンドラが同一の定数から生成される
- **外観・デザイン**: ヘルプモーダルは既存の `SettingsModal` と同様のスタイル（Radix UI Dialog + CSS カスタムプロパティ）で実装する

## スコープ

### 対象

- コンテンツタブの新規作成・閉じる・切り替え
- ターミナルタブの新規作成・閉じる・切り替え
- コンテンツ/ターミナルペインの分割切り替え
- ペイン間フォーカス移動
- ショートカット一覧表示モーダル

### 対象外

- ファイル保存（`Cmd+S`）— 読み取り専用ビューアのため不要
- 検索（`Cmd+F`）— Phase 4 以降で検討
- ウィンドウ管理（`Cmd+N` 複数ウィンドウ）— Phase 3-F の範囲

## 依存関係

- Phase 3-B（テーマ・外観設定）完了済み — SettingsModal のスタイルパターンを流用
- `contentStore.ts` / `terminalStore.ts` に既存のタブ管理アクションが実装済みであること

## 既知の制約

- xterm.js がフォーカスを持っている間はキーイベントが xterm.js に吸収されるため、
  ターミナル内で `?` を押してもヘルプは開かない（これは仕様）
- `Ctrl+Tab` は既存の「コンテンツ↔ターミナルペイン切り替え」に割り当て済みのため、
  タブ切り替えには別のキーを使用する
