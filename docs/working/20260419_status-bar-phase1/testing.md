# テスト手順書 - status-bar-phase1

## 概要

このドキュメントでは、status-bar-phase1 の基盤整備（Rust `git_branch` コマンド、IPC ラッパー、ステータスバー空枠の配置）に対するテスト手順を記載する。可能な限り手動操作で確認し、挙動が画面に出ないロジック部分（Rust の分岐）は自動テストで検証する。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- ローカルに Git がインストールされていること
- 動作確認用に「Git リポジトリのプロジェクト」と「Git リポジトリでないディレクトリ」の両方を手元に準備していること

## 手動テスト

### ケース 1: ステータスバーの枠が表示される

**手順:**

1. `npx tauri dev` でアプリを起動する
2. ウィンドウの最下段を確認する

**期待結果:**

- 画面最下段に高さ 28px（`h-7`）の帯が表示されている
- 帯の上端に 1px のボーダーがあり、既存の他のボーダーと視覚的に揃っている
- 背景色が既存の `--color-bg-elevated`（他パネルと同系）で統一されている
- 帯の中身は空（Phase 1 時点ではラベル等なし、またはプレースホルダのみ）

**確認結果:**

- [ ] OK / NG

---

### ケース 2: レイアウトの高さ崩れがない

**手順:**

1. アプリを起動後、ツリーパネルで任意のプロジェクトを開く
2. コンテンツタブを数枚開いてスクロールしてみる
3. `Ctrl+Tab` でターミナルモードに切り替え、ターミナルで `ls` 等を実行してスクロールを発生させる
4. `Cmd+\\` でコンテンツ分割、`Cmd+Shift+\\` でターミナル分割を切り替える

**期待結果:**

- メインエリア（タブバー + ビューア）のスクロール領域がステータスバー分だけ短くなっているが、コンテンツ/ターミナル自体の描画・スクロールが正常
- 分割レイアウト切り替えでも枠が乱れない
- ターミナルの下端が欠けて見えなくなる等の不具合が発生しない

**確認結果:**

- [ ] OK / NG

---

### ケース 3: ウィンドウリサイズで枠が追従する

**手順:**

1. アプリを起動
2. ウィンドウの高さ・幅を小さく／大きく変える
3. 最下段のステータスバー帯が常に表示されていることを確認

**期待結果:**

- ウィンドウ高さが極端に小さい場合もステータスバーは高さ `h-7` を保ち、メインエリアが先に縮む
- 横方向はウィンドウ幅に追従して伸縮する

**確認結果:**

- [ ] OK / NG

---

### ケース 4: IPC コマンド動作確認（開発者用）

**手順:**

1. アプリの開発者ツール（Tauri dev モード）を開く
2. コンソールで以下を実行:

   ```js
   await window.__TAURI__.core.invoke('git_branch', { cwd: '<リポジトリの絶対パス>' })
   ```

3. 非 Git ディレクトリでも同様に実行

**期待結果:**

- Git リポジトリでは現在のブランチ名（例: `"main"` / `"feature/status-bar"`）が返る
- detached HEAD の場合は 7 桁程度の短縮 SHA が返る
- 非 Git ディレクトリでは `null` が返る（例外ではなく `null`）

**確認結果:**

- [ ] OK / NG

---

### ケース 5: 既存機能の回帰なし

**手順:**

1. 以下の既存操作がこれまで通り動作することを確認:
   - ツリーからファイルを開く
   - Markdown / Code / Image / Plain の各ビューアが正常に表示される
   - ターミナル起動・入力・分割
   - `Ctrl+P` のパス検索パレット
   - `Cmd+Shift+P` のプロンプトパレット
   - キーボードショートカット一覧（`?`）

**期待結果:**

- すべて Phase 1 前と同じ挙動

**確認結果:**

- [ ] OK / NG
