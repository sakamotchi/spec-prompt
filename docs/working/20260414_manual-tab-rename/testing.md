# テスト手順書 - manual-tab-rename（P3）

## 概要

P3（手動リネーム UI）の手動テストおよび自動テスト手順。OSC と手動タイトルの優先関係、編集 UI のキー操作、ドラッグとの競合、通知タイトルへの反映までを確認する。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- Phase 1 / Phase 2 の機能（通知タイトルへのタブ名差し込み、OSC 0/1/2 による動的タイトル）が動作していること
- アプリ内設定で「Claude Code 通知」が ON になっていること（ケース 6 で必要）

---

## 手動テスト

### ケース 1: ダブルクリックでインライン編集に入る

**手順:**

1. タブ 1（`Terminal 1`）のラベル部分をダブルクリックする
2. 入力欄が現れ、既存のラベル（`Terminal 1`）が全選択された状態でフォーカスされることを確認

**期待結果:**

- ダブルクリック後 50ms 程度で入力欄が表示される
- 既存のラベルが初期値としてセットされ、全選択されている
- タブのドラッグハンドル（`draggable`）が無効化されている

**確認結果:**

- [ ] OK / NG

### ケース 2: Enter で確定、表示タイトルが変わる

**手順:**

1. ケース 1 の続きで、入力欄に `watcher` と入力
2. Enter キーを押す

**期待結果:**

- 入力欄が閉じ、タブラベルが `watcher` に変わる
- 該当タブの状態が `pinned=true, manualTitle='watcher'` になる
- 以降、OSC 2 を送ってもラベルは `watcher` のまま（ケース 5 で検証）

**確認結果:**

- [ ] OK / NG

## 自動テスト

### フロントエンドテスト（Vitest）

```bash
npm run test
```

以下が新規に追加されパスすること:

- `renameTab`: 通常のトリム / 空文字 no-op / 同一値で state 参照不変 / ペイン跨ぎ
- `unpinTab`: `pinned=true → false`・`manualTitle` クリア / すでに unpin で state 不変
- `computeDisplayTitle`: pinned+manualTitle > oscTitle > fallbackTitle の 3 段優先
- `pinned` と `setOscTitle` の共存: `oscTitle` は更新、表示は `manualTitle`
- `unpin` 後に最新の `oscTitle` に即復帰

### 型チェック

```bash
npm run build
```

TypeScript エラーがないこと。

### Rustテスト

```bash
cd src-tauri && cargo test
```

本フェーズでは Rust 側変更なし。既存 99 件全件パスを維持していること。
