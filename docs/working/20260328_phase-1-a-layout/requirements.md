# 要件定義書 - Phase 1-A: 全体レイアウト

## 概要

SpecPromptアプリケーションの基本的な2カラムレイアウトを実装する。左ペイン（プロジェクトツリー）と右ペイン（メインエリア）を持ち、ペイン幅をドラッグでリサイズでき、メインエリアで「コンテンツ」と「ターミナル」をタブで切り替えられる最小構成を実現する。

## 背景・目的

Phase 1のコア機能実装にあたり、ファイルツリー・コンテンツビューア・ターミナルを配置する骨格となるレイアウトが必要。このレイアウトが完成して初めて Phase 1-B（ファイルツリー）や Phase 1-C（ターミナル）の実装が可能になる。

## 要件一覧

### 機能要件

#### F-1: 2カラムレイアウト

- **説明**: 画面を左ペイン（プロジェクトツリー）と右ペイン（メインエリア）の2カラムに分割する
- **受け入れ条件**:
  - [ ] 左ペインと右ペインが横並びで表示される
  - [ ] 左ペインはデフォルト幅 240px 程度で表示される
  - [ ] 右ペインは残りの幅を占有する（flex-grow）
  - [ ] 画面全体に対してウィンドウ高さいっぱいに広がる

#### F-2: ペイン幅リサイズ

- **説明**: 左右ペイン間のセパレーターをドラッグして幅を自由に変更できる
- **受け入れ条件**:
  - [ ] セパレーター（幅4px程度のドラッグハンドル）がペイン間に表示される
  - [ ] セパレーターをドラッグしてペイン幅を変更できる
  - [ ] 左ペインの最小幅・最大幅が設定されており、縮みすぎ/広がりすぎを防ぐ（例: min 160px, max 480px）
  - [ ] リサイズ中はカーソルが `col-resize` になる

#### F-3: メインタブ切り替え

- **説明**: 右ペイン上部にコンテンツ/ターミナルの切り替えタブを設置し、Ctrl+Tab でも切り替えられる
- **受け入れ条件**:
  - [ ] 「コンテンツ」「ターミナル」の2つのタブが右ペイン上部に表示される
  - [ ] タブをクリックでメインエリアのコンテンツが切り替わる
  - [ ] `Ctrl+Tab` キーでコンテンツ↔ターミナルが切り替わる
  - [ ] アクティブなタブが視覚的に区別できる（スタイル差異）

### 非機能要件

- **パフォーマンス**: レイアウトの再レンダリングはペイン幅変更時のみ発生し、子コンポーネントに不要な再レンダリングを起こさない
- **ユーザビリティ**: セパレーターはホバー時にハイライトされ、ドラッグ可能であることが直感的にわかる
- **保守性**: `Layout/` コンポーネントと `SplitPane/` コンポーネントを分離し、SplitPane は将来のコンテンツ分割・ターミナル分割でも再利用できる設計にする
- **外観・デザイン**: VS Code / Linear 系のダークテーマを基調とし、デスクトップアプリとして洗練された見た目にする。カラーパレットは `src/index.css` で CSS カスタムプロパティとして定義し、全コンポーネントで一貫して使用する

## スコープ

### 対象

- `src/components/Layout/` の新規実装
- `src/components/MainArea/MainTabs.tsx` の新規実装
- `src/components/SplitPane/` の新規実装（水平分割のみ、Phase 1-A スコープ）
- `src/stores/appStore.ts` の `activeMainTab` 状態追加
- `src/App.tsx` でのレイアウト組み込み

### 対象外

- 左ペインへの実際のファイルツリー実装（Phase 1-B）
- 右ペインへのコンテンツビューア実装（Phase 2-A）
- 右ペインへのターミナル実装（Phase 1-C）
- コンテンツ分割・ターミナル分割（Phase 2-C）

## 実装対象ファイル（予定）

- `src/App.tsx`
- `src/index.css`（カラーパレット・フォント設定追加）
- `src/components/Layout/AppLayout.tsx`
- `src/components/MainArea/MainTabs.tsx`
- `src/components/SplitPane/SplitPane.tsx`
- `src/stores/appStore.ts`

## 依存関係

- Phase 0-E が完了していること（Tauri + React プロジェクトが動作する状態）
- Phase 1-D（状態管理基盤）の `appStore` の `activeMainTab` と並行して実装可能だが、`appStore` の型定義を先行して決める必要がある
- 外部ライブラリ: `@radix-ui/react-tabs`、`lucide-react`（npm install が必要）

## 既知の制約

- Tauri WebView 上で動作するため、OS ネイティブのリサイズ API は使えない。ドラッグリサイズは `mousedown` / `mousemove` / `mouseup` イベントで実装する
- Tailwind CSS v4 を使用するため、`tailwind.config.js` は不要。カスタムCSS が必要な場合は `index.css` に記述する
