# テスト手順書 - Phase 1-A: 全体レイアウト

## 概要

Phase 1-A（全体レイアウト）の手動テスト手順を記載します。
2カラムレイアウト・ペイン幅リサイズ・メインタブ切り替えの動作を確認します。
可能な限り手動操作で確認し、操作で確認できない項目のみ自動テストを記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- Phase 0-E の環境構築が完了していること

---

## 手動テスト

### ケース 1: 2カラムレイアウト表示確認

**手順:**

1. `npx tauri dev` でアプリを起動する
2. アプリウィンドウが開いたことを確認する
3. 画面レイアウトを目視で確認する

**期待結果:**

- 画面が左右2カラムに分割されている
- 左ペインが左側に、右ペイン（MainArea）が右側に配置されている
- セパレーター（仕切り線）が両ペインの間に表示されている
- 両ペインがウィンドウ高さいっぱいに広がっている

**確認結果:**

- [x] OK / NG

---

### ケース 2: ペイン幅リサイズ（ドラッグ）

**手順:**

1. アプリを起動する
2. 左ペインと右ペインの間のセパレーターにカーソルを合わせる
3. カーソルが `col-resize`（横矢印）に変わることを確認する
4. セパレーターをクリックしたままドラッグして右へ動かす
5. ドラッグを離す
6. 次に左へドラッグして元の幅付近に戻す

**期待結果:**

- カーソルが `col-resize` に変わる
- ドラッグに追従してペイン幅がリアルタイムに変化する
- ドラッグを離した後もペイン幅が変更された状態を維持する
- 左ペインが最小幅（160px程度）を下回らない
- 左ペインが最大幅（480px程度）を超えない

**確認結果:**

- [x] OK / NG

---

### ケース 3: セパレーターホバーハイライト

**手順:**

1. アプリを起動する
2. セパレーターにカーソルを乗せる
3. セパレーターの外にカーソルを移動する

**期待結果:**

- カーソルをセパレーターに乗せると背景色やボーダー色が変わり、ドラッグ可能であることがわかる
- カーソルを外すと元のスタイルに戻る

**確認結果:**

- [x] OK / NG

---

### ケース 4: メインタブ クリック切り替え

**手順:**

1. アプリを起動する
2. 右ペイン上部に「コンテンツ」と「ターミナル」の2つのタブが表示されていることを確認する
3. 「コンテンツ」タブがデフォルトでアクティブになっていることを確認する
4. 「ターミナル」タブをクリックする
5. 「コンテンツ」タブをクリックして戻す

**期待結果:**

- 「コンテンツ」と「ターミナル」の2タブが表示されている
- デフォルトで「コンテンツ」がアクティブ（ボーダーボトムやスタイル差異がある）
- 「ターミナル」クリック後、アクティブスタイルが「ターミナル」に移る
- 「コンテンツ」クリックで再び「コンテンツ」がアクティブになる

**確認結果:**

- [x] OK / NG

---

### ケース 5: メインタブ Ctrl+Tab キー切り替え

**手順:**

1. アプリを起動する
2. 「コンテンツ」タブがアクティブな状態で `Ctrl+Tab` を押す
3. さらに `Ctrl+Tab` を押す

**期待結果:**

- 1回目の `Ctrl+Tab`: 「ターミナル」がアクティブになる
- 2回目の `Ctrl+Tab`: 「コンテンツ」がアクティブになる（トグル動作）

**確認結果:**

- [x] OK / NG

---

### ケース 6: ウィンドウリサイズ後のレイアウト維持

**手順:**

1. アプリを起動する
2. アプリウィンドウを縦横にリサイズする（小さく・大きく）
3. ペイン幅リサイズを行った後にウィンドウをリサイズする

**期待結果:**

- ウィンドウリサイズ後もペインの比率・構成が崩れない
- 右ペインがウィンドウ幅に追従して広がる/縮まる
- セパレーター位置が適切に維持される

**確認結果:**

- [x] OK / NG

## 自動テスト

```bash
npm run test
```

確認項目:
- `SplitPane`: 子要素が2つレンダリングされること
- `MainTabs`: タブクリックで `activeMainTab` が変わること

### 型チェック

```bash
npm run build
```

TypeScript のコンパイルエラーがないことを確認する。

### Lintチェック

```bash
npm run lint
```

ESLint エラーがないことを確認する。
