# テスト手順書 - Phase 3-H: ウィンドウセッション復元

## 概要

ウィンドウセッション復元機能のテスト手順を記載します。ウィンドウの開閉時にセッションが正しく保存・削除され、次回起動時に復元されることを手動で確認します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト用に複数のプロジェクトフォルダが準備されていること

---

## 手動テスト

### ケース 1: 複数ウィンドウ → 終了 → 再起動で復元される

**手順:**

1. アプリを起動し、プロジェクト A を開く
2. フォルダ右クリック →「新規ウィンドウで開く」でプロジェクト B を別ウィンドウで開く
3. ツールバーの新規ウィンドウボタンで空の3つ目のウィンドウを開く
4. 全ウィンドウを閉じてアプリを終了する
5. アプリを再起動する

**期待結果:**

- 3つのウィンドウが復元される
- ウィンドウ1: プロジェクト A が開いている
- ウィンドウ2: プロジェクト B が開いている
- ウィンドウ3: 空の状態（プロジェクト未選択）

**確認結果:**

- [x] OK / NG

---

### ケース 2: ウィンドウを1つ閉じた状態で終了 → 再起動

**手順:**

1. アプリを起動し、プロジェクト A を開く
2. フォルダ右クリックでプロジェクト B を別ウィンドウで開く
3. プロジェクト B のウィンドウを閉じる（プロジェクト A のウィンドウは残す）
4. アプリを終了する
5. アプリを再起動する

**期待結果:**

- ウィンドウが1つだけ復元される（プロジェクト A のみ）
- プロジェクト B のウィンドウは復元されない

**確認結果:**

- [x] OK / NG

---

### ケース 3: 1ウィンドウのみで終了 → 再起動

**手順:**

1. アプリを起動し、プロジェクト A を開く（1ウィンドウのみ）
2. アプリを終了する
3. アプリを再起動する

**期待結果:**

- 追加ウィンドウは復元されない（通常の1ウィンドウ起動）

**確認結果:**

- [x] OK / NG

---

### ケース 4: 存在しないプロジェクトパスが保存されていた場合

**手順:**

1. アプリを起動し、プロジェクト A を別ウィンドウで開く
2. アプリを終了する
3. プロジェクト A のフォルダを削除またはリネームする
4. アプリを再起動する

**期待結果:**

- アプリがクラッシュしない
- 存在しないパスのウィンドウが空の状態で開かれる（または復元をスキップする）

**確認結果:**

- [x] OK / NG

---

### ケース 5: ウィンドウ開閉後のセッション保存確認（localStorage 直接確認）

**手順:**

1. アプリを起動する
2. 別ウィンドウを2つ開く
3. ブラウザデベロッパーツール相当（Tauri DevTools）で localStorage の `specprompt-window-sessions` キーの値を確認する
4. ウィンドウを1つ閉じる
5. 再度 localStorage の値を確認する

**期待結果:**

- 手順3: 2件のセッション情報が JSON 配列で保存されている
- 手順5: 1件に減っている

**確認結果:**

- [x] OK / NG

---

### ケース 6: 復元ウィンドウが独立して動作する

**手順:**

1. ケース 1 の手順で3ウィンドウを復元する
2. 各ウィンドウでファイルを開いたり、ターミナルを操作する

**期待結果:**

- 各ウィンドウが独立して動作する（Phase 3-G のケース 4 と同じ）

**確認結果:**

- [x] OK / NG

---

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| localStorage の `specprompt-window-sessions` を手動で壊した JSON に書き換えた場合 | アプリがクラッシュせず、セッション復元をスキップして通常起動する | OK |
| 多数のウィンドウ（5つ以上）を保存して復元した場合 | 全ウィンドウが正常に復元される | OK |
| メインウィンドウだけを閉じ（他のウィンドウが残っている状態）で再起動した場合 | セッションが保存されており、再起動時に復元される | OK |
