# テスト手順書 - status-bar-phase2

## 概要

このドキュメントでは、status-bar-phase2 の機能実装（`useGitBranch`、`BranchIndicator`、`FileTypeIndicator`、`StatusBar` 統合）に対するテスト手順を記載する。可能な限り手動操作で確認し、フック内部ロジックは Vitest で補完する。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- Phase 1 のステータスバー枠が画面下段に表示されていること
- ローカルに `git` がインストールされていること
- 動作確認用に以下を準備していること:
  - Git リポジトリのプロジェクトフォルダ（複数ブランチがあると理想）
  - Git リポジトリでないディレクトリ
  - Markdown / コード (.ts, .rs 等) / 画像 / プレーンテキスト (.txt) の各ファイル

## 手動テスト

### ケース 1: ブランチ名が初期表示される

**手順:**

1. アプリ起動後、Git リポジトリのプロジェクトを開く（ツリーパネルからプロジェクト選択）
2. 画面下段のステータスバー左端を確認する

**期待結果:**

- 左端に Git アイコン + 現在のブランチ名（例: `main`、`feature/status-bar`）が表示されている
- フォント色は他 UI の muted テキストと揃っている

**確認結果:**

- [ ] OK / NG

---

### ケース 2: ブランチ切替時に自動更新される

**手順:**

1. ケース 1 の状態でアプリ内ターミナルを開く
2. `git switch -c test/status-bar-verify` で新ブランチを作成
3. 3 秒以内にステータスバー左端の表示を確認
4. `git switch main` などで別ブランチへ切り替え、再度確認
5. 動作確認後、作業用ブランチを削除して元に戻す

**期待結果:**

- `git switch` 実行後、3 秒以内にブランチ名が更新される（手動リロード不要）
- 切替前後で表示に齟齬がない

**確認結果:**

- [ ] OK / NG

---

### ケース 3: detached HEAD で短縮 SHA が表示される

**手順:**

1. ターミナルで `git log --oneline -n 3` を実行し、表示されたコミットの一つを選ぶ
2. `git checkout <sha>` で detached HEAD 状態にする
3. ステータスバー左端を確認
4. `git switch -` で元のブランチに戻す

**期待結果:**

- detached HEAD 時は `HEAD` 文字列ではなく 7 桁程度の短縮 SHA が表示される
- 元のブランチに戻すと通常のブランチ名表示に戻る

**確認結果:**

- [ ] OK / NG

---

### ケース 4: 非 Git ディレクトリではブランチ名が表示されない

**手順:**

1. Git 初期化していないディレクトリをプロジェクトとして開く
2. ステータスバー左端を確認

**期待結果:**

- Git アイコン・ブランチ名ともに表示されない
- エラー・コンソール警告が出ない（ブランチ欄がスペースとして空く）

**確認結果:**

- [ ] OK / NG

---

### ケース 5: ファイル種別が表示される

**手順:**

1. Git リポジトリのプロジェクトを開く
2. ツリーから以下の順にファイルを開く:
   - `.md` ファイル
   - `.ts` もしくは `.rs` ファイル
   - `.png` などの画像ファイル
   - `.txt` や拡張子のないファイル
3. 各ファイルを開くたびにステータスバー右端の表示を確認する

**期待結果:**

- `.md` → `Markdown`（`FileText` アイコン）
- `.ts` / `.rs` / 他コード → `Code`（`Code` アイコン）
- `.png` / `.jpg` → `Image`（`Image` アイコン）
- `.txt` / 拡張子なし → `Plain`（`File` アイコン）

**確認結果:**

- [ ] OK / NG

---

### ケース 6: ターミナルモードではファイル種別が非表示

**手順:**

1. ケース 5 の状態で `Ctrl+Tab` を押してターミナルモードへ切替
2. ステータスバー右端を確認
3. もう一度 `Ctrl+Tab` で content モードへ戻す

**期待結果:**

- ターミナルモード時はファイル種別インジケータが非表示
- ブランチ名はターミナルモードでも表示継続
- content モードに戻ると元のファイル種別が再度表示される

**確認結果:**

- [ ] OK / NG

---

### ケース 7: 分割レイアウトでフォーカス連動

**手順:**

1. コンテンツ分割 (`Cmd+\\`) を有効化する
2. 左ペインに `.md` ファイル、右ペインに `.ts` ファイルを開く
3. 左ペインのタブをクリックしてフォーカス → ステータスバー右端が `Markdown` になるか確認
4. 右ペインのタブをクリックしてフォーカス → `Code` に切り替わるか確認

**期待結果:**

- `focusedPane` に追従してファイル種別が切り替わる

**確認結果:**

- [ ] OK / NG

---

### ケース 8: アクティブタブが空（filePath=null）の場合

**手順:**

1. 新規空タブを作成（`Cmd+T` 等）し、ファイルを開かない状態にする
2. そのタブをアクティブにしてステータスバー右端を確認

**期待結果:**

- ファイル種別は非表示（空スペース）
- ブランチ名は通常通り表示

**確認結果:**

- [ ] OK / NG

---

### ケース 9: プロジェクト切替で再取得される

**手順:**

1. プロジェクト A（Git）を開き、ブランチ名が表示されることを確認
2. プロジェクト B（別の Git リポジトリ、別ブランチ）を開く
3. ステータスバーのブランチ名を確認

**期待結果:**

- B のブランチ名に即座に更新される（A のブランチ名は残らない）
- コンソール警告・エラーなし

**確認結果:**

- [ ] OK / NG
