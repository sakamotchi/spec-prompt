# テスト手順書 - Git ステータスのプロジェクトツリー表示

## 概要

プロジェクトツリーのファイル名に Git ステータス（変更・追加・削除・未追跡）が
色とバッジで表示されることを確認する。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト対象のプロジェクトが Git リポジトリであること

---

## 手動テスト

### ケース 1: 変更ファイルの表示（unstaged）

**手順:**

1. Git リポジトリのプロジェクトを開く
2. ターミナルで `echo "test" >> README.md` を実行して既存ファイルを変更する
3. プロジェクトツリーを確認する

**期待結果:**

- `README.md` のファイル名が黄色（`#e2c08d`）で表示される
- ファイル名の右側に `M` バッジが表示される

**確認結果:**

- [x] OK

---

### ケース 2: ステージ済みファイルの表示

**手順:**

1. ターミナルで `git add README.md` を実行する
2. プロジェクトツリーを確認する

**期待結果:**

- `README.md` のファイル名が緑色（`#73c991`）で表示される
- `M` バッジが表示される

**確認結果:**

- [x] OK

---

### ケース 3: 未追跡ファイルの表示

**手順:**

1. ターミナルで `touch newfile.txt` を実行して新規ファイルを作成する
2. プロジェクトツリーを確認する

**期待結果:**

- `newfile.txt` が緑色で表示される
- `U` バッジが表示される

**確認結果:**

- [x] OK

---

### ケース 4: 削除ファイルの表示

**手順:**

1. ターミナルで追跡済みファイルを実際に削除する: `rm README.md`
2. プロジェクトツリーを確認する（ツリーからファイルが消える）
3. ターミナルで `git status` を実行し `deleted: README.md` を確認する
4. `git checkout -- README.md` で元に戻す

**期待結果:**

- 削除されたファイルはツリーから消えるため赤色表示はされない（ツリーはディスク上のファイルのみ表示）
- 親ディレクトリに変更色が伝播する（削除エントリが gitStatuses に含まれるため）

**備考:** `git rm --cached` はインデックスからの削除であり、ファイルはディスクに残るため未追跡（緑/U）として表示される。これは正常な動作。

**確認結果:**

- [x] OK

---

### ケース 5: ディレクトリへの色伝播

**手順:**

1. `src/` ディレクトリ配下のファイルを変更する
2. プロジェクトツリーで `src/` ディレクトリ名を確認する

**期待結果:**

- `src/` ディレクトリ名にも色が付いている（配下の変更を示す）
- 親ディレクトリにも色が伝播している

**確認結果:**

- [x] OK

---

### ケース 6: git コマンド実行後の自動更新

**手順:**

1. ファイルを変更し、ツリーで黄色表示を確認する
2. ターミナルで `git add .` を実行する
3. ツリーの色が自動で変わるか確認する（500ms 程度で更新）

**期待結果:**

- `git add` 後に自動でツリーの色が黄色→緑色に変わる

**確認結果:**

- [x] OK

---

### ケース 7: git commit 後のステータスクリア

**手順:**

1. `git add .` → `git commit -m "test"` を実行する
2. プロジェクトツリーを確認する

**期待結果:**

- コミット済みファイルの色・バッジが消える（通常の白色に戻る）

**確認結果:**

- [x] OK

---

### ケース 8: Git リポジトリでないプロジェクト

**手順:**

1. Git リポジトリでないフォルダをプロジェクトとして開く

**期待結果:**

- エラーが発生しない
- すべてのファイルが通常の色で表示される（バッジなし）

**確認結果:**

- [x] OK
