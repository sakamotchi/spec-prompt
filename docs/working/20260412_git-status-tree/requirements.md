# 要件定義書 - Git ステータスのプロジェクトツリー表示

## 概要

VS Code のように、プロジェクトツリーのファイル名に Git のステータス（変更・追加・削除・未追跡など）を
色で表現する機能を追加する。

## 背景・目的

現在のプロジェクトツリーはファイル名を均一な色で表示しており、どのファイルが変更済みか一目で判別できない。
Git ステータスをツリー上に可視化することで、仕様駆動開発のワークフロー（仕様を書く → 実装する → 差分を確認する）を
ツリーパネルだけで追えるようにする。

---

## 機能要件

### F-1: Git ステータスの取得

| ID | 要件 |
|----|------|
| F-1-1 | プロジェクトルートが Git リポジトリの場合、`git status --porcelain` 相当の情報を取得できる |
| F-1-2 | 各ファイルのステータス（Modified, Added, Deleted, Untracked, Renamed）を取得できる |
| F-1-3 | ステージング状態（staged / unstaged）を区別できる |

### F-2: ツリー上のステータス表示

| ID | 要件 |
|----|------|
| F-2-1 | ファイル名の色をステータスに応じて変更する |
| F-2-2 | ディレクトリに変更を含むファイルがある場合、ディレクトリ名にも色を付ける（伝播） |
| F-2-3 | ファイル名の右側にステータスバッジ（M, A, D, U 等）を表示する |

### F-3: ステータスの自動更新

| ID | 要件 |
|----|------|
| F-3-1 | プロジェクトを開いた時に自動取得する |
| F-3-2 | ターミナルで Git コマンド実行後（PTY 出力後）に自動更新する |
| F-3-3 | ファイル保存後に自動更新する |

---

## カラー定義

VS Code 互換のカラースキームを採用する:

| ステータス | 色 | 記号 | 説明 |
|----------|-----|------|------|
| Modified (unstaged) | `#e2c08d` (黄) | M | 変更あり（未ステージ） |
| Modified (staged) | `#73c991` (緑) | M | 変更あり（ステージ済み） |
| Added / Untracked | `#73c991` (緑) | A / U | 新規追加 / 未追跡 |
| Deleted | `#c74e39` (赤) | D | 削除済み |
| Renamed | `#73c991` (緑) | R | リネーム |
| Conflict | `#e5c07b` (橙) | C | コンフリクト |
| Ignored | `#636363` (灰) | - | .gitignore 対象 |

---

## 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | `git status` の実行は 500ms 以内に完了すること |
| 更新頻度 | 自動更新はデバウンス（500ms）を適用し過度な実行を防ぐ |
| Git 非対応 | プロジェクトが Git リポジトリでない場合は何も表示しない |

---

## 対象外

- Git diff の詳細表示（変更行の表示）
- Git ブランチの切り替え UI
- Git commit / push / pull の UI
- .gitignore ファイルのステータス表示（灰色表示は将来対応）

## 実装対象ファイル（予定）

- `src-tauri/src/commands/git.rs` — 新規
- `src/stores/appStore.ts` — gitStatuses 追加
- `src/components/TreePanel/TreeNode.tsx` — カラー・バッジ表示
- `src/lib/tauriApi.ts` — `getGitStatus` 追加
- `src/i18n/locales/*.json` — ラベル追加（必要に応じて）
