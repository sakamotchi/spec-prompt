# テスト手順書 - Phase 2-F: プロジェクト管理

## 概要

Config Manager・最近開いたプロジェクト履歴・プロジェクト切り替え機能の手動テスト手順を記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト用に2つ以上のフォルダを用意しておく（例: `spec-prompt` と `tmp-test-project`）

---

## 手動テスト

### ケース 1: 初回起動時のデフォルト状態

**手順:**

1. `~/Library/Application Support/spec-prompt/config.json` が存在する場合は削除する
2. アプリを起動する
3. ツリーパネルのヘッダーにあるプロジェクト名（「プロジェクト」）をクリックする

**期待結果:**

- ドロップダウンが開く
- 「最近開いたプロジェクト」の下に項目が表示されない（空）
- `~/Library/Application Support/spec-prompt/config.json` が新規作成されている

**確認結果:**

- [x] OK / NG

---

### ケース 2: プロジェクトを開くと履歴に追加される

**手順:**

1. ヘッダーの 📂 ボタンをクリックして `spec-prompt` フォルダを開く
2. ヘッダーのプロジェクト名（`spec-prompt`）をクリックする

**期待結果:**

- ドロップダウンに `spec-prompt` のパスが表示される
- 現在のプロジェクトにチェックマーク（✓）が付いている

**確認結果:**

- [x] OK / NG

---

### ケース 3: 複数プロジェクトが履歴に蓄積される

**手順:**

1. ケース 2 の後、📂 ボタンで別のフォルダ（`tmp-test-project`）を開く
2. ヘッダーのプロジェクト名をクリックする

**期待結果:**

- ドロップダウンに `tmp-test-project`（先頭）と `spec-prompt` の2件が表示される
- `tmp-test-project` にチェックマークが付いている

**確認結果:**

- [x] OK / NG

---

### ケース 4: 履歴からプロジェクトを切り替える

**手順:**

1. ケース 3 の状態からドロップダウンを開く
2. `spec-prompt` をクリックする

**期待結果:**

- ファイルツリーが `spec-prompt` の内容に切り替わる
- ドロップダウンの `spec-prompt` にチェックマークが移動する
- 開いていたコンテンツタブがすべて閉じる

**確認結果:**

- [x] OK / NG

---

### ケース 5: 切り替え時にツリーの展開状態がリセットされる

**手順:**

1. `spec-prompt` を開き、いくつかのフォルダを展開する
2. ドロップダウンから `tmp-test-project` に切り替える
3. 再度 `spec-prompt` に切り替える

**期待結果:**

- `spec-prompt` に戻ったとき、フォルダはすべて折りたたまれた状態になっている（展開状態はリセット）

**確認結果:**

- [x] OK / NG

---

### ケース 6: 切り替え時にコンテンツタブが閉じる

**手順:**

1. `spec-prompt` でいくつかのファイルをクリックしてコンテンツタブで開く
2. ドロップダウンから `tmp-test-project` に切り替える

**期待結果:**

- コンテンツタブがすべて閉じ、空の状態になる

**確認結果:**

- [x] OK / NG

---

### ケース 7: ターミナルセッションは切り替え後も維持される

**手順:**

1. ターミナルタブを開き、何かコマンドを実行する（例: `echo hello`）
2. プロジェクトを別のフォルダに切り替える

**期待結果:**

- ターミナルセッションは維持されており、以前のコマンド履歴が残っている

**確認結果:**

- [x] OK / NG

---

### ケース 8: 同じプロジェクトを再度開いても履歴が重複しない

**手順:**

1. `spec-prompt` が履歴にある状態で、📂 ボタンから再度 `spec-prompt` を開く
2. ドロップダウンを確認する

**期待結果:**

- `spec-prompt` は1件だけ表示される（重複しない）
- 先頭（最新）の位置に移動している

**確認結果:**

- [x] OK / NG

---

### ケース 9: 履歴の上限（10件）

**手順:**

1. 11件の異なるフォルダを順番に開く
2. ドロップダウンを確認する

**期待結果:**

- ドロップダウンに表示されるのは最新10件のみ
- 最初に開いたフォルダは一覧から消えている

**確認結果:**

- [x] OK / NG

---

### ケース 10: アプリ再起動後も履歴が保持される

**手順:**

1. 数件のプロジェクトを開いて履歴を作る
2. アプリを終了する
3. アプリを再起動する
4. ドロップダウンを確認する

**期待結果:**

- 再起動前と同じ履歴が表示される
- `~/Library/Application Support/spec-prompt/config.json` に保存されていることを確認

**確認結果:**

- [x] OK / NG

---

### ケース 11: config.json の内容確認

**手順:**

1. 数件のプロジェクトを開く
2. ターミナルで以下を実行する（開発・本番ともに同じパス）：
   ```bash
   cat ~/Library/Application\ Support/spec-prompt/config.json
   ```

**期待結果:**

- JSON ファイルに `recent_projects` 配列が保存されている
- 最新のプロジェクトが先頭にある

**確認結果:**

- [x] OK / NG

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| config.json が壊れた JSON の場合 | デフォルト値で起動（クラッシュしない） | OK |
| config.json の書き込み権限がない場合 | エラーをコンソールに出力、アプリは継続動作 | OK |
| 存在しないパスが履歴にある場合 | ドロップダウンに表示する（選択時エラーを出さない、新規 open ダイアログを開く） | OK |
