# テスト手順書 - タブ一括クローズアクション

## 概要

このドキュメントでは、コンテンツタブとターミナルタブに追加した一括クローズアクション（すべて／その他／右側）のテスト手順を記載します。可能な限り手動操作で確認し、操作で確認できない項目のみ自動テストを記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- 任意のプロジェクトフォルダを開いていること
- ファイルツリーに開閉できるファイルが 3 件以上あること（`a.md`, `b.md`, `c.md` 等）

## 手動テスト

### コンテンツタブ

#### ケース C-1: 右クリックでコンテキストメニューが表示される（F-1）

**手順:**

1. アプリを起動し、`a.md`, `b.md`, `c.md` をツリーから順に開く（タブが 3 つ並ぶ）
2. 任意のタブを右クリックする

**期待結果:**

- メニューが表示され、上から「右側のタブを閉じる」「その他のタブを閉じる」「すべてのタブを閉じる」「（区切り）」「タブを閉じる」が並んでいる
- メニュー外をクリックまたは `Esc` で閉じる

**確認結果:**

- [ ] OK / NG

---

#### ケース C-2: コンテンツタブ「すべてのタブを閉じる」（F-2）

**手順:**

1. `a.md`, `b.md`, `c.md` の 3 タブを開く
2. 真ん中の `b.md` タブを右クリック →「すべてのタブを閉じる」を選択

**期待結果:**

- すべてのタブが閉じ、新しい空タブ（"新しいタブ" 表記）が 1 枚残る
- 残ったタブがアクティブになっている

**確認結果:**

- [ ] OK / NG

---

#### ケース C-3: コンテンツタブ「その他のタブを閉じる」（F-3）

**手順:**

1. `a.md`, `b.md`, `c.md` の 3 タブを開き、`a.md` をアクティブにする
2. `b.md` タブを右クリック →「その他のタブを閉じる」を選択

**期待結果:**

- `b.md` のみが残り、それがアクティブになる
- `b.md` のスクロール位置・ビューモードは保持されている

**確認結果:**

- [ ] OK / NG

---

#### ケース C-4: コンテンツタブ「右側のタブを閉じる」（F-4）

**手順:**

1. `a.md`, `b.md`, `c.md`, `d.md` の 4 タブを開き、`d.md` をアクティブにする
2. `b.md` タブを右クリック →「右側のタブを閉じる」を選択

**期待結果:**

- `a.md`, `b.md` の 2 枚が残る
- アクティブだった `d.md` が消えたため、`b.md`（右クリック対象）にフォールバックしてアクティブになる

**確認結果:**

- [ ] OK / NG

#### ケース C-5: コンテンツタブ「右側のタブを閉じる」が最右端では無効化される（F-4）

3. プライマリペインの `b.md` を右クリック →「すべてのタブを閉じる」を選択

**期待結果:**

- プライマリペインは空タブ 1 枚のみ
- セカンダリペインの `x.md`, `y.md` は変化なし

**確認結果:**

- [ ] OK / NG

#### ケース T-2: ターミナルタブ「すべてのタブを閉じる」（F-6）

**手順:**

1. `+` ボタンでターミナルタブを 3 枚に増やす
2. 各ターミナルで `echo "tab1"`, `echo "tab2"`, `echo "tab3"` を実行（タブごとにユニークな出力）
3. 真ん中のタブを右クリック →「すべてのタブを閉じる」を選択

**期待結果:**

- すべてのタブが閉じ、新しい空タブ「Terminal 1」が 1 枚残る
- 新しいシェルが起動し、プロンプトが表示される
- アクティビティモニタで PTY プロセスが残留していないことを確認（macOS: `ps -ax | grep zsh`）

**確認結果:**

- [ ] OK / NG

**期待結果:**

- ピン留め状態・タブ名・既存のシェル出力（`echo "keep this"` のスクロールバック）が保持されている
- 他 2 タブの PTY が終了している
