# 要件定義書 - タブ一括クローズアクション

## 概要

コンテンツタブおよびターミナルタブに対して、以下 3 つの一括クローズアクションを追加する。

- すべてのタブを閉じる
- その他のタブを閉じる（基準タブ以外）
- 右側のタブを閉じる（基準タブの右側）

それぞれのタブ種別ごとに専用のコンテキストメニューを実装する（コンポーネントは共通化せず、コンテンツ用・ターミナル用の 2 ファイルとして独立して維持する）。

## 背景・目的

- ユーザーがタブを多数開いた際、1 件ずつ「×」または `Ctrl+W` で閉じる操作はコストが高い。
- 既存のターミナルタブには Radix ベースのコンテキストメニュー（`TabContextMenu`）が存在するが、コンテンツタブにはコンテキストメニュー自体がまだ未実装。
- VS Code・各種ブラウザ等で広く普及しているタブ操作 UX を取り入れ、SDDesk の利便性を底上げする。
- 「コンテンツとターミナルでメニュー構成は今後も独自に拡張される」という前提で、共通化（共有コンポーネント化）はあえて行わず、それぞれ独立した実装ファイルで保守する。

## 要件一覧

### 機能要件

#### F-1: コンテンツタブのコンテキストメニュー新設

- **説明**: コンテンツタブバー上の各タブを右クリックすると、Radix ContextMenu によるメニューが表示される。
- **受け入れ条件**:
  - [ ] タブ上で右クリックするとコンテキストメニューが開く
  - [ ] メニュー外クリックまたは `Esc` で閉じる
  - [ ] メニュー項目に「すべてのタブを閉じる」「その他のタブを閉じる」「右側のタブを閉じる」が含まれる
  - [ ] 既存のドラッグ＆ドロップ（タブ移動）には影響しない

#### F-2: コンテンツタブ — すべてのタブを閉じる

- **説明**: 操作したペイン内のすべてのコンテンツタブを閉じる。最後の 1 枚は空タブ（`filePath: null`）に差し替える（既存 `closeTab` 仕様に揃える）。
- **受け入れ条件**:
  - [ ] 操作したペイン内のタブがすべて閉じる
  - [ ] ペインに新規空タブが 1 枚残り、それがアクティブになる
  - [ ] もう片方のペイン（分割時）のタブには影響しない

#### F-3: コンテンツタブ — その他のタブを閉じる

- **説明**: 右クリックされたタブ以外を、操作したペイン内ですべて閉じる。
- **受け入れ条件**:
  - [ ] 基準タブのみが残り、それがアクティブになる
  - [ ] 残ったタブのスクロール位置・ビューモードは保持される
  - [ ] もう片方のペイン（分割時）のタブには影響しない

#### F-4: コンテンツタブ — 右側のタブを閉じる

- **説明**: 右クリックされたタブより右にあるタブを、操作したペイン内で閉じる。
- **受け入れ条件**:
  - [ ] 基準タブが最右端の場合はメニュー項目を disabled にする
  - [ ] アクティブタブが右側にあった場合は、基準タブをアクティブにフォールバックする
  - [ ] もう片方のペイン（分割時）のタブには影響しない

#### F-5: ターミナルタブのコンテキストメニュー拡張

- **説明**: 既存の `TabContextMenu`（`src/components/TerminalPanel/TabContextMenu.tsx`）に F-6〜F-8 のメニュー項目を追加する。コンテンツ用メニューとはコンポーネントを共有しない。
- **受け入れ条件**:
  - [ ] 既存メニュー項目（リネーム／ピン解除／プロンプトパレット起動／閉じる）の挙動は変わらない
  - [ ] 新規 3 項目が「閉じる」項目より上、または同セクション内の自然な位置に追加される

#### F-6: ターミナルタブ — すべてのタブを閉じる

- **説明**: 操作したペイン内のすべてのターミナルタブを閉じる。最後の 1 枚は新しい空タブに差し替えてシェルを再起動させる（既存 `handlePtyExited` 仕様に揃える）。
- **受け入れ条件**:
  - [ ] 閉じる各タブの PTY が `close_pty` で終了する
  - [ ] 閉じる各タブの `ptyId` について `notifyPromptPaletteOfPtyClosed` が呼ばれる
  - [ ] 最終的にペインには新規空タブが 1 枚残り、それがアクティブになる

#### F-7: ターミナルタブ — その他のタブを閉じる

- **説明**: 右クリックされたタブ以外を、操作したペイン内ですべて閉じる。
- **受け入れ条件**:
  - [ ] 閉じる各タブの PTY が `close_pty` で終了する
  - [ ] 閉じる各タブの `ptyId` について `notifyPromptPaletteOfPtyClosed` が呼ばれる
  - [ ] 基準タブのみが残り、それがアクティブになる
  - [ ] 基準タブのピン留め状態・OSC タイトル・未読マーク等は保持される

#### F-8: ターミナルタブ — 右側のタブを閉じる

- **説明**: 右クリックされたタブより右にあるタブを、操作したペイン内で閉じる。
- **受け入れ条件**:
  - [ ] 閉じる各タブの PTY が `close_pty` で終了する
  - [ ] 閉じる各タブの `ptyId` について `notifyPromptPaletteOfPtyClosed` が呼ばれる
  - [ ] 基準タブが最右端の場合はメニュー項目を disabled にする
  - [ ] アクティブタブが右側にあった場合は、基準タブをアクティブにフォールバックする

#### F-9: i18n（日英）

- **説明**: 追加するメニュー項目のラベルを ja/en 両方の locale に追加する。
- **受け入れ条件**:
  - [ ] `src/i18n/locales/ja.json` / `en.json` に各 3 キーが追加されている
  - [ ] 既存翻訳キー（`terminal.tabMenu.*`）の命名・階層と整合する
  - [ ] コンテンツ用には `content.tabMenu.*` 名前空間を新設する

### 非機能要件

- **パフォーマンス**: 一括クローズで多数の PTY を終了する場合でも UI スレッドをブロックしない（Zustand の単一 `set` 内で完結させ、PTY 終了は非同期で投げっぱなしにする）。
- **ユーザビリティ**: メニュー項目の表示順は VS Code 慣習に合わせる（右側を閉じる → その他を閉じる → すべて閉じる）。
- **保守性**:
  - コンテンツ用とターミナル用のコンテキストメニューを **意図的に共通化しない**。両者は異なるドメインオブジェクト（ContentTab / TerminalTab）と異なる追加メニュー項目（ターミナル側のリネーム・ピン解除・プロンプトパレット起動など）を持ち、将来の拡張も独立して進む見込みのため。
  - ストアの新規アクションは既存 `closeTab` のセマンティクス（最後の 1 枚は空タブに差し替え／PTY 終了通知）を踏襲する。
- **外観・デザイン**:
  - Radix ContextMenu と既存カラーパレット（`--color-bg-elevated`, `--color-border`, `--color-text-primary`, `--color-text-muted`, `--color-accent`）を使用。
  - 危険操作（クローズ系）は既存の `danger` スタイル（`#ef4444`）を踏襲。

## スコープ

### 対象

- コンテンツタブのコンテキストメニュー新規実装
- ターミナルタブのコンテキストメニュー拡張
- `contentStore` / `terminalStore` への新規アクション追加
- i18n（ja/en）追加
- 上記に対するユニットテスト（ストアレベル）と手動テスト手順書

### 対象外

- ターミナルとコンテンツでコンテキストメニューを共通化するリファクタリング
- キーボードショートカットからの一括クローズ呼び出し（必要なら別 PR）
- 「左側のタブを閉じる」「未保存タブを閉じる」「ピン留めタブ以外を閉じる」などの追加バリエーション
- タブバー右端の「…」オーバーフローメニュー

## 依存関係

- `@radix-ui/react-context-menu` — 既存依存。新規導入なし。
- `lucide-react` — `XCircle`, `ArrowRight` などのアイコンを使用予定（既存依存）。
- 既存ヘルパー `notifyPromptPaletteOfPtyClosed`（`terminalStore.ts:91`） — ターミナル側で各 PTY のクリーンアップに利用。

## 既知の制約

- 「すべてのタブを閉じる」は実質「最後の 1 枚を空タブにリセット」になる（既存 `closeTab` の挙動を踏襲）。完全に 0 枚にはしない。
  - メニューラベル上は「すべて閉じる」と表記し、振る舞いは README/testing.md で説明する。
- 分割ペインで操作した場合、対象は **操作したペインのみ**。もう片方のペインのタブには影響しない（既存 `closeTab` のスコープと同様）。
- ターミナルの一括クローズで PTY 終了は逐次 `close_pty` を invoke する。Rust 側でまとめてクローズする新コマンドの追加はしない（コスト対効果の観点）。
