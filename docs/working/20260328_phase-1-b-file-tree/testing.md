# テスト手順書 - Phase 1-B: ファイルツリー（TreePanel）

## 概要

Phase 1-B（ファイルツリー）のテスト手順を記載します。プロジェクトを開いてファイルツリーが正しく表示・操作できることを手動で確認します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト用プロジェクトフォルダが準備されていること（spec-prompt リポジトリ自体で可）

---

## 手動テスト

### ケース 1: プロジェクトを開くダイアログ

**手順:**

1. アプリを起動する
2. 左ペイン上部の「開く」ボタンをクリックする
3. フォルダ選択ダイアログが表示されることを確認する
4. `spec-prompt` リポジトリのフォルダを選択して開く

**期待結果:**

- ネイティブのフォルダ選択ダイアログが表示される
- フォルダ選択後、左ペインにファイルツリーが表示される
- ヘッダーにプロジェクト名（フォルダ名）が表示される

**確認結果:**

- [x] OK

---

### ケース 2: ファイルツリーの表示・除外確認

**手順:**

1. `spec-prompt` リポジトリをプロジェクトとして開く
2. 表示されるファイル・フォルダ一覧を確認する

**期待結果:**

- ディレクトリが先に表示され、その後ファイルがアルファベット順で表示される
- `node_modules` / `.git` / `.claude` などの隠しファイル・フォルダも含めてすべて表示される
- `docs/` / `src/` / `src-tauri/` などのディレクトリが表示される
- ファイルには `📄` 相当のアイコン、ディレクトリには `📁` 相当のアイコンが表示される

**確認結果:**

- [x] OK

---

### ケース 3: ディレクトリの展開/折りたたみ

**手順:**

1. プロジェクトを開いた状態で、折りたたまれているディレクトリ（例: `src/`）の行をクリックする
2. さらに同じ行をクリックする

**期待結果:**

- 1回目クリック: ディレクトリが展開され、子ファイル・フォルダが表示される
- 矢印アイコンが ChevronRight → ChevronDown に変わる
- 2回目クリック: ディレクトリが折りたたまれ、子が非表示になる
- 矢印アイコンが ChevronDown → ChevronRight に戻る

**確認結果:**

- [x] OK

---

### ケース 4: ファイル選択ハイライト

**手順:**

1. プロジェクトを開いた状態で、ツリー内の任意のファイル（例: `src/App.tsx`）をクリックする
2. 別のファイルをクリックする

**期待結果:**

- クリックしたファイルの行がハイライト表示される（左側のアクセントカラーボーダー + 背景色変化）
- 別のファイルをクリックすると、ハイライトが移動する

**確認結果:**

- [x] OK

---

### ケース 5: ダイアログのキャンセル

**手順:**

1. プロジェクトが開いている状態で「開く」ボタンをクリックする
2. ダイアログが表示されたらキャンセルする

**期待結果:**

- 現在表示されているツリーが変わらない

**確認結果:**

- [x] OK

---

### ケース 6: ローディング表示

**手順:**

1. 「開く」ボタンをクリックしてプロジェクトを選択する
2. ツリーが表示されるまでの間を観察する

**期待結果:**

- ツリー取得中にローディングインジケーターが表示される
- 取得完了後にツリーが表示される

**確認結果:**

- [x] NG（仕様上の制約）

**備考:**

ローカル Rust ファイルシステム読み取りが数ミリ秒で完了するため、React 18 の自動バッチング（Automatic Batching）により `setLoading(true)` と `setLoading(false)` が同一レンダーサイクルにまとめられ、スピナーが画面に描画される前に完了する。コード実装は正しく、ネットワーク越しや非常に深い階層では表示される可能性がある。ローカル高速読み取りでは体感上問題なし。
