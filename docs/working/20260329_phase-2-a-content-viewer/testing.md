# テスト手順書 - Phase 2-A: コンテンツビューア

## 概要

コンテンツビューア（MarkdownPreview / CodeViewer / PlainTextViewer）のテスト手順を記載します。ファイルツリーからファイルを選択して表示を確認します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト用プロジェクトとして `spec-prompt` リポジトリ自体を開いておく（MDファイル・TSファイル・TOMLファイル等が揃っている）

---

## 手動テスト

### ケース 1: ファイル未選択時のプレースホルダー表示

**手順:**

1. アプリを起動する
2. ファイルツリーで何もクリックしない

**期待結果:**

- コンテンツペインに「ファイルを選択してください」とミュートカラーで表示される

**確認結果:**

- [x] OK

---

### ケース 2: マークダウンファイルのプレビュー表示

**手順:**

1. ファイルツリーから `.md` ファイルをクリックする（例: `docs/working/20260329_phase-2-a-content-viewer/requirements.md`）

**期待結果:**

- 見出し（`#`）がスタイル付きで表示される
- 箇条書き（`-`）がリストとして表示される
- テーブルが HTML テーブルとして整形表示される
- コードブロック（` ``` `）にシンタックスハイライトが適用される

**確認結果:**

- [x] OK

---

### ケース 3: Mermaid ダイアグラムのレンダリング

**手順:**

1. Mermaid コードブロックを含む `.md` ファイルをクリックする（例: `docs/steering/03_architecture_specifications.md` 等）
2. 該当 Mermaid ブロックが表示されるまで待つ

**期待結果:**

- Mermaid コードブロックが SVG 図としてレンダリングされる（フローチャートまたはシーケンス図が視覚的に表示される）
- テキストコードがそのまま表示されない

**確認結果:**

- [x] OK

---

### ケース 4: TypeScript ファイルのコードビューア表示

**手順:**

1. ファイルツリーから `.ts` または `.tsx` ファイルをクリックする（例: `src/stores/appStore.ts`）

**期待結果:**

- シンタックスハイライトが適用されてコードが表示される
- 行番号が左端に表示される
- テキストは選択可能だが編集はできない（テキストカーソルが出ない）

**確認結果:**

- [x] OK

---

### ケース 5: Rust ファイルのコードビューア表示

**手順:**

1. ファイルツリーから `.rs` ファイルをクリックする（例: `src-tauri/src/commands/filesystem.rs`）

**期待結果:**

- Rust のシンタックスハイライトが適用される（キーワードが色分けされる）

**確認結果:**

- [x] OK

---

### ケース 6: TOML ファイルのコードビューア表示

**手順:**

1. ファイルツリーから `.toml` ファイルをクリックする（例: `src-tauri/Cargo.toml`）

**期待結果:**

- TOML のシンタックスハイライトが適用される

**確認結果:**

- [x] OK

---

### ケース 7: プレーンテキストファイルの表示

**手順:**

1. ファイルツリーから拡張子のないファイルまたは `.txt` ファイルをクリックする（例: `.gitignore`）

**期待結果:**

- 等幅フォントでテキストがそのまま表示される
- シンタックスハイライトなし

**確認結果:**

- [x] OK

---

### ケース 8: ファイルの切り替え

**手順:**

1. `.md` ファイルをクリックしてプレビューを表示する
2. 別の `.ts` ファイルをクリックする
3. さらに別の `.md` ファイルをクリックする

**期待結果:**

- 毎回クリックしたファイルに対応した表示モードに切り替わる
- 前のファイルの内容が残らない

**確認結果:**

- [x] OK

---

### ケース 9: 同じファイルを再クリック

**手順:**

1. ファイルをクリックしてコンテンツを表示する
2. 同じファイルを再度クリックする

**期待結果:**

- 表示が変化しない（再読み込みは発生するが表示が乱れない）

**確認結果:**

- [x] OK

---

### ケース 10: Split モードでのコンテンツ表示

**手順:**

1. ファイルツリーから `.md` ファイルをクリックする
2. `Ctrl+\` で Split モードに切り替える

**期待結果:**

- 左ペインにマークダウンプレビューが表示された状態で Split モードになる
- 右ペインにターミナルが表示される

**確認結果:**

- [x] OK

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| 大きなファイル（1000行以上）を開く | 表示が重くならず 500ms 以内に表示される | OK |
| 空のファイルを開く | エラーにならず空のビューが表示される | OK |
