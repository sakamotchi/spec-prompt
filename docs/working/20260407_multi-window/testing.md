# テスト手順書 - Phase 3-G: 複数ウィンドウ表示

## 概要

複数ウィンドウ表示機能のテスト手順を記載します。ウィンドウの独立性・プロジェクトパスの引き渡し・UI の統合を手動で確認します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト用に 2 つの異なるプロジェクトフォルダが準備されていること

---

## 手動テスト

### ケース 1: ツールバーから新規ウィンドウを開く

**手順:**

1. アプリを起動する（プロジェクトを開いていても開いていなくてもよい）
2. TreePanel ツールバーの新規ウィンドウボタンをクリックする

**期待結果:**

- 新しい OS ウィンドウが**空の状態**で開く（プロジェクト未選択）
- ツリーパネルに「フォルダを開いてください」が表示される
- タイトルバーに `SpecPrompt` のみ表示される
- 既存ウィンドウに影響がない（タブ・ターミナルがそのまま）

**確認結果:**

- [x] OK

---

### ケース 2: フォルダ右クリックから新規ウィンドウで開く

**手順:**

1. ファイルツリーでフォルダを右クリックする
2. コンテキストメニューから「新規ウィンドウで開く」を選択する

**期待結果:**

- 新規ウィンドウが開く
- 新規ウィンドウで、右クリックしたフォルダがプロジェクトとして開かれている
- ファイルツリーに対象フォルダの内容が表示される

**確認結果:**

- [x] OK

---

### ケース 3: ファイルノードの右クリックメニューを確認する

**手順:**

1. ファイルツリーでファイル（ディレクトリでないもの）を右クリックする

**期待結果:**

- コンテキストメニューに「新規ウィンドウで開く」が表示されない（フォルダのみ対象）

**確認結果:**

- [x] OK

---

### ケース 4: ウィンドウの状態が独立していること

**手順:**

1. ウィンドウ A でプロジェクト A を開き、ファイルを数個タブで開く
2. ウィンドウ B をフォルダ右クリックで起動する
3. ウィンドウ B でファイルを開く
4. ウィンドウ A のタブ状態を確認する

**期待結果:**

- ウィンドウ B でのファイルオープンがウィンドウ A のタブに影響しない
- ウィンドウ A・B でそれぞれ独立したターミナルセッションが動作する

**確認結果:**

- [x] OK

---

### ケース 5: ウィンドウタイトルにプロジェクト名が表示される

**手順:**

1. アプリを起動してプロジェクトフォルダを開く
2. OS のウィンドウタイトルバーを確認する

**期待結果:**

- タイトルバーに `SpecPrompt — {プロジェクトフォルダ名}` が表示される

**確認結果:**

- [x] OK

---

### ケース 6: プロジェクト未選択時のウィンドウタイトル

**手順:**

1. `localStorage` をクリアしてアプリを再起動する（またはプロジェクトを開かない状態にする）

**期待結果:**

- タイトルバーに `SpecPrompt` のみ表示される（プロジェクト名なし）

**確認結果:**

- [x] OK

---

### ケース 7: プロジェクト切り替え時のタイトル更新

**手順:**

1. プロジェクト A を開く（タイトル: `SpecPrompt — ProjectA`）
2. 最近開いたプロジェクトメニューからプロジェクト B に切り替える

**期待結果:**

- タイトルバーが `SpecPrompt — ProjectB` に更新される

**確認結果:**

- [x] OK

---

### ケース 8: 設定（テーマ・フォント・言語）がウィンドウ間で共有される

**手順:**

1. ウィンドウ A を起動する
2. ウィンドウ A の設定でテーマを「ライト」に変更する
3. ツールバーの新規ウィンドウボタンでウィンドウ B を起動する

**期待結果:**

- ウィンドウ B が「ライト」テーマで起動する（localStorage 経由で共有）
- ウィンドウ A はリアルタイムには変わらない（意図的な動作）

**確認結果:**

- [x] OK

---

### ケース 9: i18n — 新規ウィンドウ関連文字列の翻訳確認

**手順:**

1. 設定モーダルで英語に切り替える
2. フォルダを右クリックしてコンテキストメニューを開く
3. ツールバーの新規ウィンドウボタンにホバーして tooltip を確認する

**期待結果:**

- コンテキストメニューに「Open in New Window」が表示される
- ツールバーボタンの tooltip に「Open in New Window」が表示される

**確認結果:**

- [x] OK

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| 存在しないパスを URL クエリパラメータで渡した場合 | エラーダイアログまたは空の状態で起動（クラッシュしない） | |
| 多数のウィンドウを連続起動した場合 | 各ウィンドウが独立して正常動作する | |
| ウィンドウを閉じても他ウィンドウに影響しない | 正常に動作継続 | |
