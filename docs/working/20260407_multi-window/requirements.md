# 要件定義書 - Phase 3-G: 複数ウィンドウ表示

## 概要

SpecPrompt に OS レベルの複数ウィンドウ機能を追加する。各ウィンドウは独立した React インスタンスと Zustand ストアを持ち、異なるプロジェクトフォルダを同時に開ける。

## 背景・目的

現在の SpecPrompt は単一ウィンドウのみをサポートしており、複数プロジェクトを同時に参照する際は切り替えが必要。複数ウィンドウ機能により、Claude Code の作業中に仕様書と実装コードを別ウィンドウで並べて確認するなど、マルチタスクな作業フローをサポートする。

## 要件一覧

### 機能要件

#### F-1: 新規ウィンドウの起動

- **説明**: Tauri v2 の `WebviewWindow` API を使用して、新規アプリウィンドウを起動する
- **受け入れ条件**:
  - [ ] 新規ウィンドウが独立した OS ウィンドウとして開く
  - [ ] 既存ウィンドウに影響を与えない
  - [ ] 新規ウィンドウが `npx tauri dev` 環境でも正常に動作する

#### F-2: プロジェクトパスの引き渡し

- **説明**: URL クエリパラメータ（`?project=/path/to/project`）を使って、新規ウィンドウの初期プロジェクトを指定する
- **受け入れ条件**:
  - [ ] 右クリックメニューから「新規ウィンドウで開く」を選択すると、そのフォルダパスが新規ウィンドウに渡される
  - [ ] 新規ウィンドウ起動後、指定フォルダが自動的に開かれた状態になる
  - [ ] パスを指定しない場合は空の状態で起動する

#### F-3: ウィンドウごとの状態分離

- **説明**: 各ウィンドウは独立した React インスタンスを持ち、Zustand ストアの状態が共有されない
- **受け入れ条件**:
  - [ ] ウィンドウ A でファイルを開いても、ウィンドウ B のタブに影響しない
  - [ ] ウィンドウ A のターミナルセッションがウィンドウ B に干渉しない
  - [ ] テーマ・フォント設定は `settingsStore`（localStorage 経由）で共有される（意図的な仕様）

#### F-4: 「新規ウィンドウで開く」UI

- **説明**: ファイルツリーのコンテキストメニューと TreePanel ツールバーに新規ウィンドウ起動のエントリを追加する
- **受け入れ条件**:
  - [ ] フォルダを右クリック → コンテキストメニューに「新規ウィンドウで開く」が表示され、そのフォルダを開いた状態で起動する
  - [ ] TreePanel ツールバーに新規ウィンドウボタンが追加される（**空のウィンドウ**として起動）
  - [ ] ツールバーボタンと右クリックメニューで起動挙動が異なることが tooltip から分かる

**設計上の判断**: ツールバーボタンは空ウィンドウ起動とする。分割表示（Ctrl+\）が「同一プロジェクトの複数ファイルを並べる」ニーズを担うため、新規ウィンドウの主なユースケースは「別プロジェクトを同時に開く」となる。

#### F-5: ウィンドウタイトルの更新

- **説明**: アクティブプロジェクトのフォルダ名をウィンドウタイトルバーに表示する
- **受け入れ条件**:
  - [ ] プロジェクトを開いたとき、タイトルバーが `SpecPrompt — {プロジェクト名}` に更新される
  - [ ] プロジェクトを切り替えたとき、タイトルバーが更新される
  - [ ] プロジェクト未選択時は `SpecPrompt` のみ表示される

### 非機能要件

- **パフォーマンス**: 新規ウィンドウ起動が 3 秒以内に完了する
- **ユーザビリティ**: 既存の右クリックメニューとツールバーに自然に統合される
- **保守性**: Tauri WebviewWindow API の変更に追従しやすい薄いラッパーとして実装する
- **外観・デザイン**: コンテキストメニューは既存の `TreeContextMenu` の `MenuItem` 構造に従う。ツールバーボタンは既存のアイコンボタンスタイルを踏襲する。CSS カスタムプロパティ（`--color-*`）を使用する

## スコープ

### 対象

- Tauri `WebviewWindow` による新規ウィンドウ起動
- URL クエリパラメータによるプロジェクトパス引き渡し
- コンテキストメニュー「新規ウィンドウで開く」（フォルダ対象）
- TreePanel ツールバーへの新規ウィンドウボタン追加
- ウィンドウタイトルバーへのプロジェクト名表示

### 対象外

- ウィンドウ間のドラッグ＆ドロップによるファイル移動
- ウィンドウ間のタブ移動
- ウィンドウ位置・サイズの永続化
- ウィンドウ一覧の管理 UI

## 実装対象ファイル（予定）

- `src/lib/tauriApi.ts` — `openNewWindow(projectPath?: string)` 関数を追加
- `src/components/TreePanel/ContextMenu.tsx` — 「新規ウィンドウで開く」メニュー項目を追加
- `src/components/TreePanel/TreePanel.tsx` — ツールバーに新規ウィンドウボタンを追加
- `src/components/TreePanel/TreeNode.tsx` — ContextMenu への `onOpenNewWindow` prop 追加
- `src/main.tsx` — 起動時に URL クエリパラメータを読み取り初期プロジェクトを設定
- `src/stores/appStore.ts` — 起動時プロジェクト自動オープンのロジック追加
- `src/i18n/locales/ja.json` / `en.json` — 新規ウィンドウ関連の翻訳キー追加

## 依存関係

- Phase 1（ファイルツリー・プロジェクト管理）完了済み
- Phase 3-F（i18n）完了済み — 新規翻訳キーを追加する形で拡張

## 既知の制約

- Tauri v2 の `WebviewWindow` はフロントエンド JS から呼び出せる（Rust コマンド不要）
- 各ウィンドウは独立した WebView プロセスとして動作するため、JS メモリは共有されない
- ウィンドウ間でリアルタイム同期は行わない（localStorage の設定値は共有）
