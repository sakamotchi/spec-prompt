# 要件定義書 - Phase 2-D: パス入力支援

## 概要

プロジェクトツリーのファイル・フォルダパスをターミナルのカーソル位置に素早く挿入する機能を実装する。
マウス操作（Ctrl+クリック・右クリックメニュー）、キーボード操作（Ctrl+P パレット）の両方をサポートし、AI CLI ツールへのパス指定を効率化する。

## 背景・目的

Claude Code などの AI CLI ツールで作業する際、ファイルパスを手入力するのは非効率で誤りが起きやすい。
プロジェクトツリーから直接ターミナルへパスを挿入できることで、作業速度と正確性が向上する。

## 要件一覧

### 機能要件

#### F-1: Ctrl+クリックでパス挿入

- **説明**: ツリーのノードを Ctrl+クリック（macOS では Cmd+クリック）するとアクティブターミナルのカーソル位置にパスを挿入する
- **受け入れ条件**:
  - [ ] ファイルノードを Ctrl+クリックするとパスが挿入される
  - [ ] フォルダノードを Ctrl+クリックするとパスが挿入される
  - [ ] 通常クリック（コンテンツ表示）の動作を妨げない
  - [ ] アクティブなターミナルタブ・ペインに挿入される
  - [ ] ターミナルが開いていない場合は何も起きない（エラーなし）

#### F-2: 右クリックメニュー「パスをターミナルに挿入」

- **説明**: ツリーノードの右クリックコンテキストメニューから「パスをターミナルに挿入」を選択してパスを挿入する
- **受け入れ条件**:
  - [ ] 右クリックでコンテキストメニューが表示される
  - [ ] 「パスをターミナルに挿入」メニュー項目が存在する
  - [ ] 選択するとアクティブターミナルにパスが挿入される
  - [ ] メニューはクリック外で閉じる（Esc キーでも閉じる）

#### F-3: パス検索パレット（PathPalette）

- **説明**: Ctrl+P（macOS では Cmd+P）でオーバーレイ表示し、インクリメンタルサーチでファイルを検索・選択してパスを挿入する
- **受け入れ条件**:
  - [ ] Ctrl+P / Cmd+P でパレットが表示される
  - [ ] テキスト入力でプロジェクト内ファイルを fuzzy 検索できる
  - [ ] 結果一覧から Enter またはクリックでパスを挿入し、パレットが閉じる
  - [ ] Esc でパレットを閉じる（パスは挿入しない）
  - [ ] 表示件数は上位100件に制限する
  - [ ] 検索結果がない場合は「一致なし」メッセージを表示する

#### F-4: 複数ファイル選択・一括挿入

- **説明**: ツリーで複数ファイルを選択し、一括でパスを挿入する
- **受け入れ条件**:
  - [ ] Shift+クリックで範囲選択できる
  - [ ] Ctrl+クリック（macOS: Cmd+クリック）で個別選択を追加できる
  - [ ] 右クリックメニューから複数選択ファイルのパスを一括挿入できる
  - [ ] 挿入順はツリーの表示順（上から下）とする
  - [ ] 挿入フォーマットはスペース区切りとする（例: `path/a path/b path/c`）

#### F-5: パス形式切り替え

- **説明**: 挿入するパスの形式（相対パス / 絶対パス）を設定で切り替えられる
- **受け入れ条件**:
  - [ ] appStore にパス形式設定（`pathFormat: 'relative' | 'absolute'`）を追加
  - [ ] 相対パスはプロジェクトルートからの相対パスとする
  - [ ] 設定は永続化される（アプリ再起動後も保持）
  - [ ] 設定画面（Phase 3-B）と連携（本フェーズでは appStore の追加のみ、UI は 3-B で実装）

### 非機能要件

- **パフォーマンス**: PathPalette の検索はリスト生成を含めて 100ms 以内に初期表示する
- **ユーザビリティ**: パレットはアプリ中央にオーバーレイ表示し、背景を半透明で暗くする
- **外観・デザイン**: CSS カスタムプロパティ（`--color-bg-elevated`, `--color-accent` 等）を使用。Radix UI `@radix-ui/react-dialog` でモーダル実装

## スコープ

### 対象

- TreeNode へのキーボードイベント追加（Ctrl/Cmd+クリック、右クリック）
- コンテキストメニューコンポーネント（ContextMenu）
- PathPalette コンポーネント（検索・選択UI）
- appStore へのパス形式設定追加
- ターミナルへのパス挿入ロジック（`write_pty` 経由）

### 対象外

- パス形式の設定画面 UI（Phase 3-B で実装）
- Ctrl+P 以外のショートカット整備（Phase 3-C で実装）

## 依存関係

- Phase 1-C（ターミナル実装）: `write_pty` コマンドが必要
- Phase 2-B（ファイル監視）: ファイルリスト取得に `fileTree` を使用
- `@radix-ui/react-context-menu`: コンテキストメニュー UI
- `@radix-ui/react-dialog`: PathPalette モーダル

## 既知の制約

- ターミナルがアクティブでない（コンテンツモード表示中）場合でもパス挿入は動作する（`writePty` を呼ぶ）
- fuzzy 検索はファイルツリーの読み込み済みノードのみを対象とする（未展開ノードは対象外）
