# テスト手順書 - notification-tab-name（P1）

## 概要

P1（通知タイトルへのタブ識別子差し込み）の手動テストおよび自動テスト手順。通知発火のパスは OS 依存のため可能な限り手動確認を中心に行う。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- macOS のシステム設定で SpecPrompt の通知が許可されていること
- アプリ内設定で「Claude Code 通知」が ON になっていること（既定 ON）
- シェルから `claude` コマンドが実行できること（Claude Code CLI がインストール済み、または OSC 9 を模擬する `printf` で代用可）

---

## 手動テスト

### ケース 1: 2 タブで通知を発火し、タイトルにタブ名が含まれる

**手順:**

1. アプリを起動する（`npx tauri dev`）
2. 統合ターミナルで新規タブを 1 つ追加し、タブ 1 と タブ 2（`Terminal 1` / `Terminal 2`）が並んだ状態にする
3. アプリを Command+H などでバックグラウンドにする
4. タブ 1 のターミナルで以下を実行して OSC 9 を送出する（`claude` を直接使わず模擬）:
   ```bash
   printf '\033]9;承認が必要です (tab1)\007'
   ```
5. macOS の通知センターに現れる通知を確認する
6. アプリを再度フォアグラウンドに戻し、タブ 2 を選択して再びバックグラウンドにする
7. タブ 2 のターミナルで以下を実行:
   ```bash
   printf '\033]9;作業が完了しました (tab2)\007'
   ```
8. 通知を確認する

**期待結果:**

- 手順 5: タイトル `Claude Code — Terminal 1`、本文 `承認が必要です (tab1)` の通知が表示される
- 手順 8: タイトル `Claude Code — Terminal 2`、本文 `作業が完了しました (tab2)` の通知が表示される

**確認結果:**

- [ ] OK / NG

### ケース 3: 通知設定 OFF のとき Claude Code 由来の通知が発火しない（既存挙動の非後退）

**注意:** `notification_enabled` フラグは **PTY スポーン時にのみ反映** される（`TERM_PROGRAM` 環境変数のゲートとして機能し、Claude Code 自身の OSC 9 出力を抑止する）。
そのため設定トグル後はアプリ再起動ではなく **新規タブを開き直す** 必要がある。また、`printf` 等で手動に OSC 9 を出力する方法では Claude Code のゲートを経由しないため本ケースの検証には使えない（検出側は常時有効のため通知が発火する。既存 v0.1.11 からの仕様）。本ケースは実際の `claude` コマンド経由で検証する。

**手順:**

1. アプリ設定画面で「Claude Code 通知」を OFF にする
2. **新規タブを開く**（既存タブは `notification_enabled=true` のまま動き続ける）
3. その新規タブで `claude` を起動し、ファイル編集など承認要求が発生する操作を実行する
4. アプリをバックグラウンドに移す

**期待結果:**

- 承認要求が発生しても OS 通知は表示されない
- 設定を再度 ON に戻し、新規タブを開いて同じ操作を行うと通知が復活する

**確認結果:**

- [ ] OK / NG

### ケース 4: タブを閉じるとキャッシュが掃除される

**手順:**

1. タブを 1 つ起動（例: `Terminal 1`）
2. タブを閉じる
3. 新たに別のタブを起動（同じ `pty_id` が再利用されうる点に注意）
4. バックグラウンドにして `printf '\033]9;test\007'` を実行

**期待結果:**

- 通知タイトルは新しく起動したタブの名前（例: `Terminal 2`）になる。閉じた古いタブ名が漏れてこない

**確認結果:**

- [ ] OK / NG

---

### ケース 5: PTY スポーン直後の高速通知でもクラッシュしない

**手順:**

1. タブを新規起動し、プロンプトが出た直後（1 秒以内）に `printf '\033]9;fast\007'` を実行
2. 通知の有無にかかわらず、アプリが落ちないこと・後続操作が正常であること

**期待結果:**

- アプリがクラッシュしない
- 通知タイトルはタブ名付き、またはフォールバック `SpecPrompt / Claude Code` のいずれか（競合により前者が間に合わない可能性はあるが、どちらでも可）

**確認結果:**

- [ ] OK / NG
