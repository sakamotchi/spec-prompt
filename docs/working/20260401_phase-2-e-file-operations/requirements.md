# 要件定義書 - Phase 2-E: ファイル操作

## 概要

プロジェクトツリーから直接ファイル・フォルダの作成、リネーム、削除を行える機能を実装する。
右クリックコンテキストメニューとインライン編集 UI で操作を完結させ、ターミナルに切り替えることなくファイル管理ができる状態を目指す。

## 背景・目的

Phase 2-D でコンテキストメニューの基盤（`TreeContextMenu`）が完成した。次のステップとして、パス挿入以外のファイル操作メニュー項目を実装する。仕様駆動開発では新規ファイル・フォルダの作成が頻繁に発生するため、ツリーから直接操作できることで作業効率が向上する。

## 要件一覧

### 機能要件

#### F-1: ファイル新規作成

- **説明**: コンテキストメニューから任意の場所に空ファイルを作成する
- **受け入れ条件**:
  - [ ] ファイルまたはフォルダを右クリックすると「新規ファイル」メニューが表示される
  - [ ] メニュー選択後、ツリー内にインライン入力フィールドが表示される
  - [ ] Enter を押すとファイルが作成されツリーが更新される
  - [ ] Esc を押すとキャンセルされる
  - [ ] 作成先はフォルダを右クリックした場合そのフォルダ直下、ファイルを右クリックした場合はそのファイルと同じディレクトリ直下
  - [ ] 同名ファイルが存在する場合はエラーを表示する

#### F-2: フォルダ新規作成

- **説明**: コンテキストメニューから任意の場所にフォルダを作成する
- **受け入れ条件**:
  - [ ] 「新規フォルダ」メニュー選択後、インライン入力フィールドが表示される
  - [ ] Enter でフォルダが作成されツリーが更新される
  - [ ] Esc でキャンセルされる
  - [ ] 同名フォルダが存在する場合はエラーを表示する

#### F-3: リネーム

- **説明**: ファイル・フォルダの名前をインライン編集で変更する
- **受け入れ条件**:
  - [ ] コンテキストメニュー「リネーム」、または F2 キーでインライン編集に入る
  - [ ] 現在のファイル名が入力フィールドに表示された状態で編集開始される
  - [ ] Enter で確定、Esc でキャンセル
  - [ ] リネーム後にツリーが更新される
  - [ ] コンテンツビューアで開いていたファイルをリネームした場合、タブのタイトルが更新される

#### F-4: 削除

- **説明**: ファイル・フォルダを確認ダイアログ後に削除する
- **受け入れ条件**:
  - [ ] コンテキストメニュー「削除」を選択すると確認ダイアログが表示される
  - [ ] ダイアログで「削除」を選択するとファイル/フォルダが削除される
  - [ ] ダイアログで「キャンセル」を選択すると削除されない
  - [ ] 削除後にツリーが更新される
  - [ ] 開いていたコンテンツタブがあれば閉じる
  - [ ] 空でないフォルダの削除は確認ダイアログで警告文を変える（「X件のファイルを含む」）

#### F-5: 外部エディタで開く

- **説明**: ファイルを OS のデフォルトアプリ（VS Code、Cursor 等）で開く。SpecPrompt はビューア＋ターミナルに集中し、ファイル内容の編集は外部エディタまたは AI CLI に委ねる方針に基づく
- **受け入れ条件**:
  - [ ] コンテキストメニューに「外部エディタで開く」が表示される
  - [ ] 選択するとそのファイルの OS デフォルトアプリが起動する
  - [ ] フォルダを右クリックした場合はフォルダをデフォルトアプリ（Finder 等）で開く
  - [ ] SpecPrompt 自体はそのまま動作し続ける

### 非機能要件

- **パフォーマンス**: ファイル操作後のツリー更新は操作完了後 200ms 以内に反映
- **ユーザビリティ**: インライン入力は自動フォーカス・全選択状態で開始する
- **保守性**: Rust コマンドはすでにある `filesystem.rs` に追加する
- **外観・デザイン**: インライン入力フィールドは既存の CSS カスタムプロパティを使用し、他のツリーノードと高さを揃える（`h-7`）

## スコープ

### 対象

- ファイル新規作成（空ファイル）
- フォルダ新規作成
- ファイル・フォルダのリネーム
- ファイルの削除
- 空フォルダの削除
- 空でないフォルダの削除（警告付き）
- 外部エディタで開く（OS デフォルトアプリ起動）

### 対象外

- ファイル内容のインライン編集（編集は外部エディタまたは AI CLI に委ねる）
- ファイルの移動（ドラッグ＆ドロップによる移動）
- ファイルのコピー・ペースト
- ゴミ箱への移動（即時削除）
- 複数ファイルの一括削除
- ファイルのパーミッション変更
- 外部エディタの種類の指定（Phase 3-B 設定画面で対応予定）

## 既知の制約

- ファイル監視（Phase 2-B）がすでに動いているため、ファイル変更イベントは自動で発火する。ただしツリー再読み込みのトリガーは `file-changed` イベントではなくコマンド成功後に明示的に行う（ツリーはコンテンツビューアと別管理）
- リネーム時に開いているコンテンツタブとの同期は `contentStore` を経由する
