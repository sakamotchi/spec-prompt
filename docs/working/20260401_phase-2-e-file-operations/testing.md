# テスト手順書 - Phase 2-E: ファイル操作

## 概要

ファイル新規作成・フォルダ新規作成・リネーム・削除の手動テスト手順を記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト用プロジェクトとして `spec-prompt` リポジトリを開いておく
- ファイルツリーが表示されていること

---

## 手動テスト

### ケース 1: ファイル新規作成（フォルダを右クリック）

**手順:**

1. ツリーで `src` フォルダを右クリックする
2. コンテキストメニューから「新規ファイル」を選択する
3. インライン入力フィールドに `test_new_file.md` と入力する
4. Enter を押す

**期待結果:**

- `src/test_new_file.md` が作成され、ツリーに表示される
- インライン入力が閉じる

**確認結果:**

- [x] OK / NG

**後処理**: 作成したファイルを削除しておく

---

### ケース 2: ファイル新規作成（ファイルを右クリック）

**手順:**

1. ツリーで任意のファイル（例: `src/main.tsx`）を右クリックする
2. 「新規ファイル」を選択し、`sibling_file.ts` と入力して Enter

**期待結果:**

- `src/sibling_file.ts` が作成される（右クリックしたファイルと同じディレクトリ）

**確認結果:**

- [x] OK / NG

**後処理**: 作成したファイルを削除しておく

---

### ケース 3: ファイル新規作成をキャンセル（Esc）

**手順:**

1. フォルダを右クリックして「新規ファイル」を選択する
2. 何か入力した後、**Esc** を押す

**期待結果:**

- インライン入力が閉じる
- ファイルは作成されない

**確認結果:**

- [x] OK / NG

---

### ケース 4: 同名ファイルの作成エラー

**手順:**

1. `src` フォルダを右クリックして「新規ファイル」を選択する
2. 既存のファイル名（例: `main.tsx`）を入力して Enter

**期待結果:**

- エラーが表示される（ファイルは上書きされない）

**確認結果:**

- [x] OK / NG

---

### ケース 5: フォルダ新規作成

**手順:**

1. ツリーで `src` フォルダを右クリックする
2. 「新規フォルダ」を選択する
3. `test_folder` と入力して Enter

**期待結果:**

- `src/test_folder/` が作成され、ツリーに表示される

**確認結果:**

- [x] OK / NG

**後処理**: 作成したフォルダを削除しておく

---

### ケース 6: リネーム（コンテキストメニュー）

**手順:**

1. 前のケースで作成した `src/test_folder` を右クリックする
2. 「リネーム」を選択する
3. 入力フィールドに現在の名前（`test_folder`）が表示されていることを確認する
4. `renamed_folder` に変更して Enter

**期待結果:**

- `src/renamed_folder/` にリネームされ、ツリーが更新される

**確認結果:**

- [x] OK / NG

**後処理**: フォルダを削除しておく

---

### ケース 7: リネーム（F2 キー）

**手順:**

1. ツリーで任意のファイルをクリックして選択状態にする
2. **F2** キーを押す
3. 新しい名前を入力して Enter

**期待結果:**

- インライン入力が開き、リネームが実行される

**確認結果:**

- [x] OK / NG

---

### ケース 8: リネームをキャンセル（Esc）

**手順:**

1. ファイルを右クリックして「リネーム」を選択する
2. 名前を変更した後、**Esc** を押す

**期待結果:**

- リネームがキャンセルされ、元の名前のまま

**確認結果:**

- [x] OK / NG

---

### ケース 9: 開いているタブのファイルをリネーム

**手順:**

1. `README.md` をクリックしてコンテンツタブで開く
2. ツリーで `README.md` を右クリックして「リネーム」を選択する
3. `README_renamed.md` に変更して Enter

**期待結果:**

- ファイルがリネームされ、コンテンツタブのタイトルが `README_renamed.md` に更新される

**確認結果:**

- [x] OK / NG

**後処理**: `README_renamed.md` を `README.md` に戻す

---

### ケース 10: ファイルを削除

**手順:**

1. テスト用ファイルを作成しておく（ケース 1 を参照）
2. 作成したファイルを右クリックして「削除」を選択する
3. 確認ダイアログが表示されることを確認する
4. 「削除」ボタンをクリックする

**期待結果:**

- ファイルが削除され、ツリーから消える

**確認結果:**

- [x] OK / NG

---

### ケース 11: 削除をキャンセル

**手順:**

1. 任意のファイルを右クリックして「削除」を選択する
2. 確認ダイアログで「キャンセル」をクリックする

**期待結果:**

- ファイルは削除されない

**確認結果:**

- [x] OK / NG

---

### ケース 12: 開いているタブのファイルを削除

**手順:**

1. 任意のファイルをコンテンツタブで開く
2. そのファイルをツリーで右クリックして「削除」→「削除」を選択

**期待結果:**

- ファイルが削除される
- コンテンツタブが閉じる（または空のタブになる）

**確認結果:**

- [x] OK / NG

---

### ケース 13: 空でないフォルダを削除

**手順:**

1. ファイルを含むフォルダを右クリックして「削除」を選択する
2. 確認ダイアログに件数の警告が表示されることを確認する
3. 「削除」をクリックする

**期待結果:**

- フォルダとその中身が再帰的に削除される
- ツリーから消える

**確認結果:**

- [x] OK / NG

---

**手順:**

## 自動テスト

### フロントエンドテスト（Vitest）

```bash
npm run test
```

### Lint

```bash
npm run lint
```

### Rust テスト

```bash
cd src-tauri && cargo test
```

### Rust 型チェック

```bash
cd src-tauri && cargo check
```

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| 空文字列で Enter | 作成・リネームされない（無視） | OK |
| スペースのみのファイル名 | 作成・リネームされない | OK |
| 既存名と同じ名前でリネーム | エラーなし or 変化なし | OK |
| 読み取り専用ファイルの削除 | エラーメッセージを表示 | OK |
