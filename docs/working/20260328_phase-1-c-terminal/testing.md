# テスト手順書 - Phase 1-C: 統合ターミナル（TerminalPanel）

## 概要

Phase 1-C（統合ターミナル）のテスト手順を記載します。ターミナルタブの追加・切り替え・クローズと、PTY セッションの継続動作を手動で確認します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- アプリの「ターミナル」タブを選択した状態であること

---

## 手動テスト

### ケース 1: 初期起動

**手順:**

1. アプリを起動する
2. 右ペインの「ターミナル」タブをクリックする

**期待結果:**

- ターミナルタブが 1 つ（「Terminal 1」）表示される
- `/bin/zsh` が起動してシェルプロンプトが表示される
- タブバーに「+」ボタンがある

**確認結果:**

- [ ] OK / NG

---

### ケース 2: タブの追加

**手順:**

1. 「ターミナル」タブを表示する
2. タブバーの「+」ボタンをクリックする
3. もう 1 回「+」ボタンをクリックする

**期待結果:**

- 「Terminal 2」タブが追加され、アクティブになる
- 新しいシェルプロンプトが表示される
- 「Terminal 3」タブがさらに追加される
- タブ名が連番になっている

**確認結果:**

- [ ] OK / NG

---

### ケース 3: タブの切り替えと PTY 継続

**手順:**

1. 「Terminal 1」タブで `echo "hello"` を実行する
2. 「+」でタブを追加し「Terminal 2」で `echo "world"` を実行する
3. 「Terminal 1」タブをクリックして戻る
4. 「Terminal 2」タブをクリックして戻る

**期待結果:**

- 「Terminal 1」に戻った時、`echo "hello"` の出力が残っている
- 「Terminal 2」に戻った時、`echo "world"` の出力が残っている
- 各ターミナルで引き続きコマンドが実行できる

**確認結果:**

- [ ] OK / NG

---

### ケース 4: タブのクローズ

**手順:**

1. タブを 2 つ以上開く
2. 「Terminal 2」タブの「×」ボタンをクリックする

**期待結果:**

- 「Terminal 2」タブが閉じる
- アクティブタブが「Terminal 1」（または残った別のタブ）に切り替わる
- 残ったターミナルは正常に動作している

**確認結果:**

- [ ] OK / NG

---

### ケース 5: 最後の 1 タブは閉じられない

**手順:**

1. タブが 1 つだけの状態にする
2. そのタブの「×」ボタンをクリックする

**期待結果:**

- タブが閉じられない（ボタンが非表示 or 無効化されている）
- 引き続き 1 タブが表示されている

**確認結果:**

- [ ] OK / NG

---

### ケース 6: ペインリサイズ時のターミナル追従

**手順:**

1. ターミナルを表示した状態で、左右ペインのセパレーターをドラッグしてペイン幅を変更する

**期待結果:**

- ターミナルの表示幅がペイン幅に合わせて自動的にリサイズされる
- 文字が折り返し位置を更新する

**確認結果:**

- [ ] OK / NG

---

### ケース 7: タブ切り替え後のターミナルサイズ

**手順:**

1. タブを 2 つ開く
2. 「Terminal 1」でコマンドを実行して出力を確認する
3. 「Terminal 2」に切り替える
4. 「Terminal 1」に戻る

**期待結果:**

- 戻った時にターミナルの表示サイズが正しく（文字崩れなく）表示される

**確認結果:**

- [ ] OK / NG

## 自動テスト

```bash
npm run test
```

- `terminalStore`: タブの追加・クローズ・アクティブ切り替えが正しく動作すること

```bash
npm run lint
npm run build
```

```bash
cd src-tauri && cargo check
```
