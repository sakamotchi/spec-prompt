# テスト手順書 - prompt-palette-history-template-p2-history

## 概要

Phase 2（履歴機能）のテスト手順。**UI を直接操作して確認する手動テスト**が主軸で、ロジック単体は自動テストでカバーする。

## 前提条件

- `npm install` 済み
- `npx tauri dev` でアプリが起動できる状態
- 既存のプロンプトパレット機能（F4 起動、送信、IME、挿入フラッシュ）がグリーンであること（Phase 1 の回帰ケースを先に確認）

## 手動テスト

### ケース 1: 送信時に履歴へ記録される（T2-1）

**手順:**

1. `npx tauri dev` でアプリ起動、ターミナルタブを 1 つ開く
2. `⌘⇧P` でプロンプトパレット起動
3. textarea に `echo phase2-first` を入力
4. `⌘Enter` / `Ctrl+Enter` で送信
5. 再度 `⌘⇧P` でパレット起動
6. ヘッダの履歴アイコンをクリック、あるいは `⌘H` でドロップダウンを開く

**期待結果:**

- ドロップダウン最上行に `echo phase2-first` が表示される
- ターミナルには既存通り送信・実行されている

**確認結果:**

- [x] OK

---

### ケース 2: 送信失敗時は履歴に残らない（T2-1）

**手順:**

1. ターミナルタブを作成後、PTY プロセスを外部から `kill` するなどして無効化（代替: DevTools の Network/invoke モックで `write_pty` を拒否）
2. パレット起動 → `echo should-not-record` を入力 → `⌘Enter`
3. エラートースト（`promptPalette.error.sendFailed`）が表示されることを確認
4. パレットを再度開き履歴ドロップダウンを確認

**期待結果:**

- 履歴ドロップダウンに `echo should-not-record` が含まれない
- textarea 本文は保持されたまま（再送可能）

**確認結果:**

- [x] OK

---

### ケース 3: `↑` で直近履歴が流し込まれる（T2-2/T2-3）

**手順:**

1. ケース 1 で履歴に 1 件以上ある状態にする
2. パレットを開く（textarea は空の状態）
3. textarea にフォーカスがある状態で `↑` を 1 回押す

**期待結果:**

- textarea に最新履歴（`echo phase2-first` など）が流し込まれる
- カーソルは textarea の末尾
- 続けて `⌘Enter` で送信できる

**確認結果:**

- [x] OK

---

### ケース 4: 連続 `↑` / `↓` で過去・現在を巡回（T2-2/T2-3）

**手順:**

1. 履歴を 3 件以上積んだ状態にする（例: `a`, `b`, `c` を順に送信）
2. パレットを開く（空の状態）
3. `↑` を連続で押す
4. `↓` を連続で押す

**期待結果:**

- 1 回目 `↑`: 最新 `c`
- 2 回目 `↑`: `b`
- 3 回目 `↑`: `a`
- 4 回目 `↑`: `a`（上限でクランプ、変化なし）
- 次に `↓`: `b`
- 次の `↓`: `c`
- さらに `↓`: 空（巡回解除）

**確認結果:**

- [x] OK
