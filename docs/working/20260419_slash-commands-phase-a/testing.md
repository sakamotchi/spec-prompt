# テスト手順書 - Claude Code スラッシュコマンドサジェスト統合 Phase A

## 概要

Phase A（組み込みコマンド + バンドル Skill 静的リスト）のテスト手順書。可能な限り手動操作で確認する。自動テストは合成関数・UI 描画に関わる部分に限定する。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト対象のプロジェクトフォルダが準備されていること
- **既存**のユーザー定義プロンプトテンプレートが 1 件以上存在するとベター（衝突確認のため）。なければテスト前に `⌘T` → 新規作成で `review-pr` / body `/review pull/{{number}}` を追加する

## 手動テスト

### ケース 1: Claude Code セクションが表示される

**手順:**

1. アプリを起動する
2. ターミナルタブがアクティブな状態で `Cmd+Shift+P`（mac）/ `Ctrl+Shift+P`（Win/Linux）でパレットを開く
3. textarea に `/` を 1 文字だけ入力する

**期待結果:**

- `Claude Code` セクションに `resume` / `clear` / `model` など 20 件以上の組み込みコマンド + バンドル Skill (`debug` / `simplify` 等) が表示される
- 各行の左端に `CMD` バッジが表示される
- 各行に英語の `description` が表示される

**確認結果:**

- [x] OK

---

### ケース 2: テンプレートと同時表示

**手順:**

1. ケース 1 の続きで、`/` だけの状態を維持
2. セクションの並び順を確認

**期待結果:**

- `Claude Code` セクションの下に `Templates` セクションがある
- `Templates` セクションにユーザー定義テンプレ（`review-pr` 等）が表示される
- 各テンプレ行の左端に `TPL` バッジが表示される

**確認結果:**

- [x] OK

---

### ケース 3: fuzzy 検索で横断マッチ

**手順:**

1. パレット開いた状態で textarea に `/rev` と入力

**期待結果:**

- `Claude Code` セクションに `review` / `rewind` が絞り込まれる
- `Templates` セクションに `review-pr`（既存のもの）が絞り込まれる
- 他の無関係コマンド（`clear` 等）は表示されない

**確認結果:**

- [x] OK

---

### ケース 4: 空セクション非表示

**手順:**

1. 現在登録されているテンプレに **マッチしない** クエリを入力（例: `/xyzzy` など存在しない名前）

**期待結果:**

- `Templates` セクションが非表示になる（見出しも出ない）
- `Claude Code` セクションも候補 0 の場合は非表示
- 全セクションが空になる場合は `SlashSuggest` 全体が非表示

**確認結果:**

- [x] OK

- 送信されない・パレットは開いたまま

**確認結果:**

- [x] OK

### ケース 5-C: 候補多数時のスクロール追従

**手順:**

1. `/` のみ入力して候補を複数セクションで展開

**期待結果:**

- リストは `max-height: 40vh` 内に収まり、親ウィンドウ下端で切れない
- `activeIndex` 移動に合わせて該当行が可視範囲にスクロール追従する

**確認結果:**

- [x] OK

---

### ケース 5: キーボード操作（↑/↓ セクション境界）

**手順:**

1. `/` だけ入力した状態で `↓` キーを連打
2. Claude Code セクションの最終行を超えたところで `↓` を押す

**期待結果:**

- activeIndex のハイライトが Claude Code 最終行から `Templates` セクションの先頭行に移動する
- さらに `↓` で templates セクション内を移動
- `Templates` 最終行で `↓` を押しても動かない（リスト末尾）
- `↑` で逆方向に同様に動く

**確認結果:**

- [x] OK

**確認結果:**

- [x] OK
