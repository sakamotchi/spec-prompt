# テスト手順書 - prompt-palette-f3

F3 で担保する **受け入れ基準 9, 10, 11, 13, 14, 15**（元要件 §7）の手動確認と自動テストの実行方法。14 は F1 からの繰越。

## 概要

可能な限り `npx tauri dev` 上での手動操作で確認する。ユニット／コンポーネントテストは vitest。F1/F2 で担保済みの項目は回帰確認のみ。

## 前提条件

- `feature/prompt-edit-palette` ブランチをチェックアウト済み（F1 `d6076ed`, F2 `2773a53` を含む）
- `npm install` 済み
- `npx tauri dev` でアプリが起動している
- プロジェクトフォルダを開き、ターミナルタブを複数（最低 2 つ）起動
- macOS を主対象（日本語 IME は「ことえり」で確認）

## 手動テスト

### ケース 1: 下書きが復元される（受け入れ基準 9）

**手順:**

1. タブ A でパレットを開き `下書き A` と入力する。
2. `Esc` でパレットを閉じる。
3. 同じタブ A で `Cmd+Shift+P` で再度パレットを開く。

**期待結果:**

- textarea に `下書き A` が復元されている
- キャレット位置は末尾

**確認結果:**

- [x] OK（2026-04-18 確認）

### ケース 3: タブ閉鎖で下書きが破棄される（受け入れ基準 11）

**手順:**

1. タブ A でパレットを開き `A の下書き` と入力し `Esc` で閉じる。
2. タブ B でパレットを開き `B の下書き` と入力し `Esc` で閉じる。
3. タブ A を閉じる（`Cmd+W` または右クリック閉じる）。
4. タブ B で `Cmd+Shift+P` でパレットを開く。
5. 新規タブ C を作成し、`Cmd+Shift+P` でパレットを開く。

**期待結果:**

- 4 でタブ B の textarea に `B の下書き` が復元される
- 5 でタブ C の textarea は空（タブ A の下書きは破棄済み、B のみ残っている）

**確認結果:**

- [x] OK（2026-04-18 確認）

**期待結果:**

- パレットが自動で閉じる
- `中断対象` の下書きは破棄されている（次のタブ A 相当が作られても復元されない）

**確認結果:**

- [x] OK（2026-04-18 確認）（再現が難しい場合はスキップ可）

### ケース 7: グローバルショートカット抑止（受け入れ基準 14、F1 繰越）

**手順:**

1. パレットを開き、textarea にフォーカスがある状態で以下を順に試す:
   - `Ctrl+Tab`（本来コンテンツ/ターミナル切替）
   - `Ctrl+Shift+Tab`（本来前のタブへ）
   - `F2`（本来タブリネーム相当）
   - `Cmd+T`（本来ターミナルタブを新規作成）
   - `Cmd+W`（本来アクティブタブを閉じる）
   - `Cmd+0`（本来ツリーへフォーカス）
   - `Cmd+1`（本来 1 番目のタブへ）
   - `Cmd+\`（本来コンテンツ分割）
   - `Cmd+Shift+\`（本来ターミナル分割）
   - `?`（本来ショートカット一覧）

**期待結果:**

- いずれのショートカットも発火せず、textarea の入力として扱われる（または何もしない）
- 画面は切り替わらない、タブは増減しない、ショートカット一覧モーダルは開かない

**確認結果:**

- [x] OK（2026-04-18 確認）

### ケース 9: 送信失敗時のトースト + パレット維持（受け入れ基準 15）→ **手動 E2E はスキップ**

**状態:** 手動 E2E ではスキップ（コンポーネントテストで代替担保）

**理由:** `write_pty` 失敗を手動で再現するには Rust 側を一時改造する必要があり、F3 の通常開発フローで扱うコストが割に合わない。代わりに以下の自動テストで受け入れ基準 15 を担保する:

- `src/components/PromptPalette/PromptPalette.test.tsx`
  - 「writePty が reject した場合、toast.error が呼ばれパレットと本文を維持する」
  - `writePty` を `mockRejectedValueOnce(new Error('PTY not found'))` して、`toast.error` が `promptPalette.error.sendFailed` キーを含むメッセージで 1 回呼ばれること、`isOpen=true` と `drafts['pty-1']='echo hi'` が維持されることを検証

**本来の期待結果（実障害時の挙動仕様）:**

- エラートーストが表示される（`プロンプトの送信に失敗しました: ...`）
- パレットは閉じず、本文も残っている
- 下書きはクリアされていない（再送信可能）

**確認結果:**

- [x] スキップ（自動テストで代替）
