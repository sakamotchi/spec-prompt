# テスト手順書 - Phase 1-D: 状態管理基盤（Zustand persist）

## 概要

Phase 1-D（Zustand persist）のテスト手順を記載します。アプリを再起動して前回のセッション状態が復元されることを手動で確認します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト用プロジェクトフォルダが準備されていること（spec-prompt リポジトリ自体で可）

---

## 手動テスト

### ケース 1: プロジェクト状態の復元

**手順:**

1. アプリを起動する
2. 「開く」ボタンをクリックして `spec-prompt` リポジトリを開く
3. ファイルツリーの `src/` ディレクトリを展開する
4. `src/App.tsx` をクリックして選択状態にする
5. アプリを終了する（`Cmd+Q`）
6. アプリを再起動する

**期待結果:**

- 再起動後、`spec-prompt` リポジトリのファイルツリーが自動で表示される
- `src/` ディレクトリが展開された状態になっている
- `src/App.tsx` がハイライト（選択）された状態になっている

**確認結果:**

- [x] OK / NG

---

### ケース 2: アクティブタブの復元

**手順:**

1. 右ペインの「ターミナル」タブをクリックしてアクティブにする
2. アプリを終了する
3. アプリを再起動する

**期待結果:**

- 再起動後、「ターミナル」タブがアクティブな状態で起動する

**確認結果:**

- [x] OK / NG

---

### ケース 3: fileTree は再取得される

**手順:**

1. プロジェクトを開いた状態でアプリを終了・再起動する
2. 開発者ツール（`F12`）→ Application → Local Storage → `spec-prompt-app-store` を確認する

**期待結果:**

- `state.fileTree` キーが存在しない（保存されていない）
- 再起動後のツリーは `read_dir` で新たに取得されている

**確認結果:**

- [x] OK（自動テスト `it('fileTree は localStorage に保存されない')` で確認済み。Tauri アプリでは開発者ツールを右クリック → Inspect で開く）

---

### ケース 4: 新規起動時（localStorage 未設定）

**手順:**

1. 開発者ツールで `spec-prompt-app-store` の localStorage を削除する
2. アプリをリロードする

**期待結果:**

- エラーなく起動する
- ファイルツリーは空の状態（プロジェクト未選択）になる

**確認結果:**

- [x] OK

---

## 自動テスト

### フロントエンドテスト（Vitest）

```bash
npm run test
```

確認項目:
- `appStore` persist: `projectRoot` が localStorage に保存される
- `appStore` persist: `expandedDirs` の Set が正しくシリアライズされる
- `appStore` persist: `fileTree` は localStorage に保存されない

```bash
npm run lint

```
