# テスト手順書 - file-tree-dnd

## 概要

このドキュメントでは、ファイルツリーのドラッグ＆ドロップ機能（内部移動 / Finder からのコピー）のテスト手順を記載する。可能な限り手動操作で確認し、自動化困難な OS DnD は手動で、純関数ロジック（自己/子孫判定）と Rust コマンドは自動テストでカバーする。

## 前提条件

- macOS 環境
- `npx tauri dev` でアプリが起動していること
- テスト用プロジェクトを 1 つ用意する。最低限以下の構造を含むこと：
  - `docs/draft/01_要件.md`
  - `docs/draft/02_設計.md`
  - `docs/draft/sub/`（空でもよい）
  - `docs/reviewing/`
  - `assets/`（空）
- Finder で別途 PNG ファイル 3 枚と適当なサブフォルダ（中にファイルあり）を用意

**期待結果:**

## 自動テスト

### フロントエンドテスト（Vitest）

```bash
npm run test
```

確認対象:

- `useTreeDnd` の自己／子孫判定、同一親フォルダへの no-op、件数集計
- ハイライトに使う `dragOverPath` の更新

### 型チェック

```bash
npm run build
```

### Rustテスト

```bash
cd src-tauri && cargo test
```

確認対象:

- `copy_path` のファイル単体コピー
- `copy_path` のディレクトリ再帰コピー
- `copy_path` の同名衝突エラー
- 既存 `rename_path` 系テストがパス

### Rustの型チェック

```bash
cd src-tauri && cargo check
```
