# テスト手順書 - dynamic-tab-title（P2）

## 概要

P2（OSC 0/1/2 による動的タブタイトル）の手動テストおよび自動テスト手順。OSC シーケンスは `printf` で容易に再現できるため、手動確認を中心に行う。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- Phase 1 の `DisplayTitleCache` / `set_pty_display_title` / `setPtyDisplayTitle` が導入済みで動作していること（未マージの場合はブランチをマージ / リベースして結合する）
- アプリ内設定で「Claude Code 通知」が ON になっていること（ケース 5 で必要）

---

## 手動テスト

### ケース 1: OSC 2 でタブラベルが自動更新される

**手順:**

1. アプリを起動する
2. 統合ターミナルのタブ 1 で以下を実行:
   ```bash
   printf '\033]2;hello world\007'
   ```
3. タブラベルを確認する

**期待結果:**

- タブラベルが `hello world` に変化する（`Terminal 1` → `hello world`）
- 他のタブのラベルは変化しない

**確認結果:**

- [ ] OK / NG

---

### ケース 2: 空 OSC 2 でフォールバック名に戻る

**手順:**

1. ケース 1 を実行してタブ 1 が `hello world` になっている状態から
2. 同じタブで以下を実行:
   ```bash
   printf '\033]2;\007'
   ```
3. タブラベルを確認する

**期待結果:**

- タブラベルが `Terminal 1` に戻る

**確認結果:**

- [ ] OK / NG

---

### ケース 3: 複数タブで独立にラベルが変化する

**手順:**

1. タブを 2 つに増やす（`Terminal 1`, `Terminal 2`）
2. タブ 1 で `printf '\033]2;alpha\007'`
3. タブ 2 で `printf '\033]2;beta\007'`
4. 両タブのラベルを確認
5. タブ 2 のみ `printf '\033]2;\007'` でリセット

**期待結果:**

- 手順 4: タブ 1 は `alpha`、タブ 2 は `beta`
- 手順 5 後: タブ 1 は `alpha` のまま、タブ 2 は `Terminal 2`

**確認結果:**

- [ ] OK / NG

---

### ケース 4: 制御文字を含むタイトルがサニタイズされる

**手順:**

1. 任意のタブで以下を実行:
   ```bash
   printf '\033]2;ab\x01cd\x7Fe\007'
   ```
2. タブラベルを確認

**期待結果:**

- タブラベルが `abcde`（`\x01` と `\x7F` が除去される）

**確認結果:**

- [ ] OK / NG

---

### ケース 5: OSC タイトル変化が OS 通知タイトルにも反映される（P1 連携）

**手順:**

1. アプリ設定で通知 ON を確認
2. タブ 1 で `printf '\033]2;vim foo.ts\007'`
3. アプリをバックグラウンドにする（Cmd+H 等）
4. タブ 1 で `printf '\033]9;保存しました\007'` を実行して通知発火

**期待結果:**

- OS 通知のタイトルが `Claude Code — vim foo.ts`
- 本文は `保存しました`

**確認結果:**

- [ ] OK / NG

---

### ケース 6: OSC タイトルをリセットすると通知タイトルもフォールバックに戻る

**手順:**

1. ケース 5 の続きで、アプリをフォアグラウンドに戻す
2. タブ 1 で `printf '\033]2;\007'`（リセット）
3. 再度アプリをバックグラウンドにして `printf '\033]9;テスト\007'`

**期待結果:**

- OS 通知のタイトルが `Claude Code — Terminal 1`（フォールバック名）

**確認結果:**

- [ ] OK / NG

---

### ケース 7: 長いタイトルが UI を崩さない

**手順:**

1. 任意のタブで以下を実行:
   ```bash
   printf '\033]2;This is a very very very long title that would certainly overflow the tab label area if not truncated correctly\007'
   ```
2. タブラベルの見た目を確認
3. ラベルにマウスホバー

**期待結果:**

- タブ幅は崩れない（`max-width` + `truncate` で省略表示）
- マウスホバー時にツールチップで完全な文字列が表示される

**確認結果:**

- [ ] OK / NG

### ケース 8: 日本語タイトルが正しく表示される

**手順:**

1. 任意のタブで `printf '\033]2;作業中のドキュメント\007'`
2. タブラベルを確認

**期待結果:**

- タブラベルが `作業中のドキュメント`
- 文字化けなし

**確認結果:**

- [ ] OK / NG

### ケース 9: 同一タイトル連続更新でパフォーマンス劣化がない（任意）

**手順:**

1. 任意のタブで以下を実行:
   ```bash
   for i in {1..100}; do printf '\033]2;same\007'; done
   ```
2. タブラベルと CPU 使用を確認

**期待結果:**

- タブラベルは `same` のまま
- CPU スパイクが観測されない（Zustand の同一値スキップが効いている）

**確認結果:**

- [ ] OK / NG

---

### ケース 10: ResetTitle（OSC 2 リセット相当）でタブがフォールバックに戻る

**手順:**

1. タブ 1 で `printf '\033]2;custom\007'` → ラベル `custom`
2. タブ 1 を閉じずに新しいシェルセッションを意識した再初期化
   - 実運用では `exec zsh -l` や `reset` コマンドで再現可能
   - 再現が難しければケース 2 で代替

**期待結果:**

- タブラベルが `Terminal 1` に戻る

**確認結果:**

- [ ] OK / NG
