# プロダクト要求定義書（PRD）

**バージョン**: 1.1
**作成日**: 2026年3月28日
**最終更新**: 2026年4月14日

---

## 1. プロダクト概要

### 1.1 プロダクト名
SpecPrompt

### 1.2 目的
生成AIを用いた仕様駆動開発を効率的に実施するための軽量デスクトップアプリケーション。VS Codeの重さに依存せず、マークダウンプレビュー・AI対話・ドキュメント管理を一つの軽量アプリで実現する。

### 1.3 ビジョン
「仕様書を見ながらAIと対話し、コードを生み出す──その全てを一つの軽量ツールで」

### 1.4 主要な価値提案
- **軽量性**: VS Codeの約1/5以下のメモリ使用量（目標 200MB 以下）
- **統合ターミナル**: Claude Code などのAI CLIツールをアプリ内で直接実行
- **マークダウンプレビュー**: 仕様書をリアルタイムにレンダリング表示
- **パス入力支援**: ファイルツリーからターミナルへワンアクションでパスを挿入
- **AI 通知連携**: Claude Code の許可待ち・完了・エラーを OS ネイティブ通知で即時把握（発火元タブ名付き）

## 2. ターゲットユーザー

### 2.1 プライマリユーザー
| ユーザータイプ | 説明 | 主な利用シーン |
|--------------|------|--------------|
| 仕様駆動開発者 | AI CLIを使って仕様書からコードを生成する開発者 | 仕様書確認しながらAIと対話 |

### 2.2 ペルソナ例
**ペルソナ: 山田一郎（32歳・シニアエンジニア）**
- Claude Code を日常的に使用して仕様駆動開発を実践
- VS Code は重くて仕様書確認とターミナル操作の往復が面倒
- **ペインポイント**: 仕様書とターミナルを別々のウィンドウで行き来するのが非効率
- **期待**: 仕様書を見ながらAIに指示できる軽量な専用ツール

| 機能ID | 機能名 | 説明 | 状態 |

| 機能ID | 機能名 | 説明 | 状態 |

| FR-07 | ソースコード表示 | シンタックスハイライト付きコード表示（読み取り専用） | 未着手 |
| FR-04 | パス入力支援 | ツリーからターミナルへのパス挿入・Ctrl+Pパレット | 未着手 |
| FR-06 | プロジェクト管理 | 複数プロジェクト切り替え・最近開いたプロジェクト履歴 | 未着手 |

#### Phase 3（実装予定）: 補助機能
| 機能ID | 機能名 | 説明 | 状態 |
|--------|--------|------|------|
| FR-05 | ドキュメント状態管理 | フロントマターによる draft/reviewing/approved ステータス管理 | 未着手 |

#### Phase 4（実装済み）: AI 連携・ターミナル強化
| 機能ID | 機能名 | 説明 | 状態 |
|--------|--------|------|------|
| FR-09 | Claude Code 通知 | OSC 9 / HTTP hook を検出し OS ネイティブ通知を発火（発火元タブ名を差し込み） | 実装済み |
| FR-10 | 動的タブタイトル | OSC 0/1/2 によるターミナルタブ名の自動更新 | 実装済み |

| FR-13 | シェル終了自動クローズ | PTY exit でターミナルタブを自動クローズ | 実装済み |

### 3.2 スコープ外
- コードエディタ機能（テキスト編集）
- Git操作のGUI
- AI CLIツール自体の開発・内蔵

## 4. 非機能要件

### 4.1 性能要件
| 要件ID | 要件 | 目標値 |
|--------|------|--------|
| NFR-01 | アプリ起動時間 | 3秒以内 |
| NFR-02 | メモリ使用量 | 200MB以下 |
| NFR-03 | マークダウン更新遅延 | 500ms以内 |

### 4.2 ユーザビリティ要件
| 要件ID | 要件 | 説明 |
|--------|------|------|
| NFR-04 | 初期セットアップ不要 | インストール後すぐに基本機能が使える |
| NFR-05 | キーボードファースト | 主要操作がショートカットで完結（Ctrl+Tab、Ctrl+P等） |

### 4.3 対応プラットフォーム
| 要件ID | プラットフォーム | 優先度 |
|--------|----------------|--------|
| NFR-06 | macOS | 最優先 |
| NFR-07 | Windows | 第2優先 |
| NFR-08 | Linux | 第3優先 |
