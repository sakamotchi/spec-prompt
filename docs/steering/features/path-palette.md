# パス入力支援機能仕様書

**バージョン**: 1.1
**作成日**: 2026年3月28日
**最終更新**: 2026年4月18日

---

## 1. 概要

パス入力支援は、ファイルツリーやパス検索パレットを使って、ファイルパスをターミナルのカーソル位置に挿入する機能です。Claude CodeなどのAI CLIツールへのファイル参照を素早く行えます。

**機能ID**: FR-04

---

## 2. 機能要件

### 2.1 Ctrl+クリック挿入

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| PP-01 | ワンアクション挿入 | ファイルツリーのファイルを `Ctrl+クリック` すると、アクティブなターミナルのカーソル位置にパスが挿入される |
| PP-02 | ターミナルが開いている場合のみ動作 | ターミナルタブが1つも開いていない場合は何もしない（またはエラーを表示する） |
| PP-03 | メインタブ切り替えなし | ターミナルタブに切り替えずにパスを挿入できる（バックグラウンドでwritePtyを呼ぶ） |

### 2.2 右クリックメニュー挿入

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| PP-04 | コンテキストメニュー | ファイルツリーで右クリックするとメニューに「パスをターミナルに挿入」が表示される |
| PP-05 | アクティブターミナルに挿入 | 現在アクティブなターミナルタブに挿入する |

### 2.3 パス検索パレット（PathPalette）

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| PP-06 | Ctrl+P で起動 | `Ctrl+P` でパス検索パレットがオーバーレイ表示される |
| PP-07 | インクリメンタルサーチ | 文字を入力するたびにファイル名でリアルタイムに絞り込む |
| PP-08 | fuzzy match | 完全一致でなく、fuzzy matchで候補を絞り込む（大文字/小文字を無視） |
| PP-09 | 候補上限 | 表示する候補は上位100件に制限する |
| PP-10 | キーボード操作 | `↑↓` で候補選択、`Enter` で挿入、`Escape` でパレットを閉じる |
| PP-11 | マウス操作 | クリックで候補を選択・挿入できる |
| PP-12 | パレットを閉じる | `Escape` またはパレット外クリックで閉じる |

### 2.4 複数ファイル一括挿入

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| PP-13 | 複数選択 | `Ctrl+クリック` で複数ファイルを選択状態にする |
| PP-14 | 一括挿入 | 選択した複数ファイルのパスをスペース区切りで一括挿入する |

### 2.5 パス形式設定

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| PP-15 | 相対パス | プロジェクトルートからの相対パスで挿入する（デフォルト） |
| PP-16 | 絶対パス | 絶対パスで挿入する（設定ファイルで切り替え可能） |

### 2.6 プロンプト編集パレットとの連携（FR-15 追加分）

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| PP-17 | ディスパッチ分岐 | プロンプト編集パレット表示中（`isOpen && targetPtyId`）は PTY への `writePty` ではなく `promptPaletteStore.insertAtCaret` を呼ぶ |
| PP-18 | 呼び出し側は無改修 | `TreeNode` / `ContextMenu` / `PathPalette` の `insertPath` 呼び出しはそのまま。分岐は `usePathInsertion` の内部で行う |
| PP-19 | 挿入後のフォーカス | パレット開時は `terminal:focus` を発火させず textarea にフォーカスを維持 |
| PP-20 | Ctrl+P 確定後の戻し | パレット開時は `Ctrl+P` パレットを閉じて textarea にフォーカスを戻す（`Dialog.Content.onCloseAutoFocus`） |

## 3. 技術仕様

### 3.1 フロントエンド

**コンポーネント**: `src/components/PathPalette/`

**カスタムフック**: `src/hooks/usePathInsertion.ts`

```typescript
function usePathInsertion() {
  // activePtyId: 現在アクティブなターミナルのPTY ID
  // pathFormat: 'relative' | 'absolute'
  // insertPath(filePath: string | string[]): void
  //   1. paths を相対/絶対形式に整形
  //   2. プロンプト編集パレット表示中（isOpen && targetPtyId）なら
  //      promptPaletteStore.insertAtCaret(formatted) を呼ぶ（PTY へは書かない）
  //   3. それ以外はアクティブ PTY へ writePty(ptyId, formatted) を呼び、terminal:focus を発火
}
```

**パス形式変換**:

```typescript
function formatPath(filePath: string, projectRoot: string, format: PathFormat): string {
  if (format === 'relative') {
    return path.relative(projectRoot, filePath)
  }
  return filePath
}
```

**PathPaletteコンポーネントの動作**:

1. `useAppStore` からファイルツリーのフラットリストを生成
2. 入力文字列でファイル名をfuzzy matchで絞り込む
3. 上位100件に制限して候補リストを表示
4. `Enter` または候補クリックで `insertPath()` 呼び出し

### 3.2 状態管理

パス挿入に必要な状態は既存ストアから取得する。専用ストアは不要。

| 状態 | ストア | 用途 |
|------|--------|------|
| `activePtyId` | `terminalStore` | 挿入先のターミナルを特定 |
| `projectRoot` | `appStore` | 相対パス計算のベース |
| `fileTree` | `appStore` | パレットの候補リスト生成 |
| `pathFormat` | `appStore` または設定 | 相対/絶対パスの切り替え |
| `isPaletteOpen` | ローカルstate | パレットの表示/非表示 |

---

## 4. UX考慮事項

- Ctrl+クリック時にメインタブがターミナルでない場合でも挿入できること（バックグラウンド送信）
- パレットは現在のフォーカスを奪わず、挿入後にターミナルのフォーカスを維持する
- 候補が0件の場合は「ファイルが見つかりません」を表示する
